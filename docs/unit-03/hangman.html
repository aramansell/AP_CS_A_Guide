Hangman — Full Walkthrough

This is the program your teacher builds live. It combines String methods, conditionals, and character comparison to build a classic word-guessing game. Use this page as a reference when building your own projects.

The Goal

Pick a secret word. Let the player guess one letter at a time. Show which letters are revealed and which are still hidden.

=== HANGMAN ===
Secret word: _ _ _ _ _
Guess a letter: a
✗ No 'a' found.
_ _ _ _ _
Guess a letter: e
✓ Correct!
_ e _ _ _

Step 1: Set Up the Word and Display

String secret = "dragon";
int length = secret.length();

// Create a display showing underscores
String display = "";
for (int i = 0; i < length; i++) {
    display = display + "_ ";  // underscore + space
}
System.out.println("Secret word: " + display);

Why String instead of an array? Arrays come in Unit 6. For now, Strings and their methods (length(), charAt(), indexOf()) do the job. The display string grows as letters are revealed.

Step 2: Read a Guess and Check It

Scanner input = new Scanner(System.in);

System.out.print("Guess a letter: ");
String guessStr = input.nextLine();
char guess = guessStr.charAt(0);  // first character only

// Check if the letter exists in the secret word
if (secret.indexOf(guess) >= 0) {
    System.out.println("✓ Correct!");
} else {
    System.out.println("✗ No '" + guess + "' found.");
}

Socratic Checkpoint: String Extraction

Why do we use charAt(0) here? What would happen if the player typed multiple letters, or accidentally typed a space first?

Also, indexOf() returns -1 if the letter is not found. What does it return if the letter is found multiple times (like 'o' in "door")?

Step 3: Reveal Matching Letters

// Rebuild the display with the guessed letter revealed
String newDisplay = "";
for (int i = 0; i < length; i++) {
    if (secret.charAt(i) == guess) {
        newDisplay = newDisplay + guess + " ";  // show the letter
    } else if (display.charAt(i * 2) != '_') {
        // Already revealed — keep it
        newDisplay = newDisplay + display.charAt(i * 2) + " ";
    } else {
        newDisplay = newDisplay + "_ ";  // still hidden
    }
}
display = newDisplay;
System.out.println(display);

Socratic Checkpoint: The * 2 Trick

Look at the else if (display.charAt(i * 2) != '_') line. Why are we multiplying i by 2?

Think about how the display string is formatted: "_ _ _ _ _ ". What character is at index 1? What is at index 3?

Step 4: Loop Until Solved

In Unit 4, wrap this in a while loop:

while (display.indexOf('_') >= 0) {
    // Prompt, guess, check, rebuild display
    // ... same logic as above ...
}
System.out.println("You guessed it! The word was: " + secret);

The Full Program

import java.util.Scanner;

public class Hangman {
    public static void main(String[] args) {
        Scanner input = new Scanner(System.in);
        String secret = "dragon";
        int length = secret.length();

        // Build initial display
        String display = "";
        for (int i = 0; i < length; i++) {
            display = display + "_ ";
        }

        System.out.println("=== HANGMAN ===");
        System.out.println("Secret word: " + display);

        // Game loop — one guess for now (Unit 4 adds looping)
        System.out.print("Guess a letter: ");
        String guessStr = input.nextLine();
        char guess = guessStr.charAt(0);

        if (secret.indexOf(guess) >= 0) {
            System.out.println("✓ Correct!");
        } else {
            System.out.println("✗ No '" + guess + "' found.");
        }

        // Reveal matching letters
        String newDisplay = "";
        for (int i = 0; i < length; i++) {
            if (secret.charAt(i) == guess) {
                newDisplay = newDisplay + guess + " ";
            } else if (display.charAt(i * 2) != '_') {
                newDisplay = newDisplay + display.charAt(i * 2) + " ";
            } else {
                newDisplay = newDisplay + "_ ";
            }
        }
        display = newDisplay;
        System.out.println(display);
    }
}

Extending the Program

  • Add a word bank — pick a random word from a list (arrays, Unit 6).
  • Track wrong guesses and draw the hangman.
  • Limit guesses to 6 (or 7 with a body) before game over.

How This Connects to Your RPG

The display-rebuilding loop is the same pattern you'll use to update HP bars, inventory lists, and dungeon maps. Start with a default display, modify it based on player actions, print the new version.
