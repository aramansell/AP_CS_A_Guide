De Morgan's Laws — Flipping Conditions

The Problem

// You want to say "neither A nor B":
// "I don't have a sword AND I don't have a bow" — wordy
// "I don't have a sword OR a bow" — natural language is ambiguous

// Enter De Morgan:
!(A || B)  ==  !A && !B
!(A && B)  ==  !A || !B

The Two Laws

Law 1: NOT (A OR B) = NOT A AND NOT B

// "You can't enter if you're a goblin OR a troll."
// Flip it with De Morgan:
if (!(isGoblin || isTroll))

// Is the same as:
if (!isGoblin && !isTroll)

Law 2: NOT (A AND B) = NOT A OR NOT B

// "You can't open the chest without BOTH the key AND the password."
if (!(hasKey && knowsPassword))

// Is the same as:
if (!hasKey || !knowsPassword)
// Missing EITHER one = failure.

Why This Matters

Both versions compile to the same thing. Pick the one that's easier to read. Sometimes the flipped version is much clearer. 

// AWKWARD:
if (!(score < 0 || score > 100))

// CLEARER (flip with De Morgan):
if (score >= 0 && score <= 100)

// EVEN CLEARER FOR SOME (valid range):
if (0 <= score && score <= 100)  // math notation style

Bug Hunt: The Range Trap

A student wants to check if a d20 roll is an extreme value (a 1 or a 20). They wrote this condition:

if (!(roll > 1 && roll < 20)) {
    System.out.println("Critical extreme!"); 
}

Later, they decide they hate using the ! operator and rewrite it like this:

if (roll < 1 && roll > 20) {
    System.out.println("Critical extreme!"); 
}
  1. Why will the rewritten code never print "Critical extreme!"?
  2. How should they have applied De Morgan's Law to correctly flip the && and the relational operators?
