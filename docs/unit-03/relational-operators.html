Relational Operators & if Statements

Asking Questions About Data

A relational operator compares two values and answers true or false.

int playerHP = 12;
int enemyHP = 8;

playerHP > enemyHP;   // true — 12 is greater than 8
playerHP < enemyHP;   // false
playerHP == enemyHP;  // false — are they equal?
playerHP != enemyHP;  // true — are they NOT equal?

Bug Hunt: The Assignment Trap

A student wrote this code to check if a player has exactly 5 potions:

int potions = 5;
if (potions = 5) {
    System.out.println("You have exactly 5 potions.");
}

When they try to compile it, Java throws an error: incompatible types: int cannot be converted to boolean.

  1. Why is Java confused about this condition? (Hint: What does a single = do?)
  2. Fix the code so it properly checks for equality.

The if Statement

if (condition) {
    // runs ONLY if condition is true
}

// After the if block closes, execution continues normally
int d20 = Dice.roll(20);

if (d20 >= 15) {
    System.out.println("You hit!");
}

The else Statement

if (d20 >= 15) {
    System.out.println("You hit!");
} else {
    System.out.println("You missed!");
}
// Exactly ONE of these blocks runs. Never both. Always one.

Tinker Challenge: Critical Rolls

Roll a d20 and check for criticals. Build a program that follows these rules:

  • If the roll is exactly 20: print "CRITICAL HIT!"
  • If the roll is exactly 1: print "CRITICAL MISS!"
  • Otherwise: just print the number rolled.

Test your program by forcing the d20 variable to be 20, 1, and 10 to ensure all paths work.

Indentation and Braces

// CORRECT:
if (condition) {
    doThing();
    doOtherThing();
}

// ALSO CORRECT (single statement, braces optional):
if (condition)
    doThing();

// NEVER DO THIS (no braces, indentation lies):
if (condition)
    doThing();
    doOtherThing();  // THIS ALWAYS RUNS — it's outside the if!

Socratic Checkpoint: The Lying Indentation

Look at the "NEVER DO THIS" example above. Why does doOtherThing() always run, regardless of the condition? What does Java actually see when you omit the braces?
