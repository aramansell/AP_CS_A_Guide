Unit 3: Boolean Expressions & if Statements

Everything you've written so far runs the same way every time: declare, calculate, display. But games don't work that way. Sometimes you hit. Sometimes you miss. Sometimes the enemy is a goblin (easy). Sometimes it's a dragon (run).

This unit gives your programs the power to make decisions. By the end, you'll build a Combat System: attack rolls, hit/miss checks, critical hits, damage calculation, and enemy AI — all driven by conditions.

Concepts

TopicWhat you'll learn
Relational Operators & if <, >, <=, >=, ==, != — asking questions about your data.
Boolean Logic &&, ||, ! — combining conditions. Short-circuit evaluation.
else-if Chains Multi-way decisions. The order matters trap.
Comparing Objects equals() and compareTo() in conditions. String comparisons in combat.
De Morgan's Laws Flipping conditions. "Not (A or B)" = "Not A and Not B."

Your RPG Project

Combat System (Student RPG Guide) — Your Player (from Unit 2) now fights. Roll d20 to attack. Compare against AC to hit. Roll damage on hit. Check for critical (natural 20). Enemy fights back. Your Dice.java from Unit 1 is the engine. Your Character Creator stats determine outcomes.

RPG Thread

Combat.java is the first RPG class that makes decisions. It checks: did I hit? Was it a critical? Is the monster still alive? In Unit 4, you wrap this combat logic in a while loop — the same hit/miss checks run every round until someone falls. In Unit 5, these methods move inside the Player and Monster classes. In Unit 9, polymorphic combat lets different enemy types use different tactics.

Prerequisites

Before starting this unit, you should be comfortable with:

  • Objects, Scanner, and String methods: Unit 2
