else-if Chains — Multi-Way Decisions

More Than Two Outcomes

if/else gives you two paths. What about five?

int roll = Dice.roll(20);

if (roll == 20) {
    System.out.println("CRITICAL HIT! Double damage!");
} else if (roll >= 15) {
    System.out.println("Hit!");
} else if (roll >= 10) {
    System.out.println("Glancing blow — half damage.");
} else if (roll == 1) {
    System.out.println("CRITICAL MISS! You stumble.");
} else {
    System.out.println("Miss!");
}

Java checks conditions top to bottom. The FIRST true condition runs. Then it skips everything else. Exactly ONE block executes.

Bug Hunt: The Order Matters Trap

A student wrote this code to assign grades based on a score:

int score = 85;
String grade = "";

if (score >= 60) {
    grade = "D";
} else if (score >= 70) {
    grade = "C";  
} else if (score >= 80) {
    grade = "B";  
} else if (score >= 90) {
    grade = "A";  
}

System.out.println("Your grade is: " + grade);

When the code runs, it prints Your grade is: D. Even though 85 should be a B!

  1. Trace the execution of the code. Why did Java choose the first block and ignore the rest?
  2. Fix the code. What is the rule of thumb for ordering if / else if chains when checking ranges?

Nested if Statements

if (roll >= 15) {
    System.out.println("Hit!");
    if (roll == 20) {
        System.out.println("It's a critical!");
        damage *= 2;
    }
} else {
    System.out.println("Miss!");
}

Nesting works. Don't go deeper than 2-3 levels or it becomes unreadable.

Socratic Checkpoint: Boolean Zen

Consider the following two code snippets. They do exactly the same thing.

Snippet A:

if (enemyAlive == true) { 
    System.out.println("Keep fighting!"); 
}

Snippet B:

if (enemyAlive) { 
    System.out.println("Keep fighting!"); 
}

Why is Snippet B preferred by Java developers? What does the if statement actually expect inside its parentheses?

Similarly, look at this method returning a boolean:

if (hp > 0) {
    return true;
} else {
    return false;
}

How could you rewrite this using a single line of code? (Hint: hp > 0 is an expression that evaluates to what?)
