Comparing Objects in Conditions

The Golden Rule (Again)

Primitives use ==. Objects use .equals().
You learned this in Unit 2. Now you need it in conditions. 

// PRIMITIVES — use ==
int hp = 10;
if (hp == 0) { ... }          // correct

// OBJECTS (String) — use .equals()
String weapon = "sword";
if (weapon.equals("sword"))   // correct
if (weapon == "sword")        // WRONG — unreliable!

Null-Safe Comparisons

String name = null;

// THIS CRASHES (NullPointerException):
if (name.equals("Frodo")) { ... }

// THIS IS SAFE:
if ("Frodo".equals(name)) { ... }

// OR check for null first:
if (name != null && name.equals("Frodo")) { ... }
// Short-circuit saves us: if name is null, .equals() never runs.

compareTo for Ordering

// compareTo returns int: negative, zero, or positive
String item = "potion";
int result = item.compareTo("shield");

if (result < 0) {
    // "potion" comes before "shield" alphabetically
} else if (result > 0) {
    // "potion" comes after
} else {
    // they're equal (result == 0)
}

Switch Statement (Preview)

When comparing one variable against many possible values, switch can be cleaner than long if/else chains. AP exam covers this minimally, but it's useful. 

String action = input.nextLine();

switch (action.toLowerCase()) {
    case "attack":
        combat();
        break;
    case "defend":
        defend();
        break;
    case "flee":
        flee();
        break;
    default:
        System.out.println("Unknown command.");
}

Bug Hunt: The Fall-Through Trap

A student wrote this switch statement for a shop menu:

String choice = "buy";
switch (choice) {
    case "buy":
        System.out.println("You bought an item!");
    case "sell":
        System.out.println("You sold an item!");
    default:
        System.out.println("Thanks for visiting.");
}

When the user chooses "buy", the program prints:

You bought an item!
You sold an item!
Thanks for visiting.

Why did it print all three? What keyword is missing from the case blocks?

Tinker Challenge: Combat Command Parser

Build a program that reads a string command from the user using Scanner and executes the corresponding action:

  • "attack": print "You swing your weapon!"
  • "defend": print "You brace for impact. Next hit does half damage."
  • "potion": print "You heal for 10 HP."
  • "flee": print "You try to run away..."

If they type anything else, print "Unknown command."

Socratic Checkpoint

What happens if the user types "Attack" (with a capital A) instead of "attack"? How can you use string methods to make your parser case-insensitive?
