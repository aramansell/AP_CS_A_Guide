Student RPG Guide: Combat System

This is YOUR guide. The teacher built Number Guessing, Hangman, and Choose Your Adventure with you. Now you use those same if/else patterns — plus your Dice.java engine — to make your character fight. Instead of code blanks, you are provided with Acceptance Criteria.

The Setup

Create a new file called Combat.java and another called CombatTester.java. In Combat.java, you will write methods to handle fighting. You do not need to store state (like HP) inside the Combat class itself; instead, the methods will take HP as a parameter and return the new HP.

Part 1: playerAttack()

Requirements:

  • Method signature: public static int playerAttack(int attackBonus, int targetAC, int damageDie, String targetName)
  • Behavior:
    1. Rolls a d20.
    2. Critical Hit: If the d20 is a 20, it's an automatic hit. Print a dramatic message. Roll the damageDie, multiply it by 2, print the damage, and return the damage.
    3. Critical Miss: If the d20 is a 1, it's an automatic miss. Print a failure message. Return 0.
    4. Standard Attack: Add the d20 roll to the attackBonus. Compare the total to the targetAC.
      • If total >= targetAC, it's a hit. Roll the damageDie, print a success message with the damage, and return the damage.
      • If total < targetAC, it's a miss. Print a miss message. Return 0.

Socratic Checkpoint:

In a standard attack, why do we use >= instead of == when comparing the total to the targetAC? What would happen if we used ==?

Acceptance Test:

Add this to CombatTester.java:

int dmg = Combat.playerAttack(3, 14, 8, "Goblin");
System.out.println("Damage returned: " + dmg);

Run it many times. Verify that a natural 20 always hits (even if 20+3 < AC, though 23 is usually high enough), a natural 1 always misses, and standard rolls behave correctly. Ensure the returned damage matches the printed message.

Part 2: monsterAttack()

Requirements:

  • Method signature: public static int monsterAttack(int attackBonus, int targetAC, int damageDie, String monsterName)
  • Behavior: Monsters don't get critical hits or misses in our system (to keep it simple). They only do standard attacks.
    1. Roll a d20.
    2. Add the attackBonus.
    3. If the total >= targetAC, roll damageDie, print a message, and return the damage.
    4. Otherwise, print a miss message and return 0.

Tinker Challenge:

How much of the logic from playerAttack can you reuse here? Is there a way to write this without copying and pasting the standard attack logic? (If you can't figure it out, copy-pasting is okay for now, but we will optimize it later).

Part 3: combatRound() — One Full Exchange

Requirements:

  • Method signature: public static int combatRound(int playerAtk, int monsterAC, int playerDmg, int monsterAtk, int playerAC, int monsterDmg, String monsterName, int monsterHP)
  • Behavior:
    1. Print a header like "--- Combat Round ---".
    2. The player attacks the monster (call playerAttack).
    3. Subtract the returned damage from monsterHP. Print the monster's remaining HP.
    4. Conditional Counterattack: The monster should ONLY attack the player if the monster is still alive (HP > 0). If the monster is alive, call monsterAttack and print how much damage the player took. If the monster is dead, print a victory message.
    5. Return the monsterHP.

Socratic Checkpoint:

Why does combatRound need to return monsterHP? Look at the parameters. If we didn't return it, how would the next round of combat know the monster's new HP?

Acceptance Test:

int goblinHP = 15;
goblinHP = Combat.combatRound(3, 14, 8, 2, 13, 6, "Goblin", goblinHP);
System.out.println("Goblin HP after round 1: " + goblinHP);
goblinHP = Combat.combatRound(3, 14, 8, 2, 13, 6, "Goblin", goblinHP);
System.out.println("Goblin HP after round 2: " + goblinHP);

Run it. If the player kills the Goblin in round 1, does the Goblin still attack back? (It shouldn't!). Does the HP properly carry over from round 1 to round 2?
