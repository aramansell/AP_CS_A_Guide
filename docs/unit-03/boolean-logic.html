Boolean Logic — &&, ||, !

One Condition Isn't Enough

"Did I hit?" is one condition. "Did I hit AND is the enemy still alive?" is two. Boolean operators let you combine conditions. 

int roll = Dice.roll(20);
boolean enemyAlive = true;

if (roll >= 15 && enemyAlive) {
    System.out.println("Hit!");
}

The Three Operators

OperatorNameTrue when...
&&ANDBOTH conditions are true
||ORAT LEAST ONE condition is true
!NOTThe condition is FALSE

AND (&&) — Both Must Be True

// To hit: roll high enough AND enemy must be alive
if (roll >= 15 && enemy.isAlive()) {
    // hit!
}

// Truth table:
// true  && true  → true
// true  && false → false
// false && true  → false
// false && false → false

OR (||) — At Least One

// Player wins: enemy dead OR enemy surrendered
if (enemyHP <= 0 || enemySurrendered) {
    System.out.println("Victory!");
}

// Truth table:
// true  || true  → true
// true  || false → true
// false || true  → true
// false || false → false

NOT (!) — Flip It

boolean doorLocked = false;
if (!doorLocked) {
    System.out.println("The door opens.");
}

// !true  → false
// !false → true

Short-Circuit Evaluation

Java is lazy. It evaluates left to right and STOPS as soon as it knows the answer. 

// AND short-circuits on first false:
if (enemy != null && enemy.isAlive()) {
    // If enemy is null, enemy.isAlive() NEVER runs!
    // This prevents a NullPointerException.
}

// OR short-circuits on first true:
if (roll == 20 || roll >= 15 && enemyWeak) {
    // Wait — what order does this evaluate?
    // && has higher precedence than ||.
    // This is: roll == 20 || (roll >= 15 && enemyWeak)
}

// Use parentheses to be clear:
if (roll == 20 || (roll >= 15 && enemyWeak)) {
    // Unambiguous.
}

Operator Precedence

! → && → ||
When in doubt, use parentheses. They cost nothing and prevent bugs. 

// These are DIFFERENT:
if (a && b || c)         // (a && b) || c
if (a && (b || c))       // different!

// Be explicit:
if ((a && b) || c)       // clear intent

Tinker Challenge: Predict and Verify

Read the following code and predict what each statement will print. Write your predictions down before running the code.

int hp = 5;
boolean poisoned = true;
boolean shielded = false;

System.out.println(hp > 0 && !shielded);
System.out.println(hp <= 0 || poisoned);
System.out.println(!poisoned && shielded);
System.out.println(hp > 0 && (poisoned || shielded));

Socratic Checkpoint

  • Did any of your predictions not match the output?
  • Look closely at the last statement. In what order does Java evaluate hp > 0, poisoned, and shielded? How do the parentheses change the result?
