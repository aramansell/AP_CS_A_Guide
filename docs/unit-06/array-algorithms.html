Array Algorithms

These are the patterns you'll use forever. Every programming language has arrays, and these algorithms work in all of them. Learn them once. Use them everywhere.

Find Maximum

int[] arr = {14, 8, 16, 12, 10, 18, 7};

int maxIndex = 0;                       // start assuming first is max
for (int i = 1; i < arr.length; i++) {  // check rest
    if (arr[i] > arr[maxIndex]) {       // found bigger?
        maxIndex = i;                   // update champion
    }
}
System.out.println("Max: " + arr[maxIndex] + " at index " + maxIndex);
// → Max: 18 at index 5

Why start at index 0? Because it's a real array value. Why start loop at 1? Because we've already "checked" index 0.

Find Minimum

int minIndex = 0;
for (int i = 1; i < arr.length; i++) {
    if (arr[i] < arr[minIndex]) {   // only difference: < instead of >
        minIndex = i;
    }
}

Sum and Average

int sum = 0;
for (int value : arr) {
    sum += value;
}
double avg = (double) sum / arr.length;  // cast to double!
Integer division trap: sum / arr.length with both ints gives integer division (truncates). Cast one to double: (double) sum / arr.length.

Linear Search

public static int find(int[] arr, int target) {
    for (int i = 0; i < arr.length; i++) {
        if (arr[i] == target) {
            return i;      // FOUND! Return position immediately.
        }
    }
    return -1;             // Not found. -1 means "invalid index."
}

// Usage:
int[] stats = {14, 8, 16, 12, 10};
int pos = find(stats, 16);   // → 2
int notFound = find(stats, 99);  // → -1

Why -1? All valid array indices are 0 or higher. -1 can never be confused with a real position.

Count Matches

int count = 0;
for (int value : arr) {
    if (value > 10) {       // whatever your condition
        count++;
    }
}

Swap Two Elements

// Swap positions a and b
int temp = arr[a];    // save a's value
arr[a] = arr[b];      // b → a
arr[b] = temp;        // saved a → b

You need the temp variable. If you do a = b; b = a;, both end up with b's original value. The temp saves a's value before it's overwritten.

Reverse an Array (In Place)

int[] arr = {10, 20, 30, 40, 50};

for (int i = 0; i < arr.length / 2; i++) {
    int temp = arr[i];
    arr[i] = arr[arr.length - 1 - i];
    arr[arr.length - 1 - i] = temp;
}
// arr is now {50, 40, 30, 20, 10}

Swap first with last, second with second-to-last, etc. Stop at the middle (length / 2) — going past the middle swaps everything back!

Shift Elements Right (Insert at Front)

// Shift everything right one position (must go backwards!)
for (int i = arr.length - 1; i > 0; i--) {
    arr[i] = arr[i - 1];
}
arr[0] = newValue;  // insert new value at front
Must go backwards! If you go forward (i=0,1,2...), you'd copy arr[0] to arr[1], then copy that NEW arr[1] to arr[2], and the entire array fills with the original arr[0]. Going backwards means each element moves into a slot that's already been vacated.

Socratic Checkpoint

1. The Champion's Start

In the "Find Maximum" algorithm, we start the loop at int i = 1 instead of 0. Why? What would happen if the array only had exactly 1 element?

2. The Magic Number

When searching for a specific item in an array, we return -1 if it isn't found. Why can't we just return 0 to mean "not found"?

Bug Hunt: The Double Reverse

A student wrote code to reverse their array of loot values, but when they print it out, the array looks exactly the same as when it started! Find the bug.

int[] loot = {10, 50, 100, 500};
for (int i = 0; i < loot.length; i++) {
    int temp = loot[i];
    loot[i] = loot[loot.length - 1 - i];
    loot[loot.length - 1 - i] = temp;
}

Question: How many times does this loop run? Try tracing what happens to the number 10 on the first iteration and the last iteration.
