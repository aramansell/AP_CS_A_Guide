Student RPG Guide: Party Arrays + Multi-Combat

One file: PartyManager.java. Your RPG has fought one monster at a time since Unit 3. This guide replaces that with a party of 4 heroes fighting 3+ enemies simultaneously. Arrays of objects, null-checking, helper methods, and a turn-based combat loop.

Spec: PartyManager.java

All methods are static. No instance variables this time — just a utility class with a main() that ties everything together.

Part 1: Party Setup (in main)

  • Create a Player[] of size 4
  • Fill each slot with a different character:
    • [0] Warrior — HP:30 STR:18 DEX:12 INT:10 CON:14
    • [1] Mage — HP:18 STR:10 DEX:14 INT:18 CON:12
    • [2] Rogue — HP:22 STR:14 DEX:18 INT:12 CON:10
    • [3] Cleric — HP:26 STR:12 DEX:10 INT:14 CON:16
  • Print all four party members using a loop
  • Important: slots start null. You must assign before using.

Part 2: Helper Methods

MethodReturnsWhat It Does

countAlive(Monster[] enemies) int Same, for monsters. Method overloading: same name, different parameter type.
pickRandomAlive(Player[] party) Player Pick a random alive player. Return null if everyone is dead. Algorithm: loop to find alive targets, pick one at random.
pickRandomAlive(Monster[] enemies) Monster Same, for monsters. More overloading.
allDead(Player[] party) boolean Is every party member dead or null? (countAlive == 0)
allDead(Monster[] enemies) boolean Same, for monsters.

Part 3: multiCombat()

Signature: public static void multiCombat(Player[] party, Monster[] enemies)

This is a simplified combat loop. Don't worry about turns, initiative, or targeting choices — just the core pattern.

The outer loop — while both sides have alive members:

  1. Party attacks each round: For each alive player:
    • Pick a random alive enemy as target (use pickRandomAlive(enemies))
    • If no targets remain, break
    • Attack roll: d20 + (player.strength / 4). If >= enemy armorClass, it hits.
    • Damage: d6 + (player.strength / 4)
    • Call enemy.takeDamage(damage)
    • If enemy dies: print "<enemy> has been slain!"
    • Check if all enemies are dead → break out of party loop early
  2. Enemies attack back: For each alive enemy:
    • Pick a random alive party member
    • If no targets, break
    • Attack roll: d20 + (enemy.armorClass / 4). If >= player AC (use a fixed AC of 14), it hits.
    • Damage: enemy.attack() (which handles boss advantage)
    • Call player.takeDamage(damage)
    • If player dies: print "<player> has fallen!"
    • Check if all party members are dead → break early

After the loop: Print VICTORY or DEFEAT. Print total monsters defeated using Monster.getDefeatedCount().

Test It

In main(), after party setup:

// === Test 1: countAlive ===
System.out.println("Party alive: " + countAlive(party) + "/" + party.length);
// Expected: 4/4

// === Test 2: pickRandomAlive returns a valid player ===
Player target = pickRandomAlive(party);
System.out.println(target != null);  // true

// === Test 3: allDead on fresh party ===
System.out.println(allDead(party));  // false

// === Test 4: Create enemy group ===
Monster[] enemies = new Monster[3];
enemies[0] = new Monster("Goblin", 12, 12, 6);
enemies[1] = new Monster("Skeleton", 18, 13, 8);
enemies[2] = new Monster("Orc", 20, 14, 10);

// === Test 5: Run combat ===
multiCombat(party, enemies);
// Should see rounds print, some hits, some deaths, and a winner

// === Test 6: After combat — defeatedCount ===
System.out.println("Monsters killed: " + Monster.getDefeatedCount());
// Should be > 0 (unless party wiped immediately)

// === Test 7: Null safety — party with gaps ===
Player[] partialParty = new Player[3];
partialParty[0] = new Player("Solo", 20, 14, 12, 10, 10);
// slots 1 and 2 are null
System.out.println(countAlive(partialParty));   // 1 — nulls skipped
System.out.println(allDead(partialParty));       // false
System.out.println(pickRandomAlive(partialParty).getName()); // "Solo"
