Array Basics

An array is a single variable that holds multiple values of the same type. Think of it as a row of numbered mailboxes. Each mailbox has a number (the index) and can hold one value.

Why Arrays Exist

Without arrays, tracking 20 inventory items looks like this:

String item1 = "Sword";
String item2 = "Potion";
String item3 = "Shield";
// ... 17 more variables ...
System.out.println(item1);
System.out.println(item2);
// ... 18 more printlns ...

With arrays:

String[] items = new String[20];
items[0] = "Sword";
items[1] = "Potion";
// ...
for (int i = 0; i < 20; i++) {
    System.out.println(items[i]);
}

One variable. One loop. Done.

Declaring and Creating

Method 1: Declare + Allocate (empty)

type[] name = new type[size];

int[] scores = new int[5];       // 5 zeros: {0, 0, 0, 0, 0}
String[] names = new String[3];  // 3 nulls: {null, null, null}
double[] temps = new double[10]; // 10 zeros: {0.0, 0.0, ...}

Method 2: Declare + Initialize (with values)

int[] stats = {14, 8, 16, 12, 10};
String[] party = {"Hero", "Mage", "Rogue", "Cleric"};

Java counts the values and sets length for you. No new needed.

Accessing Elements (Indexing)

int[] arr = {10, 20, 30, 40, 50};

arr[0]  // → 10    (THE FIRST ELEMENT IS INDEX 0)
arr[1]  // → 20
arr[2]  // → 30
arr[3]  // → 40
arr[4]  // → 50    (THE LAST ELEMENT IS INDEX length-1)

arr[2] = 99;       // arr is now {10, 20, 99, 40, 50}
FIRST INDEX IS 0. Not 1. A 5-element array has indices 0, 1, 2, 3, 4. Accessing arr[5] crashes your program.

The length Property

Every array has a length field. No parentheses!

int[] arr = new int[5];
System.out.println(arr.length);  // 5

// NOT arr.length() — that's String!
// NOT arr.size()  — that's ArrayList!
TypeHow to get size
Arrayarr.length (no parens)
Stringstr.length() (with parens)
ArrayListlist.size() (with parens)

ArrayIndexOutOfBoundsException

This is the error you'll see most often. It means you tried to access an index that doesn't exist. 

int[] arr = new int[5];

arr[-1] = 10;   // CRASH! Negative index.
arr[5] = 10;    // CRASH! Index 5 doesn't exist (valid: 0-4).
arr[100] = 10;  // CRASH! Way past the end.

Valid indices always: 0 through arr.length - 1.

Default Values

When you use new type[size], Java fills every slot with a default:

TypeDefault
int[]0
double[]0.0
boolean[]false
Object arrays (String[], Player[])null
Null trap: String[] names = new String[3]; creates three slots, all null. Calling names[0].length() crashes with NullPointerException. Fill the array before using it.
