Traversing Arrays

Traversal = visiting every element in order. You do this to print, sum, count, search, or modify. Two tools: standard for loop and enhanced for loop.

Standard for Loop (Index-Based)

for (int i = 0; i < arr.length; i++) {
    // arr[i] — read or write
    // i      — know your position
}

This is your power tool. You can do everything with it:

int[] hp = {20, 15, 8, 22, 7};

// Print with index
for (int i = 0; i < hp.length; i++) {
    System.out.println("Member " + i + ": " + hp[i] + " HP");
}

// Modify elements
for (int i = 0; i < hp.length; i++) {
    hp[i] += 5;  // heal everyone by 5
}

// Iterate backwards
for (int i = hp.length - 1; i >= 0; i--) {
    System.out.println(hp[i]);  // prints last to first
}

Enhanced for Loop (for-each)

for (type variable : array) {
    // variable = current element (read-only copy)
}

"For each element in the array, do this." Clean, safe, no index to mess up: 

int[] hp = {20, 15, 8, 22, 7};

// Print all
for (int h : hp) {
    System.out.println(h + " HP");
}

// Sum all
int total = 0;
for (int h : hp) {
    total += h;
}

// Count condition
int critical = 0;
for (int h : hp) {
    if (h < 10) critical++;
}
THE TRAP: The loop variable is a copy. Changing it does NOT change the array: 
for (int h : hp) {
    h = 999;   // changes the local copy — array unchanged!
}
If you need to modify array elements, use the standard for loop.

Rule of Thumb

Start with enhanced for. It's safer — impossible to get the bounds wrong. If you hit a "can't do this with enhanced for" limit, switch to standard for. When in doubt, standard for always works.

The Accumulator Pattern

The most common array pattern: a variable that "accumulates" results as you traverse. 

// SUM accumulator
int sum = 0;                   // start at 0
for (int value : arr) {
    sum += value;              // add each element
}

// COUNT accumulator (conditional)
int count = 0;                 // start at 0
for (int value : arr) {
    if (value > threshold) {   // check condition
        count++;               // increment when true
    }
}

// STRING accumulator
String result = "";            // start empty
for (String s : words) {
    result += s + " ";        // build up a string
}

Off-By-One Errors

The most common traversal bug:

int[] arr = {10, 20, 30, 40, 50};

// WRONG: i <= arr.length runs 6 times for a 5-element array
for (int i = 0; i <= arr.length; i++) {  // CRASH on last iteration!
    System.out.println(arr[i]);
}

// RIGHT: i < arr.length runs exactly 5 times
for (int i = 0; i < arr.length; i++) {
    System.out.println(arr[i]);
}

If your loop condition uses <= with .length, check very carefully. Usually wrong.

Bug Hunt: The Ineffective Healer

The cleric casts a mass heal spell, but the party's HP doesn't go up. Find the bug.

int[] partyHP = {15, 20, 8, 12};
// Heal everyone by 10
for (int hp : partyHP) {
    hp += 10;
}
System.out.println("First member HP: " + partyHP[0]); // Still prints 15!

Question: How do you rewrite this loop so the healing actually applies to the partyHP array?

  1. Create an int[] called damageDealt with the values: {12, 5, 22, 9, 18}.
