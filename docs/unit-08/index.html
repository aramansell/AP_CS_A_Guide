Unit 8: 2D Arrays

Every array you've used so far has been one-dimensional — a line of values. But dungeon maps are grids. Game boards have rows and columns. Images are pixels arranged in width and height. You need a second dimension.

This unit introduces 2D arrays — arrays of arrays. Grid-based data structures for modeling boards, maps, and tables.

Concepts

TopicWhat you'll learn
2D Array Basics Declaration, initialization, rows vs columns, ragged arrays.
2D Array Traversal Row-major and column-major traversal. Nested loops for grid operations.
Neighbors & Boundaries Safe adjacent-cell checking. Bounds guards, direction arrays, edge cases.

Your RPG Project

GuideWhat you'll build
Dungeon Map (Student RPG Guide) Your dungeon gets a grid. Rooms, walls, player position, enemy spawns. Movement with WASD-style input. Map rendering in the console. The foundation for Unit 10's flood fill and backtracking.

RPG Thread

The dungeon grid you build here is the home for your entire RPG. Each cell holds terrain, enemies, items, or the player. In Unit 9, enemies on the grid become polymorphic — a Skeleton at [2][3] behaves differently from a Dragon at [5][1]. In Unit 10, flood fill lights connected rooms and backtracking finds paths through the grid. The 2D array is the single most reused structure in your RPG.

Prerequisites

Before starting this unit, you should be comfortable with:

  • ArrayList and dynamic collections: Unit 7
