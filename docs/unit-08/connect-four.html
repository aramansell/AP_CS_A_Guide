Connect Four — Full Walkthrough

This is the program your teacher builds live. It builds on Tic-Tac-Toe's 2D array patterns with added gravity — pieces fall to the lowest empty row in a column. Use this page as a reference when building your own projects.

The Goal

Build a two-player Connect Four game on a 6-row × 7-column board. Pieces drop to the bottom. Check for four-in-a-row in any direction.

 0 1 2 3 4 5 6
| | | | | | | |
| | | | | | | |
| | | | | | | |
| | | |X| | | |
| | |O|X| | | |
|O|X|O|X| | | |
---------------
Player X wins!

Step 1: The Board

public class ConnectFour {
    private char[][] board;  // 6 rows, 7 cols
    private char currentPlayer;

    public ConnectFour() {
        board = new char[6][7];
        for (int r = 0; r < 6; r++)
            for (int c = 0; c < 7; c++)
                board[r][c] = ' ';
        currentPlayer = 'X';
    }

Step 2: Drop a Piece (Gravity Logic)

    public boolean dropPiece(int col) {
        if (col < 0 || col > 6) {
            System.out.println("Column must be 0-6.");
            return false;
        }
        // Start from the bottom row, find first empty spot
        for (int row = 5; row >= 0; row--) {
            if (board[row][col] == ' ') {
                board[row][col] = currentPlayer;
                return true;
            }
        }
        System.out.println("Column is full!");
        return false;
    }

Walk upward from the bottom. Row 5 is the bottom, row 0 is the top. Find the first empty cell going up — that's where the piece lands.

Step 3: Check for Four-in-a-Row

Four directions: horizontal, vertical, and two diagonals. Check from every cell.

    public boolean checkWin() {
        // Check all four directions from every cell
        for (int r = 0; r < 6; r++) {
            for (int c = 0; c < 7; c++) {
                if (board[r][c] == ' ') continue;
                char player = board[r][c];

                // Horizontal (right)
                if (c <= 3 &&
                    board[r][c+1] == player &&
                    board[r][c+2] == player &&
                    board[r][c+3] == player) return true;

                // Vertical (down)
                if (r <= 2 &&
                    board[r+1][c] == player &&
                    board[r+2][c] == player &&
                    board[r+3][c] == player) return true;

                // Diagonal down-right
                if (r <= 2 && c <= 3 &&
                    board[r+1][c+1] == player &&
                    board[r+2][c+2] == player &&
                    board[r+3][c+3] == player) return true;

                // Diagonal up-right
                if (r >= 3 && c <= 3 &&
                    board[r-1][c+1] == player &&
                    board[r-2][c+2] == player &&
                    board[r-3][c+3] == player) return true;
            }
        }
        return false;
    }

Boundary checks prevent ArrayIndexOutOfBounds. Can't check horizontal at column 5 (needs 3 more to the right). Can't check vertical at row 4 (needs 3 more below). The bounds (c <= 3, r <= 2, etc.) ensure we stay in the array.

Step 4: Display and Game Loop

    public void display() {
        System.out.println(" 0 1 2 3 4 5 6");
        for (int r = 0; r < 6; r++) {
            for (int c = 0; c < 7; c++) {
                System.out.print("|" + board[r][c]);
            }
            System.out.println("|");
        }
        System.out.println("---------------");
    }

    public void play() {
        Scanner input = new Scanner(System.in);
        int moves = 0;
        while (moves < 42) {  // 6 × 7 = 42 max moves
            display();
            System.out.print("Player " + currentPlayer + ", choose column (0-6): ");
            int col = input.nextInt();

            if (dropPiece(col)) {
                moves++;
                if (checkWin()) {
                    display();
                    System.out.println("Player " + currentPlayer + " wins!");
                    return;
                }
                currentPlayer = (currentPlayer == 'X') ? 'O' : 'X';
            }
        }
        System.out.println("It's a tie!");
    }

Extending the Program

  • Add an AI that blocks three-in-a-row threats.
  • Highlight the winning four pieces on display.
  • Add an undo feature (store move history).

How This Connects to Your RPG

The gravity logic (walking from bottom to top) is the same as finding the first empty inventory slot. The multi-direction win check is the same pattern as checking for contiguous dungeon rooms or spell area-of-effect.
