2D Array Basics

A 2D array is an array of arrays — a grid with rows and columns. Think of a spreadsheet, a chess board, a pixel screen, or a dungeon map. Every cell is accessed with two indices: [row][col].

The Grid Metaphor

int[][] grid = new int[3][4];

This creates a grid with 3 rows and 4 columns:

         col 0   col 1   col 2   col 3
row 0:  [0][0]  [0][1]  [0][2]  [0][3]
row 1:  [1][0]  [1][1]  [1][2]  [1][3]
row 2:  [2][0]  [2][1]  [2][2]  [2][3]

Rows go top-to-bottom (0 = top). Columns go left-to-right (0 = left). The first index is ALWAYS the row. The second index is ALWAYS the column.

Mental model: grid is an array where each element is itself an array. grid[0] is the entire first row (a 1D array). grid[0][2] is the cell in row 0, column 2.

Declaration and Allocation

// Declare and allocate in one step (most common):
int[][] grid = new int[3][4];      // 3 rows, 4 columns

// Step by step:
double[][] matrix;                  // declares the variable
matrix = new double[5][5];         // allocates a 5x5 grid

// Initialize with values inline:
int[][] scores = {
    {95, 88, 92},
    {76, 84, 90},
    {88, 91, 86}
};

// Different types:
char[][] map = new char[10][10];           // dungeon map
String[][] roster = new String[4][3];      // seating chart
boolean[][] visited = new boolean[8][8];   // chess-board tracking

The Two Lengths

A 2D array has TWO length values. This is the most important concept in the unit: 

int[][] grid = new int[4][6];

grid.length          // → 4  (number of ROWS)
grid[0].length       // → 6  (number of COLUMNS)
grid[3].length       // → 6  (any row gives the same column count)

grid.length tells you how many rows. Each row is a 1D array — and grid[0].length (or grid[anyRow].length) tells you how many elements are in that row, which is the number of columns.

Trap: Thinking grid.length gives columns. It doesn't. grid.length = rows. grid[0].length = columns.

Accessing and Modifying Elements

int[][] grid = new int[3][4];

grid[0][0] = 7;         // row 0, col 0 = 7
grid[1][2] = 42;        // row 1, col 2 = 42
grid[2][3] = grid[0][0] + grid[1][2];  // read two cells, assign to another

int value = grid[1][2];  // value = 42

Default Values

When you allocate a 2D array with new, every cell gets a default value:

TypeDefault
int[][]0
double[][]0.0
boolean[][]false
char[][]'\0' (null character)
String[][] / object arraysnull
Char trap: char[][] map = new char[5][5]; fills every cell with '\0' (the null character), NOT a space ' '. If you check map[0][0] == ' ' on a fresh grid, it returns false.

Row-Major vs Column-Major (Preview)

A 2D array can be traversed in two orders. The outer loop decides the order: 

// Row-major: process row 0 completely, then row 1, then row 2...
for (int row = 0; row < grid.length; row++) {
    for (int col = 0; col < grid[row].length; col++) {
        // process grid[row][col]
    }
}

// Column-major: process column 0 completely, then column 1...
for (int col = 0; col < grid[0].length; col++) {
    for (int row = 0; row < grid.length; row++) {
        // process grid[row][col]
    }
}

Full details on the Traversal page.

Using Enhanced For Loops

int[][] grid = {{1, 2}, {3, 4}, {5, 6}};

// Outer enhanced for: each element is a 1D array (a row)
for (int[] row : grid) {
    // Inner enhanced for: each element is a cell value
    for (int value : row) {
        System.out.print(value + " ");
    }
    System.out.println();
}
// Output:
// 1 2
// 3 4
// 5 6
Enhanced for = read-only (mostly). You get the VALUE, not the position. You can't check neighbors, and you can't easily modify cells unless the value is an object. Use standard for loops for anything involving indices.

Ragged Arrays

A 2D array where rows have different lengths is called a ragged array. The AP exam assumes rectangular grids, but it's worth knowing they exist: 

int[][] ragged = new int[3][];  // allocate rows, not columns yet
ragged[0] = new int[4];        // row 0 has 4 columns
ragged[1] = new int[2];        // row 1 has 2 columns
ragged[2] = new int[5];        // row 2 has 5 columns

// Or inline:
int[][] triangle = {
    {1},
    {2, 3},
    {4, 5, 6}
};

Socratic Checkpoint

1. The Dimension Breakdown

double[][] battlefield = new double[5][8];

What is the exact value returned by battlefield.length? What is the exact value returned by battlefield[0].length? Which one represents rows, and which represents columns?

2. The Unseen Defaults

If you create a new char[][] dungeonMap = new char[5][5], what character is stored in dungeonMap[0][0]? If you print it, what do you see? (Hint: It is NOT a space ' '!)

Bug Hunt: The Inverted Coordinates

A designer creates a small 2D game board with 3 rows and 5 columns, but when they try to place a treasure chest in the bottom-right corner, the game crashes immediately. Find the bug.

int[][] board = new int[3][5];
// Bottom-right corner is at row index 2, column index 4
board[4][2] = 99; // Game crashes here with ArrayIndexOutOfBoundsException!

Question: Why did board[4][2] crash the program? What index order must always be used when accessing cells in a 2D array?
