Student RPG Guide: Grid-Based Dungeon Map

Your dungeon becomes a real 2D grid. No more linear room sequences. The player moves with WASD across a char[][] grid of walls, floors, treasure, and monsters. Two files: DungeonMap.java and a runner. You design the map layout. You handle collisions. You wire it to Player and Monster from past units.

References

  • Tic-Tac-Toe — 2D array board, display with nested loops, row/col validation, win-check patterns
  • Connect Four — 2D array traversal, boundary checks before accessing cells, gravity logic (bottom-up search)
  • 2D Array Basics — declare, initialize, access grid[r][c], .length vs [r].length
  • 2D Array Traversal — nested for loops, row-major traversal, boundary guards
  • Player.java (Unit 5) — your player object, moves on the map
  • Monster.java (Unit 5) — enemies on the map, trigger combat

Tile Legend

Every cell in the grid is one of these char values:

CharMeaningWalkable?Effect on entry
'#'WallNoBlocked — can't enter
'.'FloorYesNothing
'$'TreasureYesCollect gold. Cell becomes '.'
'M'MonsterYesTrigger combat. Cell becomes '.' on kill
'X'ExitYesWin condition (optional — you decide)

Player position is tracked separately with playerRow and playerCol — not stored in the grid. When displaying, overlay 'P' at (playerRow, playerCol).

FILE 1: DungeonMap.java

Instance Variables

VariableTypePurpose
mapchar[][]The dungeon grid
playerRowintPlayer's current row
playerColintPlayer's current column
playerPlayerThe Player object from Unit 5
goldCollectedintTotal gold collected this dungeon run

Constructor

  • Parameters: char[][] map, Player player
  • Store the map reference and player object
  • Find 'P' on the map: Loop through every cell. When you find 'P', set playerRow and playerCol. Replace 'P' with '.' — that tile is now just floor.
  • Initialize goldCollected = 0

display()

  • Print header: "=== DUNGEON MAP ==="
  • Print status line: "HP: X/Y | Gold: Z"
  • Nested for-loop over every row and column
  • If (r,c) == player position → print 'P'. Otherwise print map[r][c]
  • No spaces between characters on the grid line
  • Newline after each row

move(char direction)

Returns String — a description of what happened: "moved", "blocked", "treasure found", "monster defeated", "player died", "exit found".

Direction mapping (WASD):

KeyRow changeCol change
W-10
A0-1
S+10
D0+1

Validation (check in this order):

  1. Bounds check: newRow in [0, map.length-1]? newCol in [0, map[0].length-1]? If not → return "blocked"
  2. Wall check: map[newRow][newCol] == '#' → return "blocked"

If valid, update position: playerRow = newRow; playerCol = newCol;

Check what's on the new tile:

  • '$' (Treasure): Roll gold: (int)(Math.random() * 50) + 10. Add to goldCollected. Set cell to '.'. Return "treasure found".
  • 'M' (Monster): Create a Monster. Run combat (use your existing combat code from earlier units). If monster dies: set cell to '.', increment defeatedCount. If player dies: return "player died". Return "monster defeated" or "fled".
  • 'X' (Exit): Return "exit found" — the runner handles win condition.
  • '.' (Floor): Return "moved".

Getters / Status

  • isPlayerAlive() → delegates to player.isAlive()
  • getGoldCollected()
  • getPlayerHP() / getPlayerMaxHP()

FILE 2: DungeonMapRunner.java

Your main() ties everything together:

  1. Create a Player
  2. Define a char[][] dungeon layout (use the test map below)
  3. Create DungeonMap
  4. Game loop: while player is alive:
    • Display map
    • Read input (W/A/S/D or Q to quit)
    • Call dungeon.move(input)
    • Check result: "player died" → print, break. "exit found" → win, break.
  5. Print final summary: gold collected, monsters killed, victory/defeat

Test Map

Use this 8×8 layout. 'P' marks the player start.

char[][] dungeonLayout = {
    {'#', '#', '#', '#', '#', '#', '#', '#'},
    {'#', 'P', '.', '.', '$', '.', 'M', '#'},
    {'#', '.', '#', '#', '.', '#', '.', '#'},
    {'#', '.', '.', 'M', '.', '#', '$', '#'},
    {'#', '#', '.', '#', '#', '#', '.', '#'},
    {'#', '.', '.', '.', '.', '.', '.', '#'},
    {'#', '.', '#', 'M', '#', '#', 'X', '#'},
    {'#', '#', '#', '#', '#', '#', '#', '#'}
};

Test It

// === Test 1: Constructor finds P ===
Player hero = new Player("Hero", 25, 14, 12, 16, 10);
DungeonMap dungeon = new DungeonMap(dungeonLayout, hero);
System.out.println(dungeon.getPlayerRow());  // 1
System.out.println(dungeon.getPlayerCol());  // 1

// === Test 2: Walking onto floor ===
System.out.println(dungeon.move('D'));  // "moved"
System.out.println(dungeon.getPlayerCol());  // 2

// === Test 3: Walking into wall ===
dungeon.move('W');  // row 0 is all walls
System.out.println(dungeon.getPlayerRow());  // still 1 — didn't move

// === Test 4: Walking onto treasure ===
dungeon.move('D');  // move to '.'
dungeon.move('D');  // move to '$'
// Should print treasure message and gold
System.out.println(dungeon.getGoldCollected());  // > 0

// === Test 5: Walking onto monster ===
dungeon.move('S');  // move down
// Should trigger combat. Player may win or die.
System.out.println(dungeon.isPlayerAlive());

// === Test 6: Bounds check ===
for (int i = 0; i < 10; i++) dungeon.move('A');  // smash into left wall
System.out.println(dungeon.getPlayerCol());  // 0 or 1 — clamped by wall
