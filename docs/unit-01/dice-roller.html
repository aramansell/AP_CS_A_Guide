Student RPG Guide: Dice Roller

This is YOUR guide. Every dice roll, damage calculation, and loot drop in your RPG depends on this class. Get it right once, use it all year. Instead of giving you the code, this guide provides the Acceptance Criteria. It is your job to write the code that satisfies the criteria.

The Setup

Create two files in your project:

  1. Dice.java — This will contain all your dice logic.
  2. DiceTester.java — This will contain a main method to run your tests.

In Dice.java, start with a basic class outline. You will add methods to this class.

public class Dice {
    // Your methods will go here
}

Part 1: roll() — Roll a Single Die

Requirements:

  • Method signature: public static int roll(int sides)
  • Behavior: Generates a random integer between 1 and the number of sides (inclusive).

Socratic Checkpoint:

Look back at your Math.random() lab notebook. How do you construct the formula for a specific range? What is the "range" based on the sides parameter? What is the "min"?

Acceptance Test:

Add this to DiceTester.java's main method:

System.out.println("Rolling d20: " + Dice.roll(20));
System.out.println("Rolling d6: " + Dice.roll(6));

Run it multiple times. Do you ever see a 0? Do you ever see a number higher than the sides?

Part 2: rollDice() — Roll Multiple Dice and Sum

Requirements:

  • Method signature: public static int rollDice(int count, int sides)
  • Behavior: Rolls the specified count of dice, each with sides, and returns the total sum.

Tinker Challenge:

For now, since we haven't learned loops, you can hardcode this method to just roll 3 dice. Call your own roll(sides) method three times and add the results together. How do you store a "running total" across multiple lines of code?

Acceptance Test:

System.out.println("Rolling 3d6: " + Dice.rollDice(3, 6));

Run it. Can you roll below a 3? Can you roll above an 18?

Part 3: rollWithAdvantage() — Roll Twice, Take Higher

Requirements:

  • Method signature: public static int rollWithAdvantage(int sides)
  • Behavior: Rolls two dice of the specified sides. Prints both rolls so the player can see them. Returns the higher of the two rolls.

Socratic Checkpoint:

If you have two integers, roll1 and roll2, how can you ask Java which one is larger? Refer to the documentation on if statements.

Acceptance Test:

System.out.println("Rolling d20 with advantage. Final result: " + Dice.rollWithAdvantage(20));

Run it. Ensure the printed output shows both rolls, and the final returned result matches the higher of the two.

Part 4: isCritical() — Detect a Maximum Roll

Requirements:

  • Method signature: public static boolean isCritical(int roll, int sides)
  • Behavior: Returns true if the roll is equal to the sides (the maximum possible value). Returns false otherwise.

Tinker Challenge:

Can you write this method in a single line of code inside the method body? Hint: Relational operators (like ==) evaluate to a boolean value on their own.

Acceptance Test:

System.out.println("Is 20 on a d20 a crit? " + Dice.isCritical(20, 20)); // Should be true
System.out.println("Is 19 on a d20 a crit? " + Dice.isCritical(19, 20)); // Should be false

Part 5: displayResult() — Formatted Output

Requirements:

  • Method signature: public static void displayResult(String label, int sides, int result)
  • Behavior: Prints a formatted string in the pattern: [label] (d[sides]): [result]. If the result is a critical hit, append -- CRITICAL! to the string.
  • Does not return anything.

Acceptance Test:

Dice.displayResult("Attack roll", 20, 15);
Dice.displayResult("Sneak attack", 20, 20);

Expected Output:

Attack roll (d20): 15
Sneak attack (d20): 20 -- CRITICAL!
