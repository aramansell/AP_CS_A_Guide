Arithmetic & Operators

Java can do math. The operators look exactly like what you'd expect — but there is one critical trap that surprises nearly every new programmer. Let's find it.

The Rule

When both operands are int, Java does integer division. It chops off everything after the decimal point. No rounding — just gone. If ANY operand is a double, the whole expression uses floating-point math.

Where This Bites You

Bug Hunt: The Broken Average
A student wrote this code to calculate their test average. It compiles and runs, but the answer is wrong. 
int test1 = 92;
int test2 = 87;
int test3 = 95;
double average = (test1 + test2 + test3) / 3;
System.out.println(average);
The output is 91.0 instead of 91.33333333333333. Why? How would you fix it? Hint: Look at the data types in the division operation!

Compound Assignment Operators

Shorthand for "do math on a variable and store it back."

ShorthandEquivalent
x += 5;x = x + 5;
x -= 3;x = x - 3;
x *= 2;x = x * 2;
x /= 2;x = x / 2;
x %= 3;x = x % 3;
int hp = 100;
hp -= 15;    // took 15 damage — now 85
hp += 25;    // drank a potion — now 110

Operator Precedence

Java follows normal math rules: * / % before + -. Use parentheses when in doubt.

System.out.println(5 + 3 * 2);       // 11 (3*2=6, then 5+6)
System.out.println((5 + 3) * 2);     // 16 (5+3=8, then 8*2)
