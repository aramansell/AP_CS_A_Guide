Variables & Types

A variable is a named box that holds a value. You create one by saying what type of value it holds, what name you'll use to refer to it, and (optionally) what initial value to put in it.

Declaration Syntax

type name = value;
int playerHealth = 100;
double luck = 0.75;
boolean isAlive = true;

The Three Primitive Types You Need Right Now

TypeStoresExample valuesRPG uses
int Whole numbers (no decimal) 42, -5, 0, 9999 HP, score, level, dice values, gold coins
double Numbers with decimals 3.14, -0.5, 100.0, 0.75 Percentages, luck stats, precise damage multipliers
boolean True or false only true, false isAlive, hasKey, wearingArmor, questComplete

Declaration vs. Initialization

You can separate the two steps:

int score;           // declaration — box created, but empty (Java sets to 0)
score = 100;         // initialization — put a value in the box

Or combine them — this is more common:

int score = 100;     // declare AND initialize in one line

Assignment (=)

In Java, = does not mean "equals." It means "put this value into this box." You read it right-to-left: take whatever is on the right and store it in the variable on the left. 

int hp = 100;        // put 100 into hp
hp = 75;             // replace what's in hp with 75 — the old 100 is gone
hp = hp - 10;        // take current hp (75), subtract 10, store result (65) back in hp

Naming Rules

  • Use camelCase: start lowercase, capitalize each new word (playerHealth, goldAmount)
  • No spaces. No special characters except _ and $ (avoid both).
  • Can't start with a number.
  • Can't use Java keywords (int, class, public, etc.).
  • Make names descriptive. hp is fine. x is not.

Common Traps

Bug Hunt: The Strict Compiler
Type the following lines into your IDE. They will cause compiler errors (red underlines). Hover over the red underlines to read the error messages. 
int x = 3.5;
int y = "hello";
Based on the error messages, what rule is Java enforcing? How would you fix these lines so they compile?
Socratic Checkpoint
Try this line in your IDE: 
double z = 5;
Does this cause an error? Why does Java let you put an int into a double box, but not a double into an int box?
