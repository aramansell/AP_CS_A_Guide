The Three-Step Pattern

Nearly every program you write this year will follow the same structure. Learning to see and use this pattern is the single most important skill from Unit 1.

The Pattern

1 Declare — create variables to hold your data
2 Calculate — do the math, generate values, process the data
3 Display — show the results to the user

Why Three Steps?

You COULD squash everything into one line. But separating the steps gives you three huge advantages:

  1. Test each step independently. After Step 1, you can print your variables to make sure they're correct. After Step 2, you can print intermediate results. You find bugs early, not after writing 30 lines.
  2. Reuse the pattern across projects. Tip Calculator, Dice Roller, Character Creator, Combat Engine — same skeleton, different meat. Once you know the pattern, you can start ANY program by writing three comment blocks.
  3. Split work with teammates. One person handles calculations, another handles display. The steps are independent.

Applied: Tip Calculator

public class TipCalculator {
    public static void main(String[] args) {
        // ===== STEP 1: DECLARE =====
        double billAmount = 45.50;
        double tipPercent = 20;
        double tip;
        double total;
        
        // ===== STEP 2: CALCULATE =====
        tip = billAmount * (tipPercent / 100.0);
        total = billAmount + tip;
        
        // ===== STEP 3: DISPLAY =====
        System.out.println("===== TIP CALCULATOR =====");
        System.out.println("Bill: $" + billAmount);
        System.out.println("Tip:  $" + tip);
        System.out.println("Total: $" + total);
        System.out.println("==========================");
    }
}

Applied: Dice Roller

public class DiceRoller {
    public static void main(String[] args) {
        // ===== STEP 1: DECLARE =====
        int roll1;
        int roll2;
        int roll3;
        int total;
        
        // ===== STEP 2: CALCULATE =====
        roll1 = (int) (Math.random() * 6) + 1;
        roll2 = (int) (Math.random() * 6) + 1;
        roll3 = (int) (Math.random() * 6) + 1;
        total = roll1 + roll2 + roll3;
        
        // ===== STEP 3: DISPLAY =====
        System.out.println("===== DICE ROLLER =====");
        System.out.println("Die 1: " + roll1);
        System.out.println("Die 2: " + roll2);
        System.out.println("Die 3: " + roll3);
        System.out.println("Total: " + total);
        System.out.println("=======================");
    }
}

Can I Change the Order?

Tinker Challenge: Mixing the Steps
Try modifying your Tip Calculator program to display the total before the calculation step: 
System.out.println("Total: $" + total);
total = billAmount + tip;
What happens when you compile? Can you explain why Java enforces this order?

The order matters. You can't show a result you haven't computed. Declare → Calculate → Display.

From Pattern to Template

When starting a new project, many programmers write this skeleton first: 

public class ProjectName {
    public static void main(String[] args) {
        // STEP 1: Declare variables
        
        
        // STEP 2: Calculate
        
        
        // STEP 3: Display
        
        
    }
}

Then fill in each section. A blank screen is intimidating. Three labeled sections are just three small problems.
