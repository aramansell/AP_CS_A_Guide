Tip Calculator — Full Walkthrough

This project will be built together as a class. Your teacher will guide you through the process, but you will write the code and figure out the bugs. This introduces the "Three-Step Pattern" (Declare, Calculate, Display) that you will use in almost every program this year.

The Goal

Given a bill amount and a tip percentage, calculate and display the tip and total.

===== TIP CALCULATOR =====
Bill:  $45.5
Tip:   $9.1
Total: $54.6
==========================

Iteration 1: Declare the Data

Before we can do math, we need to store our data.

Task 1: Variables
Create a new class called TipCalculator with a main method.

Declare four variables to hold:
  1. The starting bill amount (set it to 45.50)
  2. The tip percentage (set it to 20)
  3. The calculated tip amount (don't set a value yet)
  4. The final total amount (don't set a value yet)
Socratic Checkpoint
What data type should you use for these variables? Why is int a bad choice for money?

Iteration 2: The Math Trap

Now we need to calculate the tip. 20 percent means 20 divided by 100.

Bug Hunt: The Zero Dollar Tip
Add this calculation to your code: 
tip = billAmount * (tipPercent / 100);
System.out.println("Debug Tip: " + tip);
Run the code. The output is Debug Tip: 0.0.

Why is the tip 0? Use what you discovered in the Arithmetic lab to fix this line of code so that it prints 9.1.

Iteration 3: Display the Receipt

Once the math is fixed, calculate the total (bill + tip). Then, replace your debug print statement with a nicely formatted receipt.

Task 2: Formatting Output
Use System.out.println and String concatenation (the + operator) to print the exact receipt shown in "The Goal" section above.

Extending the Program

Once your basic calculator works, try these upgrades:

  • Split the Check: Add a variable for the number of people and calculate how much each person owes.
  • Sales Tax: Add a tax rate variable (like 8.25%) and apply tax before calculating the tip.

How This Connects to Your RPG

The Tip Calculator is not your RPG project. Your RPG project is the Dice Roller. But they share the exact same structure: declare variables, calculate, display. When you start a new project, remember this pattern!
