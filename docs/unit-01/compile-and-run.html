Compile & Run

Java is a compiled language. You write human-readable .java files, then a program called javac translates them into .class files that the computer can execute. The computer runs the .class file — not the .java file.

The Golden Rule

Every time you change your .java file, you MUST recompile before running.
If you change the code but skip javac, you're running old code. This is the #1 mistake new Java programmers make.

The Two Commands

CommandWhat it doesWhen to use it
javac MyProgram.java Compiles your source code into MyProgram.class After every edit
java MyProgram Runs the compiled .class file After a successful compile

Note: javac uses the full filename with .java. java uses just the class name — no extension.

Example: Hello World

Create a file called Hello.java:

public class Hello {
    public static void main(String[] args) {
        System.out.println("Hello, adventurer!");
    }
}

In the terminal, in the same directory as your file:

$ javac Hello.java      (no output = success)
$ ls                     Hello.class  Hello.java
$ java Hello             Hello, adventurer!

Common Errors

Bug Hunt: The Compiler's Complaints
The compiler is your friend, but its messages can be cryptic. Try intentionally making these mistakes in a working Hello.java file and try to compile and run it:
  1. Rename the class inside the file to hello (lowercase) instead of Hello. Try to compile.
  2. Fix the class name. Now compile it successfully. Then run java Hello.java instead of java Hello.
  3. Delete the semicolon at the end of the println statement. Try to compile.
How does the compiler tell you what went wrong in each case? Discuss with your partner what these error messages mean so you can recognize them later.

The Boilerplate

Every Java program needs this structure. You'll type it hundreds of times this year: 

public class ClassName {
    public static void main(String[] args) {
        // Your code goes here
    }
}
  • public class — defines a class. The name must match the filename.
  • public static void main(String[] args) — the entry point. Java starts here.
  • Curly braces { } — group statements into blocks.
  • Semicolons ; — end each statement.

You don't need to fully understand the boilerplate yet. You will. For now, type it correctly and know that your code goes between the inner curly braces.
