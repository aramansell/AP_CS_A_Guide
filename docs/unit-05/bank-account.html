Writing a BankAccount Class — Full Walkthrough

This is the program your teacher builds live. Unlike the immutability of Card, BankAccount has state that changes: deposits, withdrawals, interest. This is where mutators (setters) shine. Use this as a reference when building your own classes.

The Goal

Create a BankAccount class with balance tracking, deposits, withdrawals with a minimum-balance guard, and interest calculation.

Created account #1001 for Alice with $500.00
Deposited $200.00 — new balance: $700.00
Withdrew $150.00 — new balance: $550.00
Withdrew $600.00? Insufficient funds!
After 5% interest: $577.50

Step 1: Instance Variables + Constructor

public class BankAccount {
    private int accountNumber;
    private String ownerName;
    private double balance;
    private static int nextAccountNumber = 1001;

    public BankAccount(String ownerName, double initialDeposit) {
        this.ownerName = ownerName;
        this.balance = initialDeposit;
        this.accountNumber = nextAccountNumber;
        nextAccountNumber++;  // auto-generate unique account numbers
    }

Static counter pattern. nextAccountNumber belongs to the class, not individual objects. Every new account gets an incremented number. Same pattern as Monster's defeatedCount.

Step 2: Getters + Display

    public int getAccountNumber() { return accountNumber; }
    public String getOwnerName()  { return ownerName; }
    public double getBalance()    { return balance; }

    public String toString() {
        return "Account #" + accountNumber + " (" + ownerName
               + "): $" + String.format("%.2f", balance);
    }

String.format("%.2f", balance) rounds to two decimal places. Without it, you might see $577.5000000000001. With it: $577.50. Clean money.

Step 3: Mutators — Deposit and Withdraw

    public void deposit(double amount) {
        if (amount <= 0) {
            System.out.println("Deposit must be positive.");
            return;   // exit early — do nothing
        }
        balance += amount;
        System.out.println("Deposited $" + String.format("%.2f", amount));
    }

    public boolean withdraw(double amount) {
        if (amount <= 0) {
            System.out.println("Withdrawal must be positive.");
            return false;
        }
        if (amount > balance) {
            System.out.println("Insufficient funds!");
            return false;
        }
        balance -= amount;
        System.out.println("Withdrew $" + String.format("%.2f", amount));
        return true;  // success
    }

Guard clauses. Check for bad input FIRST and bail out. This keeps the happy path (the actual withdrawal logic) clean and un-nested. Returning boolean lets the caller check if the operation succeeded.

Step 4: Interest (Another Mutator)

    public void applyInterest(double rate) {
        balance += balance * (rate / 100.0);  // rate is a percentage
    }
}

The Full Class + Test

public class BankAccount {
    private int accountNumber;
    private String ownerName;
    private double balance;
    private static int nextAccountNumber = 1001;

    public BankAccount(String ownerName, double initialDeposit) {
        this.ownerName = ownerName;
        this.balance = initialDeposit;
        this.accountNumber = nextAccountNumber++;
    }

    public int getAccountNumber() { return accountNumber; }
    public String getOwnerName()  { return ownerName; }
    public double getBalance()    { return balance; }

    public String toString() {
        return "Account #" + accountNumber + " (" + ownerName
               + "): $" + String.format("%.2f", balance);
    }

    public void deposit(double amount) {
        if (amount <= 0) {
            System.out.println("Deposit must be positive.");
            return;
        }
        balance += amount;
    }

    public boolean withdraw(double amount) {
        if (amount <= 0 || amount > balance) return false;
        balance -= amount;
        return true;
    }

    public void applyInterest(double rate) {
        balance += balance * (rate / 100.0);
    }
}

// Test:
public class BankTest {
    public static void main(String[] args) {
        BankAccount alice = new BankAccount("Alice", 500.0);
        System.out.println(alice);

        alice.deposit(200);
        alice.withdraw(150);
        alice.applyInterest(5.0);
        System.out.println(alice);
    }
}

Extending the Program

  • Add a transfer(BankAccount destination, double amount) method.
  • Add a transaction history — every deposit/withdrawal recorded.
  • Add minimum-balance fees if balance drops below $100.

How This Connects to Your RPG

BankAccount is the closest match to Player's HP management: setHP, takeDamage, heal. The guard-clause pattern (check for bad input, bail early) is identical. The static counter maps to Monster's defeatedCount.
