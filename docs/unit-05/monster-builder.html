Student RPG Guide: Building Monster.java

After writing Player.java, transfer the same pattern to enemies. Same anatomy — but monsters have features players don't: boss status, damage dice, and a kill counter. One file: Monster.java.

References

  • Card — immutable fields, basic class structure
  • BankAccount — mutable state, guard clauses, static counter
  • Player.java Guide — the class you just built. Monster mirrors most of Player's HP methods.
  • Methods & Static — static variables belong to the class, not individual objects
  • Constructors — overloading, this(...) constructor chaining

Spec

Instance Variables

All private.

VariableTypePurpose
nameStringEnemy name, e.g. "Goblin"
hitPointsintCurrent health
maxHitPointsintMaximum health
armorClassintDefense rating (AC)
damageDieintWhat die to roll for damage (d4=4, d6=6, d8=8, etc.)
isBossbooleanIs this a boss monster?

Static Variable

VariableTypePurpose
defeatedCountintHow many monsters have been killed total. Belongs to the class, not individual monsters. Start at 0.

Constructors (Overloaded — Two Versions)

Constructor 1 — Basic (non-boss):

  • Parameters: String name, int hp, int ac, int damageDie
  • Store all four fields
  • Set isBoss = false
  • Validation:
    • hp must be > 0 — otherwise default to 10
    • ac can't be below 6 — otherwise default to 10
    • damageDie can't be below 2 — otherwise default to 6 (d6)

Constructor 2 — Boss-capable:

  • Parameters: String name, int hp, int ac, int damageDie, boolean isBoss
  • Call Constructor 1 using this(name, hp, ac, damageDie) for shared setup
  • Then set this.isBoss = isBoss
  • Do NOT duplicate the validation logic — let Constructor 1 handle it

Getters

  • Instance getters: one for each of the six instance variables
  • Static getter: getDefeatedCount() — returns defeatedCount

HP Management

Same pattern as Player.java. These should be identical except for print messages using the monster's name.

MethodWhat It Does
setHP(int newHP) Clamp to [0, maxHitPoints].
Critical: if newHP == 0, increment defeatedCount. Only increment ONCE per death — use if (hitPoints > 0 && newHP == 0) to guard.
takeDamage(int amount)Reduce HP by amount via setHP(). Print message.
heal(int amount)Increase HP by amount via setHP(). Print message.
isAlive()Returns hitPoints > 0.

attack()

Returns int — the damage dealt. Requires Dice.java from Unit 1.

  • Normal monster: Roll damageDie once. Return the result.
  • Boss monster: Roll damageDie twice. Return the higher of the two rolls.

toString()

Bosses get a star prefix to stand out. Format:

"☆ <name> — HP:<hp>/<max> AC:<ac> d<die>"  (boss)
"  <name> — HP:<hp>/<max> AC:<ac> d<die>"  (normal)

Test It

// === Test 1: Constructor + toString ===
Monster goblin = new Monster("Goblin", 12, 12, 6);
System.out.println(goblin);
// Expected: "  Goblin — HP:12/12 AC:12 d6"

// === Test 2: Boss constructor chaining ===
Monster dragon = new Monster("Dragon", 50, 18, 12, true);
System.out.println(dragon);
// Expected: "☆ Dragon — HP:50/50 AC:18 d12"
System.out.println(dragon.isBoss());  // true

// === Test 3: Validation — bad inputs ===
Monster broken = new Monster("???", -5, 2, 0);
System.out.println(broken.getHitPoints());     // 10 (clamped)
System.out.println(broken.getArmorClass());     // 10 (clamped)
System.out.println(broken.getDamageDie());      // 6  (clamped)

// === Test 4: takeDamage and defeatedCount ===
System.out.println(Monster.getDefeatedCount()); // 0
goblin.takeDamage(5);
System.out.println(goblin.getHitPoints());      // 7
goblin.takeDamage(100);
System.out.println(goblin.getHitPoints());      // 0
System.out.println(goblin.isAlive());           // false
System.out.println(Monster.getDefeatedCount()); // 1 — incremented!

// === Test 5: defeatedCount only increments ONCE ===
goblin.takeDamage(10);  // already dead
System.out.println(Monster.getDefeatedCount()); // still 1 — not double-counted

// === Test 6: attack() — normal vs boss ===
// (Requires Dice.java. Roll enough times to see both behaviors.)
Monster rat = new Monster("Rat", 5, 8, 4);   // d4, non-boss
Monster boss = new Monster("Orc Warlord", 30, 15, 8, true); // d8, boss
System.out.println("Rat attack: " + rat.attack());    // 1-4
System.out.println("Boss attack: " + boss.attack());  // 1-8, advantage (roll 2 take higher)

// === Test 7: Boss advantage over many rolls ===
// Boss should consistently roll higher on average than a normal d8
int bossSum = 0;
for (int i = 0; i < 1000; i++) bossSum += boss.attack();
System.out.println("Boss avg (should be ~6 for d8 advantage): " + (bossSum / 1000.0));
