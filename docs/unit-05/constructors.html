Constructors

A constructor is a special method that runs when you create an object. It sets the object's initial state. Without a constructor, your object is a blank shell — all zeros, nulls, and false.

Syntax

public ClassName(parameters) {
    // initialize instance variables
}

Three rules:

  1. The name is exactly the class name.
  2. No return type. Not even void.
  3. Java calls it automatically when you write new.

Default vs Parameterized

public class Weapon {
    private String name;
    private int damage;

    // DEFAULT (no-arg) constructor
    public Weapon() {
        name = "Fists";
        damage = 1;
    }

    // PARAMETERIZED constructor
    public Weapon(String n, int dmg) {
        name = n;
        damage = dmg;
    }
}

// Usage:
Weapon w1 = new Weapon();              // uses default
Weapon w2 = new Weapon("Sword", 12);   // uses parameterized
The free constructor trap. If you write zero constructors, Java gives you a free no-arg one. If you write any constructor, Java takes the free one away. If you need a no-arg constructor, you have to write it yourself.

Constructor Overloading

You can have multiple constructors with different parameter lists. Java picks the one whose types match your new call. 

public class Player {
    private String name;
    private int hp;
    private int ac;

    // No-arg: default hero
    public Player() {
        name = "Hero";
        hp = 10;
        ac = 10;
    }

    // Name only: default hp and ac
    public Player(String n) {
        name = n;
        hp = 10;
        ac = 10;
    }

    // Full specification
    public Player(String n, int h, int a) {
        name = n;
        hp = h;
        ac = a;
    }
}

Player p1 = new Player();                 // → no-arg
Player p2 = new Player("Rogue");          // → name-only
Player p3 = new Player("Mage", 8, 14);    // → full

Calling Another Constructor: this(...)

Instead of copying constructor code, call another constructor from within one. Must be the first line. 

public Player() {
    this("Hero", 10, 10);  // calls the 3-param constructor
}

public Player(String n) {
    this(n, 10, 10);       // calls the 3-param constructor
}

public Player(String n, int h, int a) {
    name = n;
    hp = h;
    ac = a;
}

Validation in Constructors

Your constructor should reject bad data. Don't let someone create a Player with -50 HP. Use the same validation you'd put in setters. 

public Player(String n, int h, int a) {
    name = n;

    if (h <= 0) {
        hp = 10;     // minimum HP
    } else {
        hp = h;
    }

    if (a < 8) {
        ac = 8;      // minimum AC
    } else {
        ac = a;
    }
}
Shadowing alert. If your parameter has the same name as your instance variable, name = name; assigns the parameter to ITSELF. The instance variable stays unchanged. Fix: this.name = name;.

Socratic Checkpoint: The Missing Constructor

A student writes a Weapon class with one constructor: public Weapon(String n, int dmg). Later, they try to write Weapon fists = new Weapon(); and their code won't compile. Why does Java throw an error here, and what is the rule about "free" constructors?

Bug Hunt: The Null Goblin

A student wrote this constructor for a new monster type, but when they run their game, the goblin always has 0 HP and its name is null. Find the two major constructor mistakes in this code:

public class Enemy {
    private String name;
    private int hp;

    public void Enemy(String name, int hp) {
        name = name;
        hp = hp;
    }
}

Hint: Pay close attention to the method signature and what exactly is being assigned inside the method body.

Tinker Challenge: The Constructor Chain

Write a Shield class that demonstrates constructor chaining and robust data validation.

  1. Variables: name (String), defense (int), weight (double). Make sure they are correctly encapsulated.
  2. Full Constructor: Takes all three parameters. Validation rules: defense cannot be less than 1 (set to 1 if it is). weight cannot be negative (set to 0.0 if it is).
  3. Partial Constructor: Takes only name and defense. Default weight is 5.0. Do not copy your validation logic! Call the full constructor using this(...).
  4. Default Constructor: Takes no arguments. Default name is "Wooden Shield", defense is 2, weight is 3.5. Chain it to the full constructor!
