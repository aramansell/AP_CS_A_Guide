Unit 5: Writing Classes

Since Unit 2, you've been using classes. new Scanner(System.in). new Random(). new Player("Hero", 14, 12). You've called methods on objects. You've trusted that getHP() returns the right number.

This unit, you write the classes. You decide what data belongs inside. You write the constructors that build objects. You write the methods that other programmers will call. You control what's public and what stays private.

By the end, you'll write Player.java from scratch — the same class you've been using for months. Then you'll write Monster.java. Your dungeon crawler stops being a pile of int variables and becomes a system of objects.

Concepts

TopicWhat you'll learn
Class Anatomy The four sections of every class. Instance variables, constructors, and methods.
Constructors How objects are born. Parameters, defaults, validation, and constructor overloading.
Getters & Setters Controlling access to data. Encapsulation. Why private matters.
Methods & Static Writing methods that use object state. toString(). Static vs instance.
Scope & Access Where variables live. Local vs parameter vs instance. The this keyword.
Designing Classes Planning a class from a spec. What variables? What methods? What's public?

RPG Thread

This is the rewrite unit. Your Player.java goes from a static utility class (from Units 1-2) to a real object with instance variables and methods. Monster.java joins it. From now on, you instantiate: Player hero = new Player(...). In Unit 9, both become subclasses of GameCharacter. The constructor patterns you learn here become the super() calls in inheritance.
