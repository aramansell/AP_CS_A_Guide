Writing Methods & The Static Keyword

Methods are what your objects can do. A Sword can attack. A Player can level up. A Monster can roll damage. Methods use instance variables to perform actions and return results.

Method Syntax

public returnType methodName(parameters) {
    // body — uses instance variables, parameters, local variables
    return value;  // if returnType is not void
}

Methods That Use Instance State

public class Sword {
    private String name;
    private int damage;
    private int durability;

    public int attack() {
        if (durability <= 0) {
            System.out.println(name + " is broken!");
            return 0;
        }
        durability--;
        return damage;
    }

    public boolean isBroken() {
        return durability <= 0;
    }
}

Notice: attack() uses THREE instance variables (name, damage, durability). No parameters needed — the data lives in the object.

toString() — The Universal Method

Every class in Java inherits toString() from Object. The default version prints garbage: Sword@15db9742. You override it to print something useful. 

public String toString() {
    return name + " [" + damage + " dmg, " + durability + " dur]";
}

// Java calls toString() automatically:
Sword s = new Sword("Iron Sword", 12, 80);
System.out.println(s);     // Iron Sword [12 dmg, 80 dur]
String info = "Equipped: " + s;  // toString runs here too!
toString is for programmers. It's for debugging, logging, and quick inspection. If the player needs pretty output, write a separate display() method.

Static Variables

A static variable belongs to the class itself, not any one object. One copy, shared by every instance. 

public class Player {
    private String name;
    private static int playerCount = 0;  // one counter for ALL players

    public Player(String name) {
        this.name = name;
        playerCount++;   // increment every time a Player is created
    }

    public static int getPlayerCount() {
        return playerCount;
    }
}

// Usage — no object needed:
System.out.println(Player.getPlayerCount());  // 0

Player p1 = new Player("Hero");
Player p2 = new Player("Mage");
System.out.println(Player.getPlayerCount());  // 2

Static Methods

A static method belongs to the class. Call it with ClassName.method(). You don't need an object. 

public static int getPlayerCount() {
    return playerCount;
}
THE TRAP: Static methods cannot access instance variables. getPlayerCount() can read playerCount (static), but it CANNOT read name (instance). Which Player's name would it be? There's no this in a static context.

Method Overloading

Same method name, different parameter lists. Java picks the right one based on the arguments you pass: 

public int attack() {           // no parameters — uses equipped weapon
    return weapon.getDamage();
}

public int attack(int bonus) {  // one parameter — extra damage
    return weapon.getDamage() + bonus;
}

Socratic Checkpoint: The Shared Counter

Look at the playerCount example above. If a student forgot the static keyword and just wrote private int playerCount = 0;, what would happen to the logic? If they created three Player objects, what would the "count" be inside each of them? Why?

Bug Hunt: The Static Trap

A student wants to be able to quickly check their player's current weapon without creating a Player object first. They wrote this code, but the compiler is furious. Find the error and explain why it violates the laws of physics in Java:

public class Player {
    private String equippedWeapon = "Shortsword";

    public static void printWeapon() {
        System.out.println("Currently holding: " + equippedWeapon);
    }
}

Tinker Challenge: The Level Up System

Add an interactive leveling system to your Player class using instance methods.

  1. Variables: You should have hp, maxHP, and level. Add an overriding toString() method that prints them nicely.
  2. Static Tracking: Add a static variable totalLevelsGained that starts at 0. Write a static getter for it.
  3. The Method: Write a public void levelUp() method.
  4. The Logic: Inside levelUp(), increase level by 1, increase maxHP by 5, fully heal the player (hp = maxHP), and increase the static totalLevelsGained by 1.
  5. Overload It: Write a second method public void levelUp(int levels) that allows a player to level up multiple times at once. Hint: Use a loop that calls your other levelUp() method!
