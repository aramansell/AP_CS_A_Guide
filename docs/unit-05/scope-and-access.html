Scope and Access: Where Variables Live

Every variable in Java has a scope — the lines of code where it exists and can be used. Outside its scope: dead. Doesn't exist. Can't be referenced.

Three Kinds of Variables

1. Local Variables

public void doSomething() {
    int x = 5;        // 'x' is local to this method
    System.out.println(x);  // works
}   // ← x DIES here
// x no longer exists

Declared inside a method (or a loop/if block). Lives from the declaration line to the closing }. Only that method can see it.

2. Parameters

public void setDamage(int dmg) {   // 'dmg' is a parameter
    damage = dmg;                  // works inside this method
}                                  // 'dmg' dies here

Declared in the method parentheses. Gets its value from the caller. Scope: the entire method body.

3. Instance Variables

public class Player {
    private String name;      // instance variable
    private int hp;           // instance variable

    public void takeDamage(int dmg) {
        hp -= dmg;            // visible in any method of this class
    }
}

Declared in the class (outside methods). Scope: the entire class. Every method can see them. Live as long as the object lives.

The Block Rule

A variable declared inside { } dies at the matching }:

for (int i = 0; i < 10; i++) {  // 'i' born here
    System.out.println(i);
}                                // 'i' dies here
// System.out.println(i);   // ERROR: i is out of scope

if (hp > 0) {
    String msg = "Alive!";       // 'msg' born here
    System.out.println(msg);
}                                // 'msg' dies here
// System.out.println(msg);  // ERROR!

Fix: Declare Before the Block

String msg = "";           // declared OUTSIDE
if (hp > 0) {
    msg = "Alive!";        // ASSIGN inside
} else {
    msg = "Dead...";       // ASSIGN inside
}
System.out.println(msg);   // works — msg still alive

The this Keyword

this means "the current object." It resolves shadowing: 

public class Player {
    private String name;        // instance variable

    public Player(String name) {  // parameter shadows instance var
        this.name = name;    // this.name = instance, name = parameter
    }

    public String getName() {
        return this.name;     // 'this.' optional here (no shadowing)
    }
}

Bug Hunt: Shadow Strike

The developer meant to update the player's gold, but when they pick up a coin, their overall gold never changes! Find the scoping error (shadowing) that is causing the gold to vanish into thin air:

public class Player {
    private int gold = 0;

    public void pickupCoin() {
        int gold = this.gold; // look closely here!
        gold += 10;
        System.out.println("Picked up 10 gold!");
    }
}

Tinker Challenge: Scope Gauntlet

Write a method public String evaluateRoom(int monsterCount, boolean hasTrap) inside a Dungeon class that correctly manages scope.

  1. Rule 1: Declare a single String result; at the very top of the method. Do not declare any other String variables.
  2. Rule 2: Use an if/else if/else block to determine the danger level. If monsterCount > 3 and hasTrap is true, assign "DEADLY" to result. If only one is true, assign "DANGEROUS". Otherwise, assign "SAFE".
  3. Rule 3: At the very end of the method, return result.
  4. The Test: Try declaring String result = "DEADLY"; inside the first if block instead of reusing the outer variable. Watch the compiler complain when you try to return it at the end! Understand why it fails.
