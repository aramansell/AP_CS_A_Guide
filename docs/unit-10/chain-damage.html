Student RPG Guide: Chain Lightning — Recursive Damage

Your first recursive spell. Chain lightning hits one target, then jumps to adjacent enemies that haven't been hit yet. Recursion naturally models this "hit one, then try neighbors" pattern. Two files: ChainSpell.java and ChainSpellRunner.java.

How Chain Lightning Works

Given this battle grid (M = monster, @ = center target):

    .  .  M  .  .
    .  M  @  M  .
    .  .  M  .  .

The spell hits @ (center), then chains outward:

  1. Hit @ (the center — currentChain = 1)
  2. Recurse to @'s 4 neighbors — hit the M above, below, left, right (currentChain = 2)
  3. For each of THOSE, recurse to THEIR neighbors that are alive and not yet hit (currentChain = 3)
  4. Stop when: out of bounds, cell is null/dead, already visited, or maxChains reached

THE BUG: Why We Need Visited Tracking

Dense cluster of enemies:

    M  M  M
    M  @  M
    M  M  M

Without visited tracking, the top-left M gets hit TWICE — once from the top-M recursing left, once from the left-M recursing up. A boolean[][] visited grid fixes this. Mark a cell visited BEFORE dealing damage. Before recursing, check visited. This is the same pattern used in flood fill and backtracking.

Base Cases (Order Matters!)

Check in this exact order. Bounds check MUST come first — otherwise grid[row][col] crashes with ArrayIndexOutOfBoundsException.

        // (1) Out of bounds
        if (row < 0 || row >= grid.length
            || col < 0 || col >= grid[0].length)
            return 0;

        // (2) Empty cell or dead character
        if (grid[row][col] == null
            || !grid[row][col].isAlive())
            return 0;

        // (3) Already visited — prevents double-hits!
        if (visited[row][col])
            return 0;

        // (4) Max chains reached
        if (currentChain > maxChains)
            return 0;

Recursive Step

        // Mark visited FIRST — before dealing damage
        visited[row][col] = true;

        // Deal damage
        GameCharacter target = grid[row][col];
        target.takeDamage(damage);
        System.out.println("Chain " + currentChain + ": "
            + target.getName() + " takes " + damage + " damage!");

        int count = 1;  // we hit one target

        // Recurse to 4 neighbors (damage halves via integer division)
        count += chainDamage(grid, row - 1, col,
                    damage / 2, maxChains, currentChain + 1, visited);
        count += chainDamage(grid, row + 1, col,
                    damage / 2, maxChains, currentChain + 1, visited);
        count += chainDamage(grid, row, col - 1,
                    damage / 2, maxChains, currentChain + 1, visited);
        count += chainDamage(grid, row, col + 1,
                    damage / 2, maxChains, currentChain + 1, visited);

        return count;
    }
}

Key design decisions:

  • Integer division (damage / 2): Each hop does half damage. 8 → 4 → 2 → 1 → 0. The spell naturally weakens as it spreads.
  • Visited is permanent: Unlike backtracking (where you un-visit), chain damage never un-visits. Once hit, a cell stays hit for this spell cast.
  • No early exit: We always recurse to all 4 neighbors before returning. This is BFS-like — process current, then all children.

FILE 2: ChainSpellRunner.java

NEW FILE. Main method to test the chain spell on a battle grid.

public class ChainSpellRunner {
    public static void main(String[] args) {

        // 5x5 grid of GameCharacter
        GameCharacter[][] battleGrid = new GameCharacter[5][5];

        // Place monsters in a dense block (tests visited tracking):
        battleGrid[2][1] = new Monster("Goblin TL", 15, 12, 6);
        battleGrid[2][2] = new Monster("Orc Captain", 25, 14, 8);
        battleGrid[2][3] = new Monster("Goblin TR", 15, 12, 6);
        battleGrid[3][2] = new Monster("Goblin Bot", 15, 12, 6);

        // Display initial state
        System.out.println("=== INITIAL GRID ===");
        for (int r = 0; r < battleGrid.length; r++) {
            for (int c = 0; c < battleGrid[r].length; c++) {
                if (battleGrid[r][c] != null) {
                    System.out.println("(" + r + "," + c + "): "
                        + battleGrid[r][c]);
                }
            }
        }

        // Cast chain lightning at (2,2) — the Orc Captain
        System.out.println("\n=== CASTING CHAIN LIGHTNING ===");
        boolean[][] visited = new boolean[5][5];
        int hit = ChainSpell.chainDamage(
            battleGrid, 2, 2,    // target center
            8,                    // initial damage
            4,                    // max chains
            1,                    // current chain starts at 1
            visited
        );
        System.out.println("Total enemies hit: " + hit);

        // Display final state
        System.out.println("\n=== FINAL GRID ===");
        for (int r = 0; r < battleGrid.length; r++) {
            for (int c = 0; c < battleGrid[r].length; c++) {
                if (battleGrid[r][c] != null) {
                    System.out.println("(" + r + "," + c + "): "
                        + battleGrid[r][c]);
                }
            }
        }
    }
}

Expected output: 4 enemies hit, each exactly once.

Chain 1: Orc Captain takes 8 damage!
Chain 2: Goblin TL takes 4 damage!
Chain 2: Goblin TR takes 4 damage!
Chain 2: Goblin Bot takes 4 damage!
Total enemies hit: 4

Execution Trace (Cross Pattern with Visited)

Start at (2,2): visited[2][2] = true. Orc takes 8 damage.

Recurse UP to (1,2): bounds OK, grid[1][2] == null → return 0
Recurse LEFT to (2,1): bounds OK, alive, NOT visited
    → visited[2][1] = true. Goblin TL takes 4 damage.
    → UP (1,1): null → 0
    → DOWN (3,1): null → 0
    → LEFT (2,0): out → 0
    → RIGHT (2,2): VISITED → return 0  ← PREVENTED DOUBLE-HIT
Recurse RIGHT to (2,3): bounds OK, alive, NOT visited
    → visited[2][3] = true. Goblin TR takes 4 damage.
    → All 4 neighbors: visited/out/null → 0
Recurse DOWN to (3,2): bounds OK, alive, NOT visited
    → visited[3][2] = true. Goblin Bot takes 4 damage.
    → UP (2,2): VISITED → return 0  ← PREVENTED DOUBLE-HIT
    → All others: null or out → 0

Final count: 4. Each enemy hit exactly once.
