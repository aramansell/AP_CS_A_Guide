Student RPG Guide: DynamicInventory.java

Rewrite your inventory using ArrayList. Your FixedInventory from Unit 6 works — but it has a hard capacity limit and you wrote manual shift logic for remove(). ArrayList removes both problems. Same behavior. Less code. One file: DynamicInventory.java.

Spec: DynamicInventory.java

Import: java.util.ArrayList. If you do sorting: java.util.Collections and java.util.Comparator.

Reuse Item.java from Unit 6. No changes needed. Copy it into your Unit 7 folder.

Instance Variable

ONE field. No capacity limit. No itemCount. ArrayList tracks its own size.

VariableType
itemsArrayList<Item>

Constructor

  • No parameters — no capacity needed
  • Initialize: items = new ArrayList<>();

Methods

MethodReturnsWhat It Does
add(Item item) boolean If item is null → return false. Otherwise items.add(item) → return true. No capacity check.
remove(int index) Item Check index in [0, items.size()-1]. Call items.remove(index). ArrayList shifts automatically — no manual loop. Return the removed item (which remove(index) returns).
removeByName(String name) Item Use a regular for loop — you can't remove during enhanced for. Find first match by equalsIgnoreCase. Call items.remove(i). Return the item. Return null if not found.
contains(String name) boolean Enhanced for loop. Compare item.getName().equalsIgnoreCase(name). Return true if found.
getTotalValue() int Sum all gold values. Enhanced for loop.
getTotalWeight() int Sum all weights. Enhanced for loop.
countByRarity(String rarity) int Count items matching rarity. Enhanced for loop.
display() void Regular for loop (need index numbers). Print [i] <item.toString()>. If items.isEmpty(): print "(empty)".
size() int return items.size();
sortByValue() void Sort items from highest gold value to lowest. Use Collections.sort() with a Comparator. See Leaderboard walkthrough for the sorted comparison pattern — you want descending order.

Test It

// === Test 1: Creation + add ===
DynamicInventory bag = new DynamicInventory();
System.out.println(bag.size());                         // 0
bag.display();                                          // "(empty)"

bag.add(new Item("Steel Sword", 10, 250, "Rare"));
bag.add(new Item("Shield", 15, 100, "Common"));
bag.add(new Item("Ring", 1, 500, "Epic"));
System.out.println(bag.size());                         // 3
bag.display();
// Expected:
// [0] Steel Sword (Rare) - Wt:10 Val:250g
// [1] Shield (Common) - Wt:15 Val:100g
// [2] Ring (Epic) - Wt:1 Val:500g

// === Test 2: add null ===
System.out.println(bag.add(null));                      // false
System.out.println(bag.size());                         // still 3

// === Test 3: contains ===
System.out.println(bag.contains("ring"));               // true (case-insensitive)
System.out.println(bag.contains("Wand"));               // false

// === Test 4: totals + rarity count ===
System.out.println(bag.getTotalValue());                // 850
System.out.println(bag.getTotalWeight());               // 26
System.out.println(bag.countByRarity("Epic"));          // 1
System.out.println(bag.countByRarity("Legendary"));     // 0

// === Test 5: remove by index ===
Item removed = bag.remove(0);
System.out.println(removed.getName());                  // "Steel Sword"
System.out.println(bag.size());                         // 2
bag.display();
// Expected: Shield at [0], Ring at [1] — shifted automatically

// === Test 6: removeByName ===
Item ring = bag.removeByName("Ring");
System.out.println(ring.getGoldValue());                // 500
System.out.println(bag.size());                         // 1
System.out.println(bag.contains("Ring"));               // false

// === Test 7: remove invalid index ===
System.out.println(bag.remove(99));                     // null
System.out.println(bag.remove(-1));                     // null

// === Test 8: sortByValue ===
bag.add(new Item("Potion", 2, 25, "Common"));
bag.add(new Item("Wand", 3, 200, "Rare"));
bag.display();  // mixed order
bag.sortByValue();
bag.display();  // highest value first: Wand(200), Shield(100), Potion(25)

// === Test 9: No capacity limit ===
for (int i = 0; i < 100; i++) {
    bag.add(new Item("Item" + i, 1, 1, "Common"));
}
System.out.println(bag.size());                         // 103
// No "Inventory full!" — ArrayList grows automatically

Integrate Into Your RPG

Replace FixedInventory with DynamicInventory in your dungeon crawler. Same method signatures — just switch the constructor call:

// Before:
FixedInventory inventory = new FixedInventory(20);

// After:
DynamicInventory inventory = new DynamicInventory();

Everything else (.add(), .contains(), .display()) works the same. Your RPG inventory now grows forever.

Socratic Checkpoints

1. Your DynamicInventory uses an ArrayList<Item> internally. What happens if someone passes null to add(null)? Should your code handle this, and if so, how?

2. In Unit 6's FixedInventory, remove-by-name used a for loop to find the index, then a shift loop to close the gap. DynamicInventory just calls items.remove(name). How does ArrayList do both steps in one call? What's the runtime cost compared to your manual version?
