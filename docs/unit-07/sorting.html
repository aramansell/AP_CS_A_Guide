Selection Sort and Insertion Sort

Sorting is rearranging elements into order — smallest to largest, A to Z, weakest to strongest. The AP exam requires you to trace and implement two sorting algorithms on ArrayList: selection sort and insertion sort.

In real code, you'd just call Collections.sort(list). But understanding how sorting works teaches you nested loops, element comparison, and algorithmic thinking — skills that transfer to every programming problem.

Selection Sort

The Idea

Find the smallest. Put it at position 0. Find the next smallest. Put it at position 1. Repeat.

Selection sort divides the list into two regions:

  • Left side (sorted): positions 0 through i-1 — already in order.
  • Right side (unsorted): positions i through the end — still needs work.

Each pass finds the minimum in the unsorted region and swaps it into position i.

Step-by-Step Trace

Starting list: [7, 2, 9, 1, 5]

Pass (i)UnsortedMinimum foundAfter swap
0[7, 2, 9, 1, 5]1 (at index 3)[1, 2, 9, 7, 5]
1[2, 9, 7, 5]2 (at index 1)[1, 2, 9, 7, 5] (no swap needed)
2[9, 7, 5]5 (at index 4)[1, 2, 5, 7, 9]
3[7, 9]7 (at index 3)[1, 2, 5, 7, 9] (no swap needed)

After pass 3, the last element (9) is naturally in place. Done.

The Code

public static void selectionSort(ArrayList<Integer> list) {
    for (int i = 0; i < list.size() - 1; i++) {
        // Find the index of the minimum in the unsorted region
        int minIndex = i;
        for (int j = i + 1; j < list.size(); j++) {
            if (list.get(j) < list.get(minIndex)) {
                minIndex = j;
            }
        }
        // Swap the minimum into position i
        int temp = list.get(i);
        list.set(i, list.get(minIndex));
        list.set(minIndex, temp);
    }
}

Why It Works

  • Outer loop: i < list.size() - 1 — after sorting n-1 elements, the last one is in place.
  • Inner loop: j = i + 1 — everything before i is already sorted; only search the remainder.
  • Always O(n²) comparisons — no best-case shortcut. Even a sorted list gets the full scan.
  • At most n-1 swaps — one swap per pass (or zero if already in position).

Sorting Strings (Using compareTo)

public static void selectionSortStrings(ArrayList<String> list) {
    for (int i = 0; i < list.size() - 1; i++) {
        int minIndex = i;
        for (int j = i + 1; j < list.size(); j++) {
            if (list.get(j).compareTo(list.get(minIndex)) < 0) {
                minIndex = j;
            }
        }
        String temp = list.get(i);
        list.set(i, list.get(minIndex));
        list.set(minIndex, temp);
    }
}

Sorting Objects by a Field

// Sort items by gold value (ascending)
public static void selectionSortByValue(ArrayList<Item> list) {
    for (int i = 0; i < list.size() - 1; i++) {
        int minIndex = i;
        for (int j = i + 1; j < list.size(); j++) {
            if (list.get(j).getGoldValue() < list.get(minIndex).getGoldValue()) {
                minIndex = j;
            }
        }
        Item temp = list.get(i);
        list.set(i, list.get(minIndex));
        list.set(minIndex, temp);
    }
}

Insertion Sort

The Idea

Build the sorted list from left to right. Take each new element and insert it into its correct position among the already-sorted elements.

Like sorting a hand of cards: you pick up each new card and slide it into place.

Step-by-Step Trace

Starting list: [7, 2, 9, 1, 5]

iKeyActionList after
122 < 7, shift 7 right, insert 2[2, 7, 9, 1, 5]
299 > 7, no shift, insert in place[2, 7, 9, 1, 5]
311 < 9, 7, 2 — shift all right, insert 1[1, 2, 7, 9, 5]
455 < 9, 7 — shift 9 and 7 right, insert 5[1, 2, 5, 7, 9]

The Code

public static void insertionSort(ArrayList<Integer> list) {
    for (int i = 1; i < list.size(); i++) {
        int key = list.get(i);        // the element to insert
        int j = i - 1;                // start comparing with the element to its left

        // Shift larger elements one position to the right
        while (j >= 0 && list.get(j) > key) {
            list.set(j + 1, list.get(j));
            j--;
        }

        // Insert key into its correct position
        list.set(j + 1, key);
    }
}

Why It Works

  • Starts at i = 1 — the first element (index 0) is naturally "sorted" by itself.
  • The inner while loop shifts elements right until it finds where key belongs.
  • If the list is already sorted, the while loop never runs: O(n) best case.
  • Worst case O(n²) — every element shifts everything before it (reverse-sorted list).

Sorting by HP Descending (Objects)

public static void insertionSortByHP(ArrayList<Player> list) {
    for (int i = 1; i < list.size(); i++) {
        Player key = list.get(i);
        int j = i - 1;

        // Shift players with LOWER HP to the right (descending order)
        while (j >= 0 && list.get(j).getHP() < key.getHP()) {
            list.set(j + 1, list.get(j));
            j--;
        }

        list.set(j + 1, key);
    }
}

Selection Sort vs Insertion Sort

Selection SortInsertion Sort
Strategy Find minimum, swap to front Insert each element into sorted region
Comparisons (best) Always ~n²/2 n - 1 (already sorted)
Comparisons (worst) ~n²/2 ~n²/2 (reverse sorted)
Swaps / Shifts At most n - 1 swaps Up to n²/2 shifts
Best use case When swaps are expensive (large objects) Nearly-sorted data; small datasets
Stable? No (swapping can reorder equals) Yes (equals stay in original order)
