ArrayList Traversal and Algorithms

Traversing an ArrayList looks almost identical to traversing an array. The only differences: .get(i) instead of [i], and .size() instead of .length. Everything you learned about array algorithms — search, filter, min/max, accumulate — carries over directly.

Standard For Loop

Use when you need the index (for searching, replacing, removing):

ArrayList<String> party = new ArrayList<>();
party.add("Hero"); party.add("Mage"); party.add("Rogue");

for (int i = 0; i < party.size(); i++) {
    System.out.println(i + ": " + party.get(i));
    // Can modify: party.set(i, ...);
}
// Output:
// 0: Hero
// 1: Mage
// 2: Rogue

Enhanced For Loop

Use when you only need to read each element (most common for display, sum, filter):

for (String member : party) {
    System.out.println(member);
    // CANNOT modify the list itself here
}
// Output:
// Hero
// Mage
// Rogue
Can't remove while traversing with enhanced for. Removing elements while using enhanced for causes a ConcurrentModificationException. If you need to remove elements based on a condition, use a standard for loop and decrement your index after removal, or build a new list.

Linear Search

Find an element and return its index. Return -1 if not found.

public static int findItem(ArrayList<Item> inventory, String name) {
    for (int i = 0; i < inventory.size(); i++) {
        if (inventory.get(i).getName().equals(name)) {
            return i;
        }
    }
    return -1;
}

Same pattern as array search — just .get(i) instead of [i].

Filtering — Return a New ArrayList

Build a new list containing only elements that match a condition:

public static ArrayList<Item> getHeavyItems(ArrayList<Item> inventory) {
    ArrayList<Item> result = new ArrayList<>();
    for (Item item : inventory) {
        if (item.getWeight() > 25) {
            result.add(item);
        }
    }
    return result;
}

Filter by Rarity

public static ArrayList<Item> getByRarity(ArrayList<Item> inventory, String rarity) {
    ArrayList<Item> result = new ArrayList<>();
    for (Item item : inventory) {
        if (item.getRarity().equalsIgnoreCase(rarity)) {
            result.add(item);
        }
    }
    return result;
}

Accumulator — Sum, Average, Count

// Sum all gold values
public static int getTotalValue(ArrayList<Item> inventory) {
    int total = 0;
    for (Item item : inventory) {
        total += item.getGoldValue();
    }
    return total;
}

// Average
public static double getAverageValue(ArrayList<Item> inventory) {
    if (inventory.size() == 0) return 0;
    int total = 0;
    for (Item item : inventory) {
        total += item.getGoldValue();
    }
    return (double) total / inventory.size();
}

// Count by rarity
public static int countByRarity(ArrayList<Item> inventory, String rarity) {
    int count = 0;
    for (Item item : inventory) {
        if (item.getRarity().equalsIgnoreCase(rarity)) {
            count++;
        }
    }
    return count;
}

Min and Max (with Objects)

// Find the most expensive item
public static Item findMostExpensive(ArrayList<Item> inventory) {
    if (inventory.size() == 0) return null;

    Item mostExpensive = inventory.get(0);
    for (Item item : inventory) {
        if (item.getGoldValue() > mostExpensive.getGoldValue()) {
            mostExpensive = item;
        }
    }
    return mostExpensive;
}

Removing While Traversing

Removing elements shifts indices. If you loop forward and remove, you'll skip elements. The safest approach: build a new list. 

// Remove all items of a given rarity
public static void removeByRarity(ArrayList<Item> inventory, String rarity) {
    // Option A: Build a new list (safest)
    ArrayList<Item> toKeep = new ArrayList<>();
    for (Item item : inventory) {
        if (!item.getRarity().equalsIgnoreCase(rarity)) {
            toKeep.add(item);
        }
    }
    inventory.clear();
    inventory.addAll(toKeep);

    // Option B: Loop backward
    for (int i = inventory.size() - 1; i >= 0; i--) {
        if (inventory.get(i).getRarity().equalsIgnoreCase(rarity)) {
            inventory.remove(i);
        }
    }
}

Socratic Checkpoint

1. The Pure Filter

When writing a method that filters an ArrayList (like getHeavyItems), we usually create and return a brand new ArrayList rather than deleting items from the original one. Why is this considered "safer" design?

2. The Concurrent Crash

If you try to use .remove() while inside an enhanced for loop, Java panics and throws a ConcurrentModificationException. Based on how enhanced loops work, why does Java forbid this?

Bug Hunt: The Survivor

A player uses a "Banish" spell to remove all Goblins from the room, but one Goblin always seems to survive. Find the bug.

ArrayList<String> room = new ArrayList<>();
room.add("Goblin");
room.add("Goblin");
room.add("Hero");

// Try to remove all goblins
for (int i = 0; i < room.size(); i++) {
    if (room.get(i).equals("Goblin")) {
        room.remove(i);
    }
}

Question: Trace the value of i and the state of the room list step-by-step. What happens to the second Goblin when the first Goblin is removed at i = 0?
