ArrayList Basics

An ArrayList is a resizable list — like an array, but it grows when you add elements and shrinks when you remove them. No fixed size. No manual counting. Just a list that works.

Why ArrayList Exists

Unit 6's inventory had a hard problem:

Item[] inventory = new Item[20];  // 20 slots, forever
int itemCount = 0;               // manual tracking, error-prone

// Adding item #21? Crash or "Inventory full!" — you lose.

ArrayList removes the problem:

ArrayList<Item> inventory = new ArrayList<>();
inventory.add(new Item("Sword", 3, 50));
inventory.add(new Item("Potion", 1, 25));
// ... add 100 more, no problem

No capacity. No itemCount. The list handles everything.

Importing ArrayList

ArrayList lives in java.util. Always import it at the top of your file:

import java.util.ArrayList;

Generics: ArrayList<E>

The angle brackets < > specify what type of element the list holds. This is called a generic — it makes the list type-safe. 

ArrayList<String>  names;     // a list of Strings
ArrayList<Integer> scores;    // a list of Integers (not int!)
ArrayList<Double>  temps;     // a list of Doubles
ArrayList<Player>  party;     // a list of your Player objects
ArrayList<Item>    inventory; // a list of your Item objects

You cannot use primitive types in generics:

ArrayList<int> list;    // COMPILE ERROR — can't use int
ArrayList<Integer> list; // CORRECT — use the wrapper class

Wrapper Classes

ArrayList only works with objects, not primitives. Java provides wrapper classes that box each primitive into an object:

PrimitiveWrapper Class
intInteger
doubleDouble
booleanBoolean
charCharacter

Autoboxing and Unboxing

Java converts between primitives and wrappers automatically:

ArrayList<Integer> list = new ArrayList<>();
list.add(5);           // autoboxing: int 5 → Integer automatically
list.add(10);          // same thing

int x = list.get(0);   // unboxing: Integer → int automatically
int y = list.get(1);   // same thing

You write int values and Java handles the conversion. This is why ArrayList<Integer> feels almost like int[].

Declaring and Creating

// Declare + instantiate (most common):
ArrayList<String> names = new ArrayList<>();

// The <> on the right is the "diamond operator" — Java infers the type.
// You could write: new ArrayList<String>()
// But <> is shorter and means the same thing.
Empty list. A new ArrayList starts with 0 elements. size() returns 0. There's nothing in it until you add().

The #1 Trap: remove(int) vs remove(Object)

For ArrayList<Integer>, remove(5) does not remove the number 5. It removes the element at index 5. 

ArrayList<Integer> list = new ArrayList<>();
list.add(3); list.add(7); list.add(2); list.add(9);
list.add(4); list.add(8); list.add(1); list.add(6);

list.remove(5);        // Removes element at INDEX 5 — the 8!
list.remove(2);        // Removes element at INDEX 2 — the 2!

// To remove by VALUE, cast to Integer:
list.remove((Integer) 7);  // Removes the number 7
list.remove((Integer) 3);  // Removes the number 3
Why this happens: Java sees remove(5) and matches it to remove(int index) because 5 is an int. To call remove(Object), you must pass an Integer object — that's what the cast (Integer) does.

Socratic Checkpoint

1. The Wrapper Mystery

Why do we have to write ArrayList<Integer> instead of ArrayList<int>? What is Java actually doing behind the scenes when you add(5)?

2. The Removal Trap

ArrayList<Integer> scores = new ArrayList<>();
// (Imagine we added the numbers 10, 20, 30, 40 here)
scores.remove(2);

Did this code just delete the number 2 from the list, or the number 30? Explain how to delete the other one.

Bug Hunt: The Impatient Index

A player is setting up their custom loadout, but the game crashes on line 3. Find the bug.

ArrayList<String> loadout = new ArrayList<>();
loadout.add(0, "Primary Weapon");
loadout.add(2, "Secondary Weapon"); // Game crashes here!

Question: What is the size() of the ArrayList when line 3 tries to run? What are the valid index numbers you can insert at?

Tinker Challenge: Dynamic Spellbook

Goal: Manage a wizard's growing spellbook.

  1. Create an empty ArrayList<String> called spells.
  2. Use .add() to append "Fireball" and "Heal" to the list.
  3. Use the other version of .add(index, element) to squeeze "Shield" directly into index 0 (pushing the others down).
  4. Print the list. Then use .remove() to delete "Heal" by providing the String itself (not the index number).
  5. Boss Move: Print the list's .size(). Then use .clear() and print the size again to prove it's empty.
