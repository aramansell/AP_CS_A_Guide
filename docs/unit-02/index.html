Unit 2: Using Objects

In Unit 1, you used primitive types — int, double, boolean. They hold exactly one value. For everything else, Java has objects. Objects bundle data AND behavior together. A String is an object. A Scanner is an object. Your Player character will be an object.

By the end of this unit, you'll build a Character Creator: the player types their name, you roll random stats using your Dice engine from Unit 1, and you print a formatted character sheet that introduces them to the dungeon.

Concepts

TopicWhat you'll learn
Objects & Classes What objects are. Constructors, methods, the dot operator.
The String Class length(), substring(), indexOf(), equals(), compareTo().
Scanner — Reading Input nextLine(), nextInt(), nextDouble(). Getting input from the keyboard.
The Math Class abs(), pow(), sqrt(), min(), max(). Math extends what primitive operations can do.
Wrapper Classes Integer, Double. Constants like Integer.MAX_VALUE. Parsing strings to numbers.

Your RPG Project

Character Creator (Student RPG Guide) — Read the player's name with Scanner. Roll stats (strength, dexterity, constitution, intelligence) using your Dice engine. Print a formatted character sheet. This Player.java object follows you through every unit for the rest of the year.

RPG Thread

The stats you roll here (STR, DEX, INT, CON) determine everything later. In Unit 3, STR becomes your attack bonus in combat. In Unit 4, CON determines starting HP. In Unit 9, these stats become instance variables on the Player class. The name you read here? Printed on the character sheet in Unit 3, displayed on the dungeon map in Unit 8.
