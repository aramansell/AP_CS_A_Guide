Rock-Paper-Scissors — Full Walkthrough

This is the program your teacher builds live. It combines Scanner (Unit 2) with conditionals (Unit 3 preview) to make a playable game. Use this page as a reference when building your own projects.

The Goal

Read the player's choice, generate a random choice for the computer, then determine the winner.

Rock, Paper, or Scissors? rock
You chose: rock
Computer chose: scissors
You win! Rock crushes scissors.

Step 1: Declare Variables

Scanner input = new Scanner(System.in);

String playerChoice;
int computerNum;
String computerChoice;

Why int for the computer? Generate a random 0/1/2 first, then map to a string. Same pattern as the Slot Machine. Easier to generate random numbers than random strings.

Step 2: Read Player, Generate Computer

System.out.print("Rock, Paper, or Scissors? ");
playerChoice = input.nextLine().toLowerCase();

computerNum = (int) (Math.random() * 3);

if (computerNum == 0) {
    computerChoice = "rock";
} else if (computerNum == 1) {
    computerChoice = "paper";
} else {
    computerChoice = "scissors";
}

System.out.println("You chose: " + playerChoice);
System.out.println("Computer chose: " + computerChoice);

Why toLowerCase()? Makes the comparison reliable. "Rock" and "rock" should be the same choice. toLowerCase() standardizes before we check.

Step 3: Determine the Winner

if (playerChoice.equals(computerChoice)) {
    System.out.println("It's a tie!");
} else if (playerChoice.equals("rock") && computerChoice.equals("scissors")) {
    System.out.println("You win! Rock crushes scissors.");
} else if (playerChoice.equals("paper") && computerChoice.equals("rock")) {
    System.out.println("You win! Paper covers rock.");
} else if (playerChoice.equals("scissors") && computerChoice.equals("paper")) {
    System.out.println("You win! Scissors cut paper.");
} else {
    System.out.println("Computer wins!");
}

Why .equals() and not ==? Strings are objects. == compares memory addresses, not content. .equals() compares the actual text. This is covered in detail in the String reference.

Alternative approach: Instead of 5 if/else branches, you could chain conditions: check ties, then check the three ways the player wins, and everything else is a computer win. Cleaner code.

The Full Program

import java.util.Scanner;

public class RockPaperScissors {
    public static void main(String[] args) {
        Scanner input = new Scanner(System.in);

        // ===== STEP 1: DECLARE =====
        String playerChoice;
        int computerNum;
        String computerChoice;

        // ===== STEP 2: INPUT + GENERATE =====
        System.out.print("Rock, Paper, or Scissors? ");
        playerChoice = input.nextLine().toLowerCase();

        computerNum = (int) (Math.random() * 3);
        if (computerNum == 0) computerChoice = "rock";
        else if (computerNum == 1) computerChoice = "paper";
        else computerChoice = "scissors";

        System.out.println("You chose: " + playerChoice);
        System.out.println("Computer chose: " + computerChoice);

        // ===== STEP 3: JUDGE =====
        if (playerChoice.equals(computerChoice)) {
            System.out.println("It's a tie!");
        } else if (playerChoice.equals("rock") && computerChoice.equals("scissors")) {
            System.out.println("You win! Rock crushes scissors.");
        } else if (playerChoice.equals("paper") && computerChoice.equals("rock")) {
            System.out.println("You win! Paper covers rock.");
        } else if (playerChoice.equals("scissors") && computerChoice.equals("paper")) {
            System.out.println("You win! Scissors cut paper.");
        } else {
            System.out.println("Computer wins!");
        }
    }
}
