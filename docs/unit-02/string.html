The String Class

You've been using Strings since hello world. But String is not a primitive — it's an object. That means it comes with methods. A LOT of them.

Strings Are Objects

String name = "Frodo";          // shortcut — Java does this for you
String name2 = new String("Frodo"); // explicit — same thing

// Strings have methods. Call them with a dot.
System.out.println(name.length());  // 5 — counts characters
System.out.println(name.toUpperCase()); // "FRODO"

Important: Strings are immutable. Methods like toUpperCase() don't change the original — they return a new String. 

String s = "hello";
s.toUpperCase();
System.out.println(s); // still "hello" — the original didn't change!

s = s.toUpperCase();   // NOW s is "HELLO" — we reassigned

The Essential Methods

length() — how many characters

"dragon".length();   // 6
"".length();          // 0 — empty string
" ".length();         // 1 — space is a character

substring(start, end) — slice out part

String s = "dungeon";
s.substring(0, 3);    // "dun" — characters 0,1,2 (stops BEFORE 3)
s.substring(3);       // "geon" — from 3 to end

indexOf(thing) — find position

"sword".indexOf("o");   // 2 — first 'o' is at index 2
"sword".indexOf("z");   // -1 — not found
"sword".indexOf("or");  // 2 — finds whole substring

equals(other) — compare contents

String a = "sword";
String b = "sword";
String c = "shield";

a.equals(b);  // true — same characters
a.equals(c);  // false

// NEVER use == to compare Strings!
a == b        // SOMETIMES true, sometimes false — DON'T DO THIS

compareTo(other) — ordering

"axe".compareTo("bow");     // negative — "axe" comes before "bow"
"bow".compareTo("axe");     // positive
"axe".compareTo("axe");     // 0 — they're equal

Tinker Challenge: String Slicing

String weapon = "Excalibur";
System.out.println(weapon.length());
System.out.println(weapon.substring(0, 3));
System.out.println(weapon.indexOf("cal"));
System.out.println(weapon.substring(weapon.indexOf("cal")));

Write your prediction for each line. Then run and check. Did the second substring behave differently than the first? Why?

String + Concatenation

int hp = 42;
System.out.println("HP: " + hp);          // "HP: 42"
System.out.println("Damage: " + (10 + 5)); // "Damage: 15"
System.out.println("Damage: " + 10 + 5);   // "Damage: 105" — TRAP!

+ runs left-to-right. "Damage: " + 10 becomes "Damage: 10", then "Damage: 10" + 5 becomes "Damage: 105". Use parentheses: "Damage: " + (10 + 5).

Bug Hunt: String Traps

Bug 1: The Identical Twins
String s1 = new String("magic");
String s2 = new String("magic");
if (s1 == s2) {
    System.out.println("They match!");
} else {
    System.out.println("They are different!");
}

Run this. Why does it say they are different? Fix the code so it correctly identifies them as matching.

Bug 2: The Stubborn String
String word = "dragon";
word.toUpperCase();
System.out.println(word);

Run this. Why is it still lowercase? Fix the code so it prints "DRAGON".
