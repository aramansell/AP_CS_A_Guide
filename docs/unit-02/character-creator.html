Student RPG Guide: Character Creator

This is YOUR guide. The teacher built Mad Libs and Rock-Paper-Scissors with you. Now you use those same skills — Scanner, String concatenation, Dice.roll() — to build a character generator on your own. Instead of code blanks, you are provided with Acceptance Criteria: if your code passes every test, it works.

The Setup

Create a new file called CharacterCreator.java. You will write four methods and a main driver. Your program reads the player's name with Scanner, rolls four RPG stats (Strength, Dexterity, Intelligence, Constitution) using your Dice.java engine from Unit 1, and prints a formatted character sheet.

Part 1: readName() — Get the Player's Name

Requirements:

  • Method signature: public static String readName(Scanner input)
  • Prints a prompt (use System.out.print — no newline — so the user types on the same line)
  • Calls input.nextLine() to read whatever the user types
  • Returns the name as a String

Socratic Checkpoint:

Why use System.out.print instead of System.out.println for the prompt? What would happen visually if you used println?

Acceptance Test:

Add this to a tester:

Scanner sc = new Scanner(System.in);
String name = CharacterCreator.readName(sc);
System.out.println("Got: " + name);

Run it. Verify the prompt appears correctly and whatever you type is returned.

Part 2: rollStats() — Roll Four Ability Scores

Requirements:

  • Method signature: public static int[] rollStats()
  • Rolls four stats using your Dice.java engine from Unit 1
  • Each stat uses 3d6: Dice.rollDice(3, 6)
  • Returns an int array in this order: {strength, dexterity, intelligence, constitution}

Socratic Checkpoint:

Why is Dice.rollDice(3, 6) used instead of Dice.roll(18) for a stat? What does 3d6 mean, and why does the sum distribution matter in a game?

Acceptance Test:

int[] stats = CharacterCreator.rollStats();
System.out.println("STR: " + stats[0]);
System.out.println("DEX: " + stats[1]);
System.out.println("INT: " + stats[2]);
System.out.println("CON: " + stats[3]);
// Each stat should be between 3 and 18 (3d6 range)
// Run multiple times to verify the range

Part 3: printSheet() — Display the Character Sheet

Requirements:

  • Method signature: public static void printSheet(String name, int[] stats)
  • Prints a formatted character sheet matching this template:
--- CHARACTER SHEET ---
Name: [name]
STR: [stats[0]]
DEX: [stats[1]]
INT: [stats[2]]
CON: [stats[3]]
-----------------------

Socratic Checkpoint:

The stats parameter is an array. What do stats[0], stats[1], etc. represent? What happens if you pass an array that only has 2 elements?

Acceptance Test:

int[] testStats = {14, 12, 16, 10};
CharacterCreator.printSheet("Thorn Ironvein", testStats);
// Verify the output matches the template exactly

Part 4: main() — Tie It All Together

Requirements:

  • Creates a Scanner connected to System.in
  • Calls readName(input) to get the player's name
  • Calls rollStats() to get the stat array
  • Calls printSheet(name, stats) to display the character sheet
  • Closes the Scanner with input.close()

Socratic Checkpoint:

Why is the Scanner closed at the end? What would happen if you tried to call input.nextLine() after closing it?

Acceptance Test:

Run CharacterCreator.main(). Type a name when prompted. Verify the full character sheet prints with all four stats between 3 and 18.
