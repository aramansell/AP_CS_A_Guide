The Math Class

In Unit 1, you learned arithmetic: +, -, *, /, %. The Math class gives you more powerful operations. All Math methods are static — you call them on the class itself, not on an object.

How Static Methods Work

// With a regular object:
Scanner input = new Scanner(System.in);  // make an object
String s = input.nextLine();             // call method on the object

// With a static method:
double result = Math.sqrt(25);           // call on the CLASS, no "new"

You never write new Math(). Math methods are tools you use directly. Think of Math as a toolbox — you don't build a new toolbox, you just reach in.

Essential Methods

Math.random() — you already know this one

double r = Math.random();  // 0.0 to 0.999...
// Combine with multiply/add/cast for any range

Math.abs(x) — absolute value

Math.abs(-5);    // 5
Math.abs(5);     // 5
Math.abs(-3.7);  // 3.7

// Useful for: "how far apart are two numbers?"
int diff = Math.abs(playerX - enemyX);

Math.pow(base, exponent) — raise to a power

Math.pow(2, 3);   // 8.0  (2³)
Math.pow(10, 2);  // 100.0
// Returns double — cast to int if you need an integer

Math.sqrt(x) — square root

Math.sqrt(25);    // 5.0
Math.sqrt(2);     // 1.414...
// Distance formula: Math.sqrt(x*x + y*y)

Math.min(a, b) and Math.max(a, b)

Math.min(3, 7);   // 3
Math.max(3, 7);   // 7

// Clamp a value between min and max:
int clamped = Math.max(0, Math.min(value, 100));

Tinker Challenge: Math Predictions

// What prints?
System.out.println(Math.abs(-12));
System.out.println(Math.pow(3, 2));
System.out.println(Math.max(Math.min(7, 3), 5));
System.out.println((int) Math.sqrt(10));

Write down your prediction for each line before running it. Did any of them surprise you? Why does the square root of 10 print as 3?

RPG Connection

Where does this show up in your game?

  • Damage calculation: Math.max(1, baseDamage + modifier) — minimum 1 damage
  • Distance: Math.abs(playerX - treasureX) — how close are you?
  • Stat caps: Math.min(20, strength + bonus) — max 20 strength
