Objects & Classes

The Problem With Primitives

An int holds a number. A double holds a decimal. A boolean holds true or false. But what if you need to represent a player character? A character has a name AND strength AND health AND gold. That's multiple pieces of data that belong together.

This is what objects are for. An object bundles related data and the operations you can perform on that data into one thing.

Classes Are Blueprints

A class is the blueprint. The object is the thing built from the blueprint. 

// The BLUEPRINT (class)
public class Player {
    String name;
    int strength;

    // Constructor — builds the object
    public Player(String n) {
        name = n;
        strength = 10;
    }

    // Method — something the object can do
    public void sayHello() {
        System.out.println("I am " + name + "!");
    }
}

Once the blueprint exists, you build objects from it: 

// The FIRST OBJECT — built from the blueprint
Player frodo = new Player("Frodo");

// The "." operator lets you talk to the object
frodo.sayHello();              // prints "I am Frodo!"
System.out.println(frodo.name); // prints "Frodo"

The Three Key Parts

1. The Constructor — runs when you write new. Sets up the new object.

public Player(String n) {   // constructor
    name = n;
    strength = 10;
}

Player p = new Player("Gimli");  // calls constructor

2. Instance Variables — the data that belongs to each object. Every Player has their own copy.

String name;    // each Player has their own name
int strength;   // each Player has their own strength

3. Methods — what the object can do. Called with the dot operator.

public void sayHello() {   // a method
    System.out.println("Hello!");
}

myPlayer.sayHello();        // call it with a dot

Tinker Challenge: Multiple Objects

Player a = new Player("Aragorn");
Player b = new Player("Boromir");

System.out.println(a.name);
System.out.println(b.name);

What prints? Write your prediction, then run it. Does creating b overwrite a's name?

Socratic Checkpoint: The Reference Trap

Player x = new Player("Legolas");
Player y = x;

x.name = "Gandalf";
System.out.println(y.name);

What prints? This is important. Run it. Why did changing x also change y? What exactly does the variable y hold?

Bug Hunt: Broken Blueprints

Bug 1: The Missing Word

Run this code:

Player p = Player("Frodo");

You get a "Cannot find symbol" error. Why? What keyword is required to build a new object from a blueprint?

Bug 2: The Empty Pointer

Run this code:

Player p;
p.sayHello();

You get a NullPointerException. This is one of the most famous crashes in Java. You declared the variable p, but what didn't you do? How do you fix it?
