Scanner — Reading Input From the Keyboard

Your programs so far have been one-directional: they print at you. Scanner lets the user talk back. It reads what the user types.

Setting Up Scanner

Three things you always need:

import java.util.Scanner;  // 1. Import — top of file, before the class

public class MyProgram {
    public static void main(String[] args) {
        Scanner input = new Scanner(System.in);  // 2. Create — once

        System.out.print("Enter your name: ");   // prompt
        String name = input.nextLine();          // 3. Read — blocks until Enter

        System.out.println("Hello, " + name + "!");
    }
}

Reading Different Types

String s = input.nextLine();   // whole line (including spaces)
int n = input.nextInt();       // next integer (skips whitespace)
double d = input.nextDouble(); // next decimal number
Bug Hunt: The Phantom Enter Key
Write a short program using Scanner that asks for a number, then a word: 
System.out.print("Age: ");
int age = input.nextInt();      

System.out.print("Name: ");
String name = input.nextLine(); 

System.out.println("You are " + age + " and your name is " + name);
Run it. Type 17, hit Enter. Wait, what happened? Did it let you type a name?
Observation: Why did the program skip the name input? What key did you press after typing "17" that the computer might have misread as your "Name"?
Socratic Checkpoint
The #1 Scanner bug is the "Input Buffer Trap". nextInt() reads the number "17" but leaves the Enter key you pressed in the buffer. The next nextLine() reads that leftover Enter and instantly thinks you gave it a blank line.
How can you add a single line of code after nextInt() to "eat" the leftover Enter key before it asks for the name?

Common Traps

  • Forgot to import — import java.util.Scanner; at the top
  • Buffer trap — extra nextLine() after nextInt/nextDouble
  • next() vs nextLine() — next() reads one word, nextLine() reads everything
  • Closing Scanner — don't close System.in Scanner (it closes System.in forever)
