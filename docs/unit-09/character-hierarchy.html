Student RPG Guide: GameCharacter + Refactored Player/Monster

Three files to rewrite. Player.java and Monster.java have shared the same HP management code since Unit 5. Extract the overlap into a single abstract parent: GameCharacter.java. Then refactor Player and Monster to extend it. Less duplication. One takeDamage() to maintain.

References

  • Abstract Classes — abstract keyword, abstract methods, why you can't new an abstract class
  • extends and super — inheritance syntax, super() constructor call, super.method()
  • Method Overriding — same signature in parent and child, @Override, calling super.toString()
  • Polymorphism — GameCharacter[] holding both Player and Monster, writing methods that work on any GameCharacter

What Moves to GameCharacter

Open your current Player.java and Monster.java side by side. Here's what's duplicated:

Duplicate CodeMoves to

What stays in Player/Monster is what makes them different: Player has stats (str/dex/int/con) and attack bonus calculation. Monster has armor class, damage die, boss flag, and defeated counter.

FILE 1: GameCharacter.java

NEW FILE. Abstract class. You can never write new GameCharacter(...) — that's the whole point.

Class Declaration + Instance Variables

public abstract class GameCharacter {
    protected String name;
    protected int hitPoints;
    protected int maxHitPoints;
}

Why protected and not private? Subclasses (Player, Monster) access these directly. If they were private, subclasses couldn't touch them — they'd need getters for their own data.

Constructor

    public GameCharacter(String name, int hp) {
        this.name = name;
        this.maxHitPoints = hp;
        this.hitPoints = hp;
    }

Abstract classes CAN have constructors. Subclasses call them with super(name, hp).

Shared Getters

    public String getName()         { return name; }
    public int getHitPoints()       { return hitPoints; }
    public int getMaxHitPoints()    { return maxHitPoints; }

    public boolean isAlive() {
        return hitPoints > 0;
    }

HP Management

    public void takeDamage(int amount) {
        hitPoints -= amount;
        if (hitPoints < 0) hitPoints = 0;
        System.out.println(name + " takes " + amount + " damage! HP: "
                           + hitPoints + "/" + maxHitPoints);
    }

    public void heal(int amount) {
        hitPoints += amount;
        if (hitPoints > maxHitPoints) hitPoints = maxHitPoints;
        System.out.println(name + " heals " + amount + " HP! HP: "
                           + hitPoints + "/" + maxHitPoints);
    }

Abstract Method

    public abstract int attack();

No body. No braces. Just a semicolon. Every concrete subclass MUST implement this. Player rolls a d8. Monster rolls its damage die (advantage for bosses). The parent class doesn't care HOW — it only declares "you must have an attack() method."

toString()

    public String toString() {
        return name + " (HP: " + hitPoints + "/" + maxHitPoints + ")";
    }

Subclasses override this and call super.toString() to prepend the shared part.

Full file: ~50 lines. Half your old Player.java. And you never touch this code again — all future HP changes happen here.

FILE 2: Player.java — Refactored

REWRITE your existing Player.java. Most of it disappears — GameCharacter handles name, HP, and damage.

public class Player extends GameCharacter {
    private int strength;
    private int dexterity;
    private int intelligence;
    private int constitution;

    public Player(String name, int hp, int str, int dex,
                  int intel, int con) {
        super(name, hp);          // MUST be the first line
        this.strength = str;
        this.dexterity = dex;
        this.intelligence = intel;
        this.constitution = con;
    }

    // Getters (unchanged from Unit 5)
    public int getStrength()      { return strength; }
    public int getDexterity()     { return dexterity; }
    public int getIntelligence()  { return intelligence; }
    public int getConstitution()  { return constitution; }

    public int getAttackBonus() {
        return strength / 4;      // integer division
    }

    // MUST implement the abstract method
    public int attack() {
        return Dice.roll(8);     // player rolls a d8
    }

    // Override to add stats
    public String toString() {
        return super.toString()
            + " STR:" + strength
            + " DEX:" + dexterity
            + " INT:" + intelligence
            + " CON:" + constitution;
    }
}

What disappeared: name field, hp fields, isAlive(), takeDamage(), heal(), getName(), getHitPoints(), getMaxHitPoints() — all inherited for free.

New constraint: The constructor's first line MUST be super(name, hp). Java enforces this. If you forget, the compiler says "call to super must be first statement in constructor."

FILE 3: Monster.java — Refactored

REWRITE your existing Monster.java. Same deal.

public class Monster extends GameCharacter {
    private int armorClass;
    private int damageDie;
    private boolean isBoss;
    private static int defeatedCount = 0;

    // Basic constructor
    public Monster(String name, int hp, int ac, int die) {
        super(name, hp);
        this.armorClass = ac;
        this.damageDie = die;
        this.isBoss = false;
    }

    // Boss constructor — chains to basic constructor
    public Monster(String name, int hp, int ac, int die, boolean boss) {
        this(name, hp, ac, die);   // calls constructor above
        this.isBoss = boss;
    }

    // Getters (unchanged)
    public int getArmorClass()  { return armorClass; }
    public int getDamageDie()   { return damageDie; }
    public boolean isBoss()     { return isBoss; }

    // MUST implement the abstract method
    public int attack() {
        int roll = Dice.roll(damageDie);
        if (isBoss) {
            int roll2 = Dice.roll(damageDie);
            if (roll2 > roll) roll = roll2;  // advantage
        }
        return roll;
    }

    // Override to count defeats
    public void takeDamage(int amount) {
        super.takeDamage(amount);
        if (!isAlive()) {
            defeatedCount++;
        }
    }

    public static int getDefeatedCount() {
        return defeatedCount;
    }

    // Override toString
    public String toString() {
        String result = super.toString()
            + " AC:" + armorClass + " d" + damageDie;
        if (isBoss) result = "☆ " + result;
        return result;
    }
}

Chain constructor: this(name, hp, ac, die) calls the basic constructor, which calls super(name, hp). Chain of delegation.

Override takeDamage: Call super.takeDamage(amount) first (the parent does the HP math and printing), then check if dead and increment the counter.

Test It

// === Test 1: Can't instantiate abstract class ===
// GameCharacter gc = new GameCharacter("Test", 10);  // COMPILE ERROR

// === Test 2: Player inherits everything ===
Player hero = new Player("Aragon", 30, 16, 14, 12, 18);
System.out.println(hero.getName());         // Aragon
System.out.println(hero.getHitPoints());    // 30
System.out.println(hero.isAlive());         // true
System.out.println(hero.getAttackBonus());  // 4 (= 16/4)
System.out.println(hero);                   // Aragon (HP: 30/30) STR:16 DEX:14 INT:12 CON:18

// === Test 3: Monster inherits everything ===
Monster goblin = new Monster("Goblin", 12, 10, 6);
System.out.println(goblin.getName());       // Goblin
System.out.println(goblin.getArmorClass()); // 10
System.out.println(goblin);                 // Goblin (HP: 12/12) AC:10 d6

// === Test 4: Boss with advantage attack ===
Monster boss = new Monster("Dragon", 50, 16, 12, true);
System.out.println(boss.isBoss());          // true
System.out.println(boss);                   // ☆ Dragon (HP: 50/50) AC:16 d12

// === Test 5: takeDamage works on both ===
hero.takeDamage(10);
System.out.println(hero.getHitPoints());    // 20
goblin.takeDamage(15);
System.out.println(goblin.getHitPoints());  // 0
System.out.println(goblin.isAlive());       // false
System.out.println(Monster.getDefeatedCount()); // 1

// === Test 6: heal works ===
hero.heal(5);
System.out.println(hero.getHitPoints());    // 25
hero.heal(999);
System.out.println(hero.getHitPoints());    // 30 (clamped to max)

// === Test 7: Heal doesn't revive ===
goblin.heal(10);
System.out.println(goblin.getHitPoints());  // still 0 (dead is dead)

// === Test 8: Polymorphism — GameCharacter array ===
GameCharacter[] party = new GameCharacter[3];
party[0] = new Player("Mage", 20, 8, 12, 18, 10);
party[1] = new Monster("Orc", 18, 12, 8);
party[2] = new Monster("Troll", 25, 14, 10, true);  // boss
for (GameCharacter gc : party) {
    System.out.println(gc.getName() + " attacks for " + gc.attack());
    // Each calls its OWN attack() — Player rolls d8, Orc rolls d8, Troll rolls d10 (adv)
}

// === Test 9: Polymorphic damage ===
for (GameCharacter gc : party) {
    gc.takeDamage(5);  // works on Player AND Monster — one method call
}
