Unit 9: Inheritance

You've written Player.java. You've written Monster.java. They have different names, different stats, different behavior. But they share a lot of the same code: hit points, attack methods, name fields. Copy-paste is the enemy of good code.

This unit introduces inheritance — the mechanism that lets one class inherit everything from another and then add its own specialties. Player and Monster both extend a shared GameCharacter class. Method overriding lets each subclass change how it attacks. Polymorphism lets you treat a Skeleton, Dragon, and Goblin all as the same type — a collection of GameCharacter objects.

Concepts

TopicWhat you'll learn
extends & super Creating subclasses. Calling parent constructors and methods.
Method Overriding Replacing a parent method with specialized behavior.
Abstract Classes Forcing subclasses to implement specific methods.
Polymorphism Treating multiple types as one. Polymorphic arrays and ArrayLists.

Your RPG Project

GameCharacter + Refactored Player/Monster (Student RPG Guide) — Extract everything Player and Monster share into a GameCharacter superclass. Player and Monster become subclasses. Create specialized enemy types — Skeleton, Dragon, Goblin — each with their own overridden attack method. Polymorphic combat: one ArrayList<GameCharacter> that holds your hero AND all enemies.

RPG Thread

This is the unification unit. Every class you've built across Units 1-8 gets organized into a hierarchy. Player and Monster share GameCharacter. Inventory items become subclasses of Item (Weapon, Armor, Potion). In Unit 10, recursive AoE spells target everything on the board — and because of polymorphism, one method handles Skeletons, Dragons, and Players identically.
