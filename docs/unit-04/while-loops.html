while Loops

A while loop repeats a block of code as long as a condition is true. It's an if statement that runs again and again.

Syntax

while (condition) {
    // body — runs REPEATEDLY while condition is true
}

Before each iteration, Java checks the condition. If true → run the body. Then check again. When the condition becomes false → stop. Skip to the code below.

Example: Countdown

int hp = 5;
while (hp > 0) {
    System.out.println("HP: " + hp);
    hp--;  // WITHOUT THIS → infinite loop!
}
System.out.println("Defeated!");

Three Essential Parts

Every while loop needs these three things:

  1. Initialize — set up the variable before the loop.
  2. Test — the condition checked at the top.
  3. Update — change something inside the body so the loop eventually ends.
// 1. INITIALIZE
int round = 1;

// 2. TEST                     3. UPDATE
while (round <= 5) {           round++;  // ← without this, infinite loop!
    System.out.println("Round " + round);
}

Application: Combat Rounds

Replace your single-round Combat.java with a loop:

int playerHP = 20;
int enemyHP = 12;
int round = 1;

while (playerHP > 0 && enemyHP > 0) {
    System.out.println("--- Round " + round + " ---");

    // Player attacks (your U3 code here)
    int pRoll = Dice.roll(20);
    if (pRoll == 20) {
        int dmg = (Dice.roll(8) + 3) * 2;
        enemyHP -= dmg;
        System.out.println("CRIT! " + dmg + " damage!");
    } else if (pRoll + 3 >= 12) {
        int dmg = Dice.roll(8) + 3;
        enemyHP -= dmg;
        System.out.println("Hit! " + dmg + " damage!");
    } else {
        System.out.println("Miss!");
    }

    if (enemyHP <= 0) break;  // enemy dead → stop

    // Enemy attacks
    int eRoll = Dice.roll(20);
    if (eRoll >= 14) {
        int dmg = Dice.roll(6) + 2;
        playerHP -= dmg;
        System.out.println("Goblin hits! " + dmg + " damage!");
    } else {
        System.out.println("Goblin misses!");
    }

    System.out.println("You: " + playerHP + " HP | Goblin: " + enemyHP + " HP");
    round++;
}

if (playerHP > 0) {
    System.out.println("Victory!");
} else {
    System.out.println("Defeat...");
}

Why break? After the player's turn, the enemy might already be dead. break exits the loop immediately — no need for the enemy to attack a corpse.

Sentinel Loops

A sentinel loop keeps going until the user enters a special value to stop. Useful for: "Attack, defend, or flee?" menus that repeat until "quit." 

Scanner input = new Scanner(System.in);
String command = "";

while (!command.equals("quit")) {
    System.out.print("> ");
    command = input.nextLine();

    if (command.equals("attack")) {
        System.out.println("You swing your sword!");
    } else if (command.equals("look")) {
        System.out.println("A dark corridor stretches ahead.");
    }
}
System.out.println("Goodbye.");

🧠 Socratic Checkpoint: The Zero-Iteration Loop

Look at the code below:

int potions = 0;
while (potions > 0) {
    System.out.println("Drinking potion...");  // NEVER runs
    potions--;
}

Think about it: How many times will "Drinking potion..." print? Why is it sometimes useful to have a loop that might run zero times (for example, in an inventory system)?
