Unit 4: Iteration (Loops)

Your programs so far do things once. One attack roll. One character sheet. One tip calculation. But games loop: combat rounds repeat until someone falls. You explore room after room. Your inventory has multiple items. You roll damage for every attack.

This unit teaches repetition: while loops, for loops, and nested loops. By the end, you'll build a Dungeon Crawler — combat rounds, room-by-room exploration, and an inventory system, all wrapping the code you've built across Units 1–3.

Concepts

TopicWhat you'll learn
while Loops Repeat while a condition is true. Combat rounds, sentinel patterns.
for Loops Count-controlled repetition. Dungeon rooms, inventory iteration.
Loops + Strings Traversing strings. Finding, counting, and transforming characters.
Nested Loops Loops inside loops. Dungeon grid, multiplication tables, patterns.
Tracing Loops Trace tables. How many times does this loop run? Common off-by-one bugs.
No teacher demos for this unit. Unit 4 is all hands-on exploration. The RPG guides below are the primary build — the teacher works through one with you, then you tackle the other on your own.

RPG Thread

Loops are how your RPG stays alive. The combat system from Unit 3 was one round — now a while loop makes it multi-round until someone falls. For loops generate room after room in the dungeon. In Unit 6, you'll loop through inventory arrays. In Unit 8, nested loops traverse the dungeon grid. Unit 10 recursion replaces some of these loops.
