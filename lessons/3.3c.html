3.3c — Combat Round Resolver: Who Goes First?

Expand Application ~30 min

Build: Initiative Resolver

In tabletop RPGs, combat order is determined by an initiative roll. The highest roll goes first. Build a system that compares two initiative rolls and determines the combat order.

Create a class called InitiativeResolver. Hardcode two int values: playerRoll and enemyRoll (values from 1-20).

Rules

  1. If playerRoll > enemyRoll: player goes first. Print the turn order.
  2. If enemyRoll > playerRoll: enemy goes first. Print the turn order.
  3. If they're tied: compare a int playerDex and int enemyDex — higher dex wins the tie.
  4. If STILL tied after dex: flip a coin (use Math.random() > 0.5) — heads means player wins.
  5. Print the full resolution: who rolled what, who had higher dex, and who goes first.

Edge Cases

What happens if both players roll the same AND have the same dex? Does your tiebreaker work? Test it with matching values.
