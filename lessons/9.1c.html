9.1c — Polymorphic Party: Mixed Character Types

Expand Application ~30 min

Build: Party with Polymorphism

The real power of inheritance is polymorphism: a Character[] array can hold Warriors, Mages, and any future subclass — and each one behaves according to its own overridden methods.

Using your Character/Warrior/Mage classes from 9.1b:

  1. Create a Character[] party = new Character[4].
  2. Fill it: Warrior("Aldric", 40, 12, 5), Mage("Mira", 25, 8, 50), Warrior("Grom", 50, 15, 7), Mage("Lyra", 20, 6, 40).
  3. Create an Enemy (just use Character): Character("Dragon", 80, 10).
  4. Loop through the party: each member attacks the Dragon. Print results.
  5. The Dragon attacks back — it picks a random party member each round. Use a loop until either the Dragon or the entire party is dead.

Polymorphism in Action

When you call party[i].attack(dragon), Java calls the Mage's attack() for Mages and Warrior's attack() for Warriors — even though the array type is Character[]. This is dynamic dispatch. What would you need to change to add a Rogue class?
