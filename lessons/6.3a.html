6.3a — Array Algorithms: Search, Shift, Swap

Discovery Lab ~25 min

Traversal is the engine. Algorithms are the things you build with it. Finding an item, inserting, removing, swapping — these are the core operations you'll use in every array-based program. The AP exam tests these patterns heavily.

Experiment: Find and Count

Create ArraySearch.java. Build these methods:

// Linear search: return index of target, or -1 if not found
public static int findIndex(int[] arr, int target) { ... }

// Count matches
public static int count(int[] arr, int target) { ... }

// Check if ALL elements meet a condition
public static boolean allAbove(int[] arr, int threshold) { ... }

Test each with the array {45, 72, 33, 72, 18, 72}. What does findIndex(arr, 72) return — first match or all matches?

Experiment: Swap and Shift

Run this code and trace what happens to the array after each operation:

int[] items = {10, 20, 30, 40, 50};

// Swap positions 1 and 3
int temp = items[1];
items[1] = items[3];
items[3] = temp;

// Print after swap

// Shift left: remove position 1, slide everything over
for (int i = 1; i < items.length - 1; i++) {
    items[i] = items[i + 1];
}
items[items.length - 1] = 0;  // clear the last slot

// Print after shift

Draw the array on paper at each step. This is how deletion works.

Shift Direction Trap: When removing an element by shifting left, you MUST go forward (left to right). If you go backward, you'll copy the same value into every slot. Try it — reverse the shift loop and see what happens. Read the Array Algorithms docs Bug Hunt for the classic shift-direction mistakes.

Build: Insertion

Write a method insert(int[] arr, int index, int value) that inserts value at the given position by shifting everything right. The last element falls off. Test cases:

  • Insert 99 at index 2: {10,20,30,40,50} → {10,20,99,30,40}
  • Insert at index 0: should work at the front
  • Insert at the last index: should work at the end

Socratic Checkpoint

  1. When inserting, why do you have to shift from right to left? What happens if you shift left to right?
  2. Why does the swap algorithm need a temporary variable? What if you just wrote items[1] = items[3]; items[3] = items[1];?
  3. Look at the docs section on the fencepost pattern. What is a fencepost problem and when does it show up?

Checkpoint — before moving on, you must be able to:

  • Implement linear search and count-by-condition
  • Swap two array elements using a temp variable
  • Insert and remove elements with proper shift direction
  • Explain the fencepost pattern
