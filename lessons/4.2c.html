4.2c — for Loops: Build a Dungeon Encounter Generator

Expand Application ~30 min

Dungeon Masters generate encounters: for each room, roll on a table to determine what's inside. Today you build a procedural encounter generator that creates a multi-room dungeon.

Build: DungeonGenerator.java

The program generates a dungeon with exactly 8 rooms. Each room gets:

  • A room number (1-8)
  • A random encounter type (roll 1-100):
RollEncounterDescription
1-50Empty RoomNothing but cobwebs
51-75Goblin (1-3)Small, weak, numerous
76-90SkeletonUndead, resists piercing
91-98ChestContains 10-50 gold
99-100Dragon!Boss encounter

Summary: 2 empty, 5 goblins, 1 skeleton, 1 chest, 1 dragon

Socratic Checkpoint

  1. You need to count how many goblins appear across all rooms. Where does the counter go — inside the loop or outside? And where do you update it?
  2. For "Goblin (1-3)", how do you generate a random number from 1 to 3? The formula is in the Math docs — find it.
  3. The summary line says "1 dragon" even when there are zero. How would you only print "X dragon" if X > 0? What statement do you need?

Tinker: Difficulty Modifier

Add a difficulty setting. The player can choose Easy, Normal, or Hard. Modify the roll ranges:

  • Easy: Dragon only on 100. More empty rooms (1-60).
  • Normal: Current table.
  • Hard: Dragon on 95-100. Fewer empty rooms (1-40).

You'll need to change the roll ranges — but the encounter types stay the same. Can you write this without duplicating all the range checks?

Bug Hunt: The Missing Room 8

A student wrote for (int i = 0; i < 8; i++) and printed "Room " + i. The output showed Room 0 through Room 7. Where's Room 8? What's the fix — change the loop or change the print statement?

Checkpoint — before moving on, you must be able to:

  • Use if-else chains inside a for loop to categorize data
  • Accumulate running totals across loop iterations
  • Generate random numbers within custom ranges using Math.random()
  • Conditionally print summary data only when counters are non-zero
