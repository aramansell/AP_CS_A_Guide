4.4a — Nested Loops: Loops Inside Loops

Discovery Lab ~25 min

One loop does one thing repeatedly. A nested loop is a loop inside another loop — the inner loop runs to completion for every single iteration of the outer loop. This is how you process grids, tables, and multi-dimensional problems.

The Pattern

for (int row = 1; row <= 3; row++) {       // OUTER: 3 iterations
    for (int col = 1; col <= 4; col++) {   // INNER: 4 iterations each
        System.out.print("(" + row + "," + col + ") ");
    }
    System.out.println();  // newline after each row
}

Total iterations: 3 × 4 = 12. The inner loop resets from scratch each time.

Experiment: See the Nesting

Create GridLab.java. Run the pattern above. Then:

  1. Change the outer loop to run 5 times instead of 3. What changes?
  2. Change the inner loop to print just the col value with spaces. What does the output look like?
  3. Add a print statement INSIDE the outer loop but BEFORE the inner loop. Then add one AFTER the inner loop. What's the difference?
  4. Multiply row × col and print the result. What famous table are you generating?
Variable Shadowing Trap: Open the Nested Loops docs. Find "The Neverending Dungeon" Bug Hunt. Read the broken code. Why does using the same variable name i in both loops cause an infinite loop? What's the fix?

Build: Dungeon Grid

Read the "Dungeon Grid" section in the docs. It shows a 5×5 map using a 2D char array. Create DungeonMap.java. Write nested loops that print the map. The inner loop prints one cell, the outer loop adds a newline after each row.

Then modify it: instead of printing the raw character, use if/else to print the room description. '.' → "[Empty]", 'E' → "[Enemy]", 'T' → "[Treasure]", 'B' → "[BOSS]".

Build: Triangle Patterns

Then reverse it — print the triangle upside down. What do you change in the inner loop's test condition?

Socratic Checkpoint

  1. Read "The Multiplication Rule" in the docs. If the outer loop runs 50 times and the inner loop runs 50 times, how many total iterations? Why does nesting multiply instead of add?
  2. The docs show "Combat with Multiple Enemies" — outer loop is rounds, inner loop is enemies. What would happen if you swapped them? (Enemies as outer, rounds as inner?) Trace through it.
  3. In the Dungeon Grid, why does the outer loop use row < dungeon.length and the inner loop use col < dungeon[row].length? What does dungeon[row].length give you?

Checkpoint — before moving on, you must be able to:

  • Write nested loops that process every cell of a grid
  • Predict the total number of iterations (outer × inner)
  • Use different loop variables for inner and outer loops
  • Make the inner loop depend on the outer loop variable
  • Explain when nested loops are the right tool
