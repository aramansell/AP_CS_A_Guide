3.2c — Combat Condition Builder: Multi-Factor Attacks

Expand Application ~30 min

Build: Sneak Attack Detector

RPG combat often has special attacks that only trigger when multiple conditions are met. You'll build a system that checks for sneak attacks, counter-attacks, and deathblows — all using boolean logic.

Create a class called CombatConditions. Set up these boolean variables (try different combinations):

  • boolean isHidden
  • boolean enemyIsDistracted
  • boolean playerHasDagger
  • boolean enemyHealthBelow25
  • boolean playerHealthBelow25

Rules to Implement

  1. Sneak Attack: player must be hidden AND have a dagger. Print "Sneak attack! x3 damage."
  2. Opportunity Attack: enemy is distracted OR player is hidden. Print "Opportunity attack! +5 to hit."
  3. Execution: enemy health below 25% AND player has dagger. Print "Execution! Instant kill."
  4. Desperate Strike: player health below 25% AND enemy is NOT distracted. Print "Desperate strike! x2 damage but -5 defense."

Test with at least 3 different combinations of boolean values. Can you find a scenario where TWO rules trigger simultaneously?
