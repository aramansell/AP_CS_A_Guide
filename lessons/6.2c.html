6.2c — Party Manager: Traverse Your RPG Team

Expand Application ~30 min

Build: Party Roster

RPG parties have multiple members. You need to traverse them to heal everyone, find the strongest, or check if anyone is down. Traversal patterns make this easy.

Create a class called PartyManager:

  1. Declare arrays to hold party data for 4 members:
    • String[] names = {"Aldric", "Mira", "Grom", "Lyra"}
    • int[] hp = {35, 28, 42, 0}
    • int[] maxHp = {35, 30, 45, 25}
  2. Method void healAll(int amount): traverse hp array, add amount (cap at maxHp).
  3. Method int countAlive(): traverse and count members where hp > 0.
  4. Method String findStrongest(): return name of member with highest current HP.
  5. Method boolean isPartyWiped(): return true if ALL hp == 0.

Design Question

You're using three parallel arrays (names, hp, maxHp). What are the risks of this approach? What would happen if you sorted one array but not the others? Is there a better way?
