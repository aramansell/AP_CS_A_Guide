2.3c — Scanner: Build a Combat Dialogue Loop

Expand Application ~30 min

Character creation was a one-shot process — ask questions, get answers, print a sheet. But real RPGs are interactive loops: the game asks the player what to do, the player responds, the game reacts, and it repeats.

This is your first interactive game loop. The program doesn't end after one action — it keeps asking until the player chooses to quit. Every turn-based RPG works this way.

Build: CombatDialogue.java

Create a simple turn-based combat dialogue. The player fights a goblin. Each turn, the game shows the player's HP and the goblin's HP, then asks what the player wants to do. 

═══════════════════════════════
  COMBAT — Thorn vs Goblin
═══════════════════════════════
  Your HP: 30    Goblin HP: 15

  What do you do?
  [1] Attack
  [2] Drink potion
  [3] Flee
  Choice:

The game must:

  • Read the player's choice with nextInt()
  • If choice is 1: deal random damage (5-10) to goblin, goblin hits back for 3-6
  • If choice is 2: heal 8 HP, goblin hits back
  • If choice is 3: flee (prints "You flee!" and ends)
  • If goblin HP reaches 0: player wins
  • If player HP reaches 0: player loses
  • After each turn, display updated HP

Socratic Checkpoint

  1. You need a loop. What kind? Should it be a for loop or a while loop? Why? (If you haven't learned loops yet, look ahead at the while loops docs — the concept is simple: "keep doing this until someone wins.")
  2. Where do you check for the win/lose condition? Before the turn? After? Both?
  3. If the player enters 4 or -1 or "hello", what happens? How would you detect and handle bad input?

Tinker: Add More Actions

Extend the combat system with these additional actions:

  • Special Attack (choice 4): deals 10-20 damage but you take 5 damage from exhaustion
  • Scan (choice 5): reveals the goblin's exact stats (doesn't cost a turn if the goblin doesn't attack back)
  • What happens to your buffer traps when you add more nextInt() reads?

Bug Hunt: The Non-Numeric Crash

Try typing "attack" instead of a number when prompted for a choice. What error do you get? Why does nextInt() crash on text input?

For now, just be aware of this limitation. Later, you'll learn how to read everything as a String with nextLine() and convert manually — which solves this problem completely.

Checkpoint — before moving on, you must be able to:

  • Build an interactive loop that reads player input repeatedly
  • Mix nextInt() with conditional logic to create game mechanics
  • Handle the win/lose state inside a loop
  • Identify where buffer traps appear in multi-turn input patterns
