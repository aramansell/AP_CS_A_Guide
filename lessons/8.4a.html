8.4a — DungeonMap: The 2D Grid Foundation

RPG Build ~20 min

Your DungeonCrawl game has been room-by-room so far. But real dungeons are grids — rooms connected by corridors, with walls, treasure, and monsters at specific coordinates. Today you create the grid itself: a 2D char array with wall borders and the ability to set and read individual tiles.

Open the Dungeon Map Builder (Student RPG Guide). Follow Part 1.

Build: DungeonMap.java — The Grid

Create DungeonMap.java:

  • Private field: char[][] tiles
  • Constructor takes rows and cols, fills entire grid with '.' (floor)
  • Place wall borders: top row, bottom row, left col, right col all '#'
  • setTile(int row, int col, char symbol) — sets a single tile
  • getTile(int row, int col) — returns tile or ' ' if out of bounds
  • display() — prints the entire grid with a space after each char

Socratic Checkpoint

  1. You fill the grid with '.' then overwrite edges with '#'. Why two passes? Could you do it in one nested loop?
  2. getTile() returns ' ' for out-of-bounds instead of crashing. Why is this safer? Where in your future code will this matter?
  3. display() uses tiles.length for rows and tiles[0].length for columns. What happens if the grid has 0 rows?

Acceptance — your DungeonMap must:

  • Display a bordered rectangle of floor tiles
  • Allow setting and reading individual tiles
  • Handle out-of-bounds reads without crashing
