8.4a — Dungeon Map Builder: 2D Grid RPG

RPG Build ~60 min

Your DungeonCrawl game has been room-by-room so far. But real dungeons are grids — rooms connected by corridors, with walls, treasure, and monsters at specific coordinates. You're building a tile-based dungeon map using a 2D char array.

Open the Dungeon Map Builder (Student RPG Guide). Follow the specification.

Part 1: DungeonMap.java

Create DungeonMap.java:

  • Private field: char[][] tiles
  • Constructor takes rows and cols, fills entire grid with '.' (floor)
  • Place wall borders: top row, bottom row, left col, right col all '#'
  • setTile(int row, int col, char symbol) — sets a single tile
  • getTile(int row, int col) — returns tile or ' ' if out of bounds
  • display() — prints the entire grid with a space after each char

Part 2: Room Placement

Add to DungeonMap:

  • addRoom(int startRow, int startCol, int height, int width) — carves out a rectangular room: walls '#' border, floor '.' inside. Must stay within map bounds.
  • addDoor(int row, int col) — sets tile to '+' (door). Must be on an existing wall.
  • addCorridor(int row1, int col1, int row2, int col2) — draws a straight-line '.' path between two points. Keep it simple: horizontal then vertical.

Part 3: Entity Placement

Add entity markers:

  • placePlayer(int row, int col) — sets tile to 'P'. Only one allowed.
  • placeMonster(int row, int col) — sets tile to 'M'
  • placeTreasure(int row, int col) — sets tile to 'T'
  • count(char symbol) — returns how many tiles match the given symbol

Integrate Into Your RPG

Wire DungeonMap into DungeonCrawl. Replace the room-number tracking with map coordinates. Player starts at 'P'. Movement: read direction (N/S/E/W), check if destination is '.' or 'T' (not '#' or out of bounds), move the 'P'. When the player steps on 'M', trigger combat. When they step on 'T', give loot.

Socratic Checkpoint

  1. Your addRoom() fills a rectangular area. How do you avoid overwriting the outer border walls?
  2. When the player moves, you need to erase the old 'P' and draw a new one. What was on the old tile before the player stood there — and how do you remember it?
  3. Read the docs section on line-of-sight. Why is checking every tile between two points harder than it looks?

Checkpoint — before moving on, you must be able to:

  • Build a 2D char grid with walls, rooms, corridors, and entities
  • Move a player token through the grid with boundary checking
  • Integrate tile-based movement into the game loop
