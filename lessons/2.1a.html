2.1a — Objects & Classes: What's an Object?

Discovery Lab ~15 min

Unit 1 was all about primitive types: int, double, boolean. Each one holds exactly one value. That's like a single piece of paper.

An object is like a folder. It bundles multiple pieces of data AND the operations that work on that data — all in one package. In Java, almost everything you use day-to-day is an object.

Socratic Checkpoint

  1. What's in a player character? Name, HP, strength, gold, inventory... Why would a player character be a terrible candidate for a single int variable?
  2. You already used objects in Unit 1. System.out is an object. "hello".length() — what kind of object is "hello"? How many objects have you used without realizing it?

The Three-Step Object Pattern

Every object in Java follows this pattern:

// 1. DECLARE — give it a type and a name
Scanner input;

// 2. CONSTRUCT — create it with new
input = new Scanner(System.in);

// 3. USE — call methods on it with the dot operator
String name = input.nextLine();

Or all at once:

Scanner input = new Scanner(System.in);  // declare + construct
String name = input.nextLine();            // use
Key insight: new is the magic word that creates objects. Without new, you're just declaring a variable that points to nothing (null).

Try It: The Dice Object

In Unit 1.7, you built a Dice.java utility class. But you used it with static methods — Dice.roll(20). That works for utilities, but it's not how most objects work. Try this:

Step 1: Create DiceObject.java — make a non-static version:

public class DiceObject {
    public int roll(int sides) {
        return (int)(Math.random() * sides) + 1;
    }
}

Step 2: Create DiceTester.java and do this:

public class DiceTester {
    public static void main(String[] args) {
        // 1. DECLARE
        DiceObject d20;

        // 2. CONSTRUCT
        d20 = new DiceObject();

        // 3. USE
        int result = d20.roll(20);
        System.out.println("You rolled a " + result + " on a d20!");
    }
}

Socratic Checkpoint

  1. What keyword did you use to CREATE the DiceObject?
  2. What operator did you use to CALL the roll method?
  3. Why did you need new DiceObject() here but NOT for Dice.roll(20) in Unit 1.7?
  4. If you forget new and just write DiceObject d20; then d20.roll(20);, what happens? Try it.

Bug Hunt: NullPointerException — Your New Nemesis

The most common error you'll see this unit. It means you tried to use an object that was never created with new.

  1. Write a program that declares a String variable but never assigns it, then call .length() on it. What error do you get?
  2. Fix it by assigning a value. What changes?
  3. Write a program that declares a Scanner, never calls new, and tries to read input. Same error?

Checkpoint — before moving on, you must be able to:

  • Explain what an object is and how it differs from a primitive
  • Write the three-step pattern (declare, construct, use) from memory
  • Identify the new keyword and explain what it does
  • Recognize and fix a NullPointerException
