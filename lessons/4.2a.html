4.2a — for Loops: When You Know the Count

Discovery Lab ~25 min

while loops are for unknown counts — "keep fighting until someone dies." But what about "generate exactly 5 rooms" or "print inventory items 1 through 10"? When you know the count, a for loop packs all the loop control into one clean line.

A for loop bundles initialize, test, and update into its header. It's not different from while — it's just organized differently.

The Pattern

for (initialize; test; update) {
    // body
}
// A while loop:
int i = 0;           // initialize
while (i < 5) {      // test
    System.out.println(i);
    i++;             // update
}

// The SAME thing as a for loop:
for (int i = 0; i < 5; i++) {
    System.out.println(i);
}

Take your CombatLoop.java from 4.1a. The combat loop is a while loop because you don't know how many rounds. But can you add a round counter that prints the round number? Try it.

Build: Counting Patterns

The for Loops docs have a section called "Counting Patterns." It shows five different loop headers. Write a program that creates all five loops and prints what each one outputs.

Before running, PREDICT the output of each one. Write it down. Then run. How many did you get right?

Which pattern would you use for: printing even numbers? Printing numbers 10, 20, 30 up to 100? Countdown from 10?

Socratic Checkpoint

  1. The docs say: "while = unknown repetitions, for = known repetitions." Think of a game situation where you'd use NEITHER — where the repetition needs both a counter AND an unknown exit condition. How would you write that?
  2. Read "The Missing Item" Bug Hunt in the docs. Why does the buggy inventory loop crash? What's the difference between starting at 0 vs starting at 1 when your array has 5 items?
  3. Look at the enhanced for loop at the bottom of the docs. When would you use for (String item : items) instead of for (int i = 0; ...)? What can the index version do that the enhanced version can't?

Checkpoint — before moving on, you must be able to:

  • Write a for loop with any of the five counting patterns
  • Explain when to use for vs while
  • Explain loop variable scope (why i disappears after the loop)
  • Rewrite a while loop as a for loop and vice versa
