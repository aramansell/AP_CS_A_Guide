2.2a — String Is an Object

Discovery Lab ~30 min

You've used Strings since your first program. But String is not a primitive — it's a full-blown object with methods. Today you explore what Strings can do on your own.

Launch — Predict This

Open a new file StringLab.java. For each block below, write your prediction first, then run the code, then record what actually happened.

Section 1: length()

String weapon = "Excalibur";
System.out.println(weapon.length());         // predict: ____
System.out.println("dragon".length());       // predict: ____
System.out.println("".length());             // predict: ____
System.out.println(" ".length());            // predict: ____

What's the length of an empty String? Of a space? Why does the second line even work — "dragon" isn't a variable, it's just there?

Section 2: substring()

String dungeon = "Underdark";

System.out.println(dungeon.substring(0, 5));   // predict: ____
System.out.println(dungeon.substring(5));      // predict: ____
System.out.println(dungeon.substring(1, 4));   // predict: ____

What's the difference between substring(5) with one argument vs two arguments? Run experiments until you can write a rule for how substring works.

Section 3: indexOf()

String spell = "Fireball";

System.out.println(spell.indexOf("F"));     // predict: ____
System.out.println(spell.indexOf("e"));     // predict: ____
System.out.println(spell.indexOf("ball"));  // predict: ____
System.out.println(spell.indexOf("x"));     // predict: ____

What does indexOf return when it can't find the thing? If there are two 'l' characters in "Fireball", which one does indexOf find?

Section 4: equals() vs ==

String a = new String("sword");
String b = new String("sword");

System.out.println(a == b);       // predict: ____
System.out.println(a.equals(b));  // predict: ____

Both variables hold the word "sword". Why does == say false? What is == actually checking?

Section 5: The Immutability Trap

String s = "hello";
s.toUpperCase();
System.out.println(s);  // predict: ____

If this prints "hello" (not "HELLO")... the method didn't change anything. You need to figure out why. Open the String docs and search for the word "immutable." What does it mean? What do you have to do differently to actually get "HELLO"? Try things until it works. Don't ask the instructor until you've tried at least two different approaches.

Tinker Challenge: Password Checker

Write a program that checks a String password against these rules:

  • Length must be at least 8 characters
  • Must contain the '@' symbol somewhere
  • The first character must NOT be a number

Try it with passwords like "hello", "pass@word", and "1password@". Print whether each one passes.

Stuck on how to check length or find the '@' symbol? Open the String docs — the methods you need are listed there. Figure out which ones match these tasks.

Checkpoint — before moving on, you must be able to:

  • Call length(), substring(), indexOf(), equals(), and compareTo() and predict their output
  • Explain why == doesn't work for comparing String contents
  • Explain String immutability — what happens when you call toUpperCase() without reassigning
  • Use substring and indexOf together to extract a piece of a String
