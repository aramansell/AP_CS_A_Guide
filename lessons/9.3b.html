9.3b — Polymorphism: Building a Type-Aware System

Basic Application ~25 min

Polymorphism isn't just about calling attack() — it's about writing code that works with any Creature without knowing which specific type it is. Today you build systems that treat all creatures the same way, yet each creature behavior is unique.

Build 1: Creature Healer

Create CreatureHealer.java. A healer NPC visits your party and heals every creature:

  • Method: healAll(Creature[] party) — loops through every creature
  • Each creature gets 20% of its max HP back (round to int)
  • After healing, print each creature's name and new HP
  • Only heal living creatures — dead ones stay dead
// Test with mixed party:
Creature[] party = {new Goblin("Snag"), new Dragon("Pyrax"), new Skeleton("Bones")};
// Apply damage first (you decide how much)
// Then healAll()
// Output each creature's status

Socratic Checkpoint

  1. The healer method signature says Creature[] party. But the actual objects are Goblin, Dragon, Skeleton. How does the loop work when it calls party[i].heal() — does it call the Creature version or the child's version?
  2. If Goblin overrides heal() but Dragon doesn't, which heal() does a Dragon object get when called through a Creature reference?
  3. You need max HP. Where is it stored? Is it in Creature, or must each subclass define its own?

Build 2: Creature Sorter

Create CreatureSorter.java. Given an array of Creatures, sort them by a criteria:

  • sortByHP(Creature[] creatures) — highest HP first
  • sortByAttack(Creature[] creatures) — highest attack first
  • printCreatureList(Creature[] creatures) — formatted output

You'll need to compare creatures by their stats. If you haven't learned sorting algorithms yet, just find the creature with the highest HP, print it, then find the next highest, repeat — a simple selection approach using nested loops.

Tinker: Filter by Type

Add a method filterByType(Creature[] creatures, String typeName) that returns only creatures whose class name matches. Use creature.getClass().getSimpleName() — check the Inheritance docs for how getClass() works. This lets you isolate all Goblins from a mixed party.

Bug Hunt: The Null Creature

What if the party array has null slots (dead creatures removed)? party[i] == null — your loop must skip these without crashing. Where do you put the null check — before every method call? Before the loop body? How does this pattern look?

Checkpoint — before moving on, you must be able to:

  • Operate on Creature[] arrays polymorphically without casting
  • Sort/filter mixed-type collections using parent type references
  • Check for null references before calling methods
  • Explain which method version runs when called through a parent reference
