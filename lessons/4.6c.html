4.6c — Dungeon Crawl: Nested Loops & Multi-Floor Dungeon

RPG Build ~25 min

The Goal

Your DungeonLoop runs one floor of 5 rooms. But real dungeons have multiple floors — each floor is a series of rooms, and the dungeon is a series of floors. That's nested loops: an outer loop over floors, an inner loop over rooms. You'll build a DungeonCrawl.java that chains together room exploration, combat encounters, treasure, a final boss, and an end-of-dungeon summary.

Open the Dungeon Crawl Extended (Student RPG Guide). Follow the specification. This combines everything from Unit 4 into one system.

generateRoom() — Random Encounter Engine

Extract the room logic from DungeonLoop into a reusable method with scaling:

public static boolean generateRoom(int[] playerHP, int playerAC, int playerAtk,
                                 int[] gold, String[] inventory,
                                 int floor, int roomNumber)
  • Parameters use arrays so the method can modify HP, gold, and inventory in place — primitives can't be modified by a called method
  • Random room type: 0 = combat, 1 = treasure, 2 = trap
  • Combat: scaled monsters with while-loop attack pattern from 4.6a
  • Treasure: random gold + append to inventory string
  • Trap: damage, player can die

Socratic Checkpoint

  1. Why are playerHP, gold, and inventory passed as arrays instead of plain int and String? What's different about array parameters?
  2. You're building a monster name array: Goblin, Skeleton, Orc, etc. The index formula in the docs is (roomNumber + floor - 2) % names.length. Trace it: floor 1, room 1 → what index? floor 2, room 3 → what index?

exploreFloor() — Multi-Room Floor Loop

A floor has 3 + floor rooms (later floors are longer). For-loop over rooms, calling generateRoom() each time. After all rooms: restore floor × 3 HP (rest at end of floor).

runDungeon() — Multi-Floor Outer Loop

Outer for loop: floor 1 to totalFloors. Calls exploreFloor(). After all floors: bossFight(), then printSummary(). Four levels of loops: outer for (floors) → inner for (rooms) → generateRoom → while (combat).

Socratic Checkpoint

  1. Draw the loop structure. Outer for controls floors, inner for controls rooms, while controls combat rounds. Which loop runs the most times? Which runs the fewest?
  2. Later floors have more rooms (3 + floor). Floor 1 has 4 rooms. Floor 5 has 8 rooms. Is that fair to the player? What else could you scale to keep it balanced?

bossFight() + printSummary()

Boss: Ancient Dragon if ≥ 3 floors, Ogre Lord otherwise. Uses the same while-loop combat pattern. The boss CAN critical hit. Victory adds floors × 50 gold.

printSummary(): show HP remaining vs starting, total gold, inventory items (split on comma, loop through).

Tinker: Inventory Tracking

The docs use a comma-separated String for inventory. But that's fragile — what if an item name contains a comma? Design a better way to store multiple items. We'll explore this in Unit 6 with arrays.

Checkpoint — before moving on, you must be able to:

  • Write nested for loops (floors containing rooms)
  • Explain why array parameters let methods modify caller data
  • Build a multi-level game loop: while (combat) inside for (rooms) inside for (floors)
  • Trace through all four loop levels for a given input
