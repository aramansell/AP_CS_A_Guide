7.2c — Enemy Horde Manager: Spawn, Damage, Remove

Expand Application ~30 min

Build: Horde Manager

In action RPGs, enemies constantly spawn and die. The enemy list is dynamic — ArrayList is the natural choice. But removing dead enemies while iterating requires the safe pattern you just learned.

Create a class called HordeManager. Use TWO parallel ArrayLists: ArrayList<String> enemyNames and ArrayList<Integer> enemyHP.

  1. void spawn(String name, int hp): add a new enemy.
  2. void damageAll(int amount): subtract amount from every enemy's HP.
  3. void removeDead(): traverse backwards and remove any enemy with HP <= 0. Print a death message for each.
  4. void displayHorde(): print all living enemies with their HP.

Stress Test

Spawn 6 enemies with HP 10. Call damageAll(4). Remove dead. DamageAll(4). Remove dead. How many are left? Call removeDead() when none are dead — does it crash?
