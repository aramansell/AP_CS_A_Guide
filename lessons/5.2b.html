5.2b — Overloaded Constructors: Multiple Ways to Build

Basic Application ~20 min

Build: Weapon with Defaults

Sometimes you want to create an object with full detail; other times you want a quick default version. Overloaded constructors give you both.

Create a class called Weapon:

  1. Instance variables: String name, int damage, String damageType.
  2. Constructor 1: takes name only. Sets damage = 5, damageType = "physical".
  3. Constructor 2: takes name and damage. Sets damageType = "physical".
  4. Constructor 3: takes all three values.
  5. IMPORTANT: Each constructor should call the most detailed one using this(name, damage, "physical") or similar. Don't duplicate assignment code.

Test

Create weapons using all three constructors. What happens if you only provide a constructor with all 3 params but try to call new Weapon("Sword")? Why doesn't Java just use defaults?
