5.5a — Player.java: Your RPG's Hero

RPG Build ~60 min

For three units, your RPG has used scattered variables — int playerHP, String playerName, scattered across DungeonCrawl. Every time you need player stats, you pass four arguments. Every time you update HP, you write the same validation code. It's time to wrap all of that into a single Player class.

Open the Player Builder (Student RPG Guide). This is a step-by-step guided build. Follow the specification exactly. Your completed Player.java must pass the acceptance tests at the bottom of the guide.

Build: Player.java

Create Player.java with these specifications:

  • Private instance variables: name (String), hp (int), maxHp (int), ac (int), strength (int), dexterity (int), constitution (int)
  • Constructor takes all seven values. Clamp hp between 0 and maxHp.
  • Getters for all fields. Boolean getter: isAlive()
  • takeDamage(int amount) — subtract, clamp to 0
  • heal(int amount) — add, clamp to maxHp
  • toString() — formatted like "Hero [HP: 20/25 AC:14 STR:12]"

Integrate Into Your RPG

Open your DungeonCrawl.java. Replace every scattered player variable:

// BEFORE
int playerHP = 25;
int playerMaxHP = 25;
int playerAC = 14;
String playerName = "Hero";

// AFTER
Player player = new Player("Hero", 25, 14, 12, 16, 10);

// Replace: playerHP -= damage     → player.takeDamage(damage)
// Replace: if (playerHP > 0)      → if (player.isAlive())
// Replace: playerHP += healAmt    → player.heal(healAmt)

Run your dungeon crawler. Does everything still work? Fix any compile errors (type mismatches, method name typos).

Socratic Checkpoint

  1. Your old code had if (playerHP > 0) scattered everywhere. Now it's player.isAlive(). Why is the second version better even though it does the same thing?
  2. If you decided a player is dead at HP ≤ 0, where would you change that? Your old code vs your new code — how many file changes in each?

Acceptance — your Player.java must:

  • Compile without errors
  • Pass all acceptance tests in the Student RPG Guide
  • Integrate into DungeonCrawl.java with no logic changes
