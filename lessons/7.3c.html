7.3c — Leaderboard: ArrayList of Objects

Type: Expand Application. Strings were easy. Now store your own objects in an ArrayList, sort them, and find the top performers. The full leaderboard pattern.

Expand Application ~30 min

The Goal

Build a Player class with a name and score, store them in an ArrayList, sort by score, and print a ranked leaderboard.

=== LEADERBOARD ===
1. Alice      - 2150
2. Diana      - 1880
3. Bob        - 1720
4. Charlie    - 1610
5. Eve        - 1540

Reference: Leaderboard Walkthrough — full reference.

Build It: Player Class + Leaderboard

Create Player.java:

public class Player {
    private String name;
    private int score;

    public Player(String name, int score) {
        this.name = name;
        this.score = score;
    }

    public String getName() { return name; }
    public int getScore() { return score; }

    public String toString() {
        return name + " - " + score;
    }
}

Create Leaderboard.java. Add at least 5 players:

import java.util.ArrayList;

public class Leaderboard {
    public static void main(String[] args) {
        ArrayList<Player> players = new ArrayList<>();

        players.add(new Player("Alice", 2150));
        players.add(new Player("Bob", 1720));
        players.add(new Player("Charlie", 1610));
        players.add(new Player("Diana", 1880));
        players.add(new Player("Eve", 1540));

        // YOUR CODE: print unsorted
    }
}

The Sort Challenge

Sort the ArrayList by score, highest first. You CANNOT use Collections.sort() yet — that requires Comparable (coming in 7.4). Instead, use the selection sort pattern you learned with arrays:

// Selection sort by score (descending)
for (int i = 0; i < players.size() - 1; i++) {
    int maxIndex = i;
    for (int j = i + 1; j < players.size(); j++) {
        if (players.get(j).getScore() > players.get(maxIndex).getScore()) {
            maxIndex = j;
        }
    }
    // swap players[i] with players[maxIndex]
    Player temp = players.get(i);
    players.set(i, players.get(maxIndex));
    players.set(maxIndex, temp);
}

Socratic Checkpoint

  1. Why does the inner loop start at i + 1 instead of 0?
  2. What changes if you want ascending order (lowest first)?
  3. How is this sort structurally identical to the card shuffle swap pattern? What's the only thing that changed?

Extend It (pick one)

  • Add the ability to update a player's score and re-sort.
  • Track "games played" per player and compute average score per game.
  • Add new players from Scanner input and insert them in the correct sorted position (no full re-sort).
