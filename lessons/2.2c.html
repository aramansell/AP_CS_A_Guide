2.2c — String Methods: Build a Combat Command Parser

Expand Application ~30 min

Text-based RPGs work by parsing commands: the player types something like attack goblin and the game figures out what they meant. Today you build a command parser that splits player input into a verb and a target.

What is parsing? Parsing means taking a raw string and extracting structured information from it. "attack goblin" is just characters — until you find the space, split on it, and recognize "attack" as a verb and "goblin" as a target.

Build: CombatCommandParser

Create CombatParser.java with a main method that does this:

  1. Hard-code a command string: String command = "attack goblin";
  2. Find the space between the verb and the target
  3. Extract the verb (everything before the space)
  4. Extract the target (everything after the space)
  5. Print: "Action: [verb], Target: [target]"
// Example output for "cast heal ally":
Action: cast
Target: ally

The String docs have substring and indexOf examples. You've already built something similar in 2.2b with the email extractor. This is the same pattern — but now you need to handle commands that might have extra spaces.

Bug Hunt: Extra Spaces and Edge Cases

Test your parser with these edge cases. Some will break. Fix them one at a time.

"attack    dragon"    // multiple spaces between verb and target
"  attack goblin  "    // leading and trailing spaces
"attack"                // no target at all
""                      // empty command

What method removes leading/trailing whitespace? Check the String docs. What should your parser do when there's no target? (Hint: print a message like "No target specified" instead of crashing.)

Tinker: Handle Multiple Commands

What if commands aren't just "verb target"? What if they're longer? 

"attack goblin with sword"   → verb: attack, target: goblin, tool: sword
"cast heal on ally"          → verb: cast, target: heal, recipient: ally

Extend your parser to handle three-part commands. You'll need substring and indexOf with a start position. The String docs show how to find the second occurrence of a character.

Build: Command Validator

Not every command is valid. Add validation to your parser:

  • Only these verbs are allowed: attack, cast, use, flee
  • If the verb isn't recognized, print "Unknown command: [verb]"
  • For attack and cast, a target is required — if missing, print an error
  • For flee, no target is needed

Socratic Checkpoint

  1. You used equals() to check the verb. What would happen if you used == instead? Why?
  2. If the player types "ATTACK GOBLIN", does your parser recognize it? What String method would fix this?
  3. Look at the String docs. Find split(). How could you rewrite your entire parser using split instead of indexOf + substring? Which approach is cleaner? Which gives you more control?

Checkpoint — before moving on, you must be able to:

  • Parse a command string into verb and target using substring and indexOf
  • Handle edge cases: multiple spaces, leading/trailing spaces, missing target
  • Validate commands against a list of allowed verbs
  • Explain the tradeoff between split() and manual indexOf parsing
