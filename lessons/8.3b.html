8.3b — Tic-Tac-Toe (Build Together)

Type: Basic Application. The teacher builds, you follow along. A 2D array as a game board — print it, place moves, and detect wins. Every grid-based game starts with this skeleton.

Basic Application ~35 min

The Goal

Build a two-player Tic-Tac-Toe game using a 3×3 char[][] board. Players alternate placing X and O until someone wins or the board is full.

  0 1 2
0 [ ][ ][ ]
1 [ ][ ][ ]
2 [ ][ ][ ]

Player X: enter row col > 1 1

  0 1 2
0 [ ][ ][ ]
1 [ ][X][ ]
2 [ ][ ][ ]

Iteration 1: Create the Board

Create TicTacToe.java:

public class TicTacToe {
    public static void main(String[] args) {
        char[][] board = new char[3][3];

        // Initialize with spaces
        for (int r = 0; r < 3; r++) {
            for (int c = 0; c < 3; c++) {
                board[r][c] = ' ';
            }
        }

        printBoard(board);
    }

    public static void printBoard(char[][] board) {
        System.out.println("  0 1 2");
        for (int r = 0; r < 3; r++) {
            System.out.print(r + " ");
            for (int c = 0; c < 3; c++) {
                System.out.print("[" + board[r][c] + "]");
            }
            System.out.println();
        }
    }
}

Socratic Checkpoint

  1. Why use char[][] instead of String[][]? What's the difference?
  2. What would the board look like if you forgot to fill it with spaces? Try it.
  3. Why is printBoard static? What would need to change to make it non-static?

Iteration 2: Place Moves

Add Scanner and let two players alternate:

Scanner input = new Scanner(System.in);
char currentPlayer = 'X';

while (true) {
    System.out.print("Player " + currentPlayer + ": enter row col > ");
    int row = input.nextInt();
    int col = input.nextInt();

    board[row][col] = currentPlayer;
    printBoard(board);

    // Switch player
    if (currentPlayer == 'X') {
        currentPlayer = 'O';
    } else {
        currentPlayer = 'X';
    }
}

Bug Hunt

What happens if a player types "1 1" on a square that already has an X or O? It overwrites it! Add a check: only place the move if the square is still empty. Also: what if they type "5 5"? Add bounds checking.

Iteration 3: Win Detection (The Hard Part)

You need to check 8 possible wins: 3 rows, 3 columns, 2 diagonals. DO NOT write 8 separate if-statements. Use loops:

public static boolean checkWin(char[][] board, char player) {
    // Check rows
    for (int r = 0; r < 3; r++) {
        if (board[r][0] == player &&
            board[r][1] == player &&
            board[r][2] == player) {
            return true;
        }
    }
    // YOUR CODE: check columns (same idea, swap r and c)
    // YOUR CODE: check two diagonals
    return false;
}

Tinker Challenge

The win check above works but doesn't scale — what if you wanted a 5×5 board? Rewrite the row check using a loop that checks every cell in the row without hardcoding each column index. (Hint: you'll need a boolean flag inside the outer loop.)

Socratic Checkpoint

  1. Why is the grid-based Tic-Tac-Toe board the perfect model for your RPG's TileMap? What's the same?
  2. How would you adapt this board for a Connect Four game (6 rows, 7 columns, pieces drop to the bottom)?
