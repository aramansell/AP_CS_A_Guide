3.3a — else-if Chains: More Than Two Paths

Discovery Lab ~25 min

if/else gives you two paths. But what about four? Or ten? That's where else if chains come in.

Java checks conditions top to bottom. The FIRST true condition runs. Then it skips everything else. Exactly ONE block executes.

The Pattern

if (condition1) {
    // first priority
} else if (condition2) {
    // second priority
} else if (condition3) {
    // third priority
} else {
    // if nothing matched
}

Build: Damage Classifier

Write an if/else if chain that reads a damage number and prints the correct classification. Here are the ranges:

  • 0 → "Miss!"
  • 1–5 → "Scratch"
  • 6–10 → "Solid hit"
  • 11–15 → "Heavy blow"
  • 16+ → "Devastating!"

Test every edge! What happens when you pass in exactly 5? Exactly 6? Exactly 15? Exactly 16? Make sure each number lands in exactly one category.

Order Matters Trap: Before you read ahead, try to predict: if you write your conditions as if (dmg > 0) first, then else if (dmg > 5), then else if (dmg > 10)... what goes wrong? Run it and find out. Then fix it. Why does checking from LOWEST to HIGHEST (or HIGHEST to LOWEST) matter?

Socratic Checkpoint — Boolean Zen

Open the else-if Chains docs. Find the Socratic Checkpoint section labeled "Boolean Zen." Then answer:

  1. Why is if (enemyAlive) preferred over if (enemyAlive == true)? What does the if statement actually expect inside its parentheses?
  2. Look at this code: 
    if (hp > 0) {
    return true;
} else {
    return false;
}
    How could you rewrite this in a single line? (Hint: hp > 0 itself evaluates to something. What?)

Build: Grade Letter Converter

Use Scanner to read a numeric score (0–100). Use an if/else if chain to print the corresponding letter grade: 90+=A, 80+=B, 70+=C, 60+=D, below 60=F.

Read the docs for a hint about the order trap — they have a bug hunt that shows exactly the mistake students make with this problem. Learn from it before you write your code!

Build: Combat Result Printer

Roll a d20 using your Dice class. Print the result based on the roll:

  • 20 → "CRITICAL HIT!"
  • 15–19 → "Hit!"
  • 10–14 → "Glancing blow"
  • 2–9 → "Miss!"
  • 1 → "CRITICAL MISS!"

Which direction should your conditions go — top-down from 20, or bottom-up from 1? Think about the order trap before you start.

Checkpoint — before moving on, you must be able to:

  • Write multi-way decisions using if/else if/else chains
  • Explain why the ORDER of conditions matters
  • Test edge values to verify exactly one branch executes
  • Apply Boolean Zen — use boolean variables directly in conditions
