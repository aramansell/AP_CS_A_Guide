5.4b — The Card Class (Build Together)

Type: Basic Application. The teacher builds, you follow along. Your first multi-field class — private fields, a constructor, getters, and a toString. Every class you write from now on follows this skeleton.

Basic Application ~30 min

The Goal

Build a Card class with a suit and a rank. Then write a CardTester that creates a few cards and prints them.

Ace of Spades
7 of Hearts
Queen of Diamonds

Reference: Card Class Walkthrough — the full reference with the constructor formula.

Iteration 1: The Skeleton

Create Card.java. This is the minimum structure every class needs:

public class Card {
    // Step 1: PRIVATE fields
    private String suit;
    private String rank;

    // Step 2: CONSTRUCTOR (same name as class, NO return type)
    public Card() {
        suit = "Spades";
        rank = "Ace";
    }
}

Bug Hunt

Create a CardTester.java with a main method. Try Card c = new Card(); then System.out.println(c.suit);. What error do you get? Why? Fix it WITHOUT making suit public.

Iteration 2: Getters

Since the fields are private, you need public methods to read them:

public String getSuit() {
    return suit;
}

public String getRank() {
    return rank;
}

Now in CardTester: c.getSuit() works. Try it.

Iteration 3: The Constructor Trap

Right now, every Card is the Ace of Spades. That's useless. We want new Card("Hearts", "7"). But if you just add parameters, this breaks:

// If you ADD a parameterized constructor:
public Card(String s, String r) {
    suit = s;
    rank = r;
}

// Then this BREAKS in CardTester:
Card c = new Card();  // ERROR: no default constructor!

Socratic Checkpoint

  1. Why does adding a parameterized constructor remove the default one? (The compiler only gives you a free default if you write NO constructors at all.)
  2. How do you fix this so BOTH new Card() and new Card("Hearts", "7") work?

Iteration 4: toString()

Right now System.out.println(c) prints garbage like Card@15db9742. Add a toString():

public String toString() {
    return rank + " of " + suit;
}

Now System.out.println(c) prints "Ace of Spades". Why does it automatically call toString()? Check the docs.

Checkpoint — Your Card class must have:

  • Two private fields: suit and rank
  • A default constructor AND a parameterized constructor
  • getSuit() and getRank()
  • A toString() method
  • Tested in CardTester with at least 3 different cards
