1.5a — Tip Calculator (Build Together)

Type: Basic Application. The Slot Machine used Math.random() for game logic; the Tip Calculator uses arithmetic and the three-step pattern for real-world math. Same skeleton, different meat.

Basic Application ~30 min

The Goal

Given a bill amount and a tip percentage, calculate and display the tip and total.

===== TIP CALCULATOR =====
Bill:  $45.5
Tip:   $9.1
Total: $54.6
==========================

Iteration 1: Declare the Data

Create a new class called TipCalculator with a main method. Declare four variables:

  1. The starting bill amount — set it to 45.50
  2. The tip percentage — set it to 20
  3. The calculated tip amount — don't set a value yet
  4. The final total amount — don't set a value yet

Socratic Checkpoint

What data type should you use for these variables? Why is int a bad choice for money?

Iteration 2: The Math Trap

Now calculate the tip. 20 percent means 20 ÷ 100.

Bug Hunt: The Zero Dollar Tip

Add this calculation to your code:

tip = billAmount * (tipPercent / 100);
System.out.println("Debug Tip: " + tip);

Run it. The output is Debug Tip: 0.0. Why?

Fix it. This is the integer division trap. What happens when you do 20 / 100 with ints? How do you force Java to do floating-point division instead?

Iteration 3: Display the Receipt

Once the math is fixed, calculate total (bill + tip). Then replace your debug print with a nicely formatted receipt matching the Goal output above.

Socratic Checkpoint

  1. Compare this program to the Slot Machine. What's the same? What's different?
  2. Why does the three-step pattern work for BOTH programs when they do such different things?
