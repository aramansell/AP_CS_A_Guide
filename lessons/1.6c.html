1.6c — Reflect & Design

Expand Application ~10 min

Discuss with a partner or journal:

  1. When we added the "split between friends" feature, which section of the three-step pattern changed? (Declare? Calculate? Display?)
  2. When we added the "random tip" feature, which section changed?
  3. What if we needed to add sales tax? Where would that go in the three-step pattern?

Design Challenge

If you wanted to build a "restaurant bill splitter" from scratch tomorrow — one that handles tax, tip, and splitting — what would your skeleton look like? Write it out: just the class, main method, and three comment blocks.

Now sketch the variables. What would you need to declare? Bill amount, tax rate, tip percentage, number of people... what else?

Checkpoint — before moving on, you must be able to:

  • Design a program skeleton from scratch using the three-step pattern
  • Identify which step needs to change when a new feature is added
  • Explain why the three-step pattern makes adding features easier
