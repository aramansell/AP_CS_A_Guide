2.3a — Scanner: The Program Listens

Basic Application ~25 min

Your programs have been one-way — you talk AT the user. Scanner reverses the flow: it reads what the user types into the keyboard.

Scanner Setup — Three Things Always Required

Every Scanner program needs these three pieces. Create a file called ScannerSetup.java and type this in:

import java.util.Scanner;  // 1. IMPORT — top of file, before the class

public class ScannerSetup {
    public static void main(String[] args) {
        Scanner input = new Scanner(System.in);  // 2. CREATE — the object

        System.out.print("Enter your name: ");   // prompt the user
        String name = input.nextLine();          // 3. READ — wait for Enter

        System.out.println("Hello, " + name + "!");
    }
}

Socratic Checkpoint

  1. Where in this code do you see the three-step object pattern (declare, construct, use)?
  2. Why is System.out.print used instead of println for the prompt?
  3. What would happen if you forgot the import line? Try it and see the error.

Reading Different Types

String wholeLine = input.nextLine();    // reads everything until Enter
int number = input.nextInt();           // reads the next integer
double decimal = input.nextDouble();    // reads the next decimal number

THE Buffer Trap — Your #1 Scanner Bug

Copy this code into a file called BufferTrap.java and run it:

import java.util.Scanner;

public class BufferTrap {
    public static void main(String[] args) {
        Scanner input = new Scanner(System.in);

        System.out.print("Age: ");
        int age = input.nextInt();

        System.out.print("Name: ");
        String name = input.nextLine();

        System.out.println("You are " + age + " and your name is " + name);
    }
}

Run it. Type 17 then press Enter. What happened? Did it skip the name? Why?

Socratic Checkpoint — Diagnose the Trap

  1. What happened when you ran it? Describe the behavior carefully.
  2. You typed a number, then pressed Enter. The number got read, but did the Enter keypress get consumed too? What evidence do you have for your answer?
  3. Open the Scanner docs. Find the section about mixing nextInt and nextLine. What does it say about why this happens?
  4. What's the fix? The docs show a one-line fix. Find it. Try it. Don't copy-paste — understand why it works.
int age = input.nextInt();
input.nextLine();  // ← What does this do? Why here?

System.out.print("Name: ");
String name = input.nextLine();  // now this works correctly

You must be able to explain WHY this fix works, not just that it does. If you can't explain it yet, re-read the Scanner docs.

Build It: Weapon Chooser

Create WeaponChooser.java:

  • Ask "What is your weapon?" — read it with nextLine()
  • Ask "How much damage does it do?" — read it with nextInt()
  • Print: "Your [weapon] does [damage] damage."
  • Fix the buffer trap if you ask for damage first!

Tinker Challenge: Two-Player Setup

Extend WeaponChooser to ask for TWO weapons (one for each hand). Read the name and damage for each. Print a summary. Where do the buffer traps appear?

Checkpoint — before moving on, you must be able to:

  • Set up Scanner: import, create, read — from memory
  • Use nextLine(), nextInt(), and nextDouble() correctly
  • Diagnose and fix the buffer trap (nextInt followed by nextLine)
  • Explain WHY the buffer trap happens (leftover Enter key)
