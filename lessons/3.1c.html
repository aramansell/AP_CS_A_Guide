3.1c — Combat Hit Check: Building a Damage System

Expand Application ~30 min

Build: Hit or Miss?

In most RPGs, an attack doesn't always land. There's a hit chance, a damage roll, and sometimes a critical hit. You now know enough about if statements to build the core of a combat damage calculator.

Create a class called CombatHitCheck. It should:

  1. Declare int attackRoll and assign it a value from 1-20 (hardcode a few test values).
  2. Declare int targetAC (armor class) — set it to 12.
  3. If attackRoll >= targetAC: the attack hits. Print damage.
  4. If attackRoll == 20: it's a critical hit — double the damage.
  5. If attackRoll == 1: it's a critical miss — no damage, and print a funny message.
  6. Otherwise: the attack misses.

Test Values

Run your program with attackRoll = 20, 15, 8, 1. Does each produce the correct output? Trace through your if/else chain on paper for the 15 case — which condition fires first?

Think: Order Matters

You have two special cases (20 and 1) and a general case. If you check attackRoll >= targetAC before checking attackRoll == 20, what happens? Why must special cases come first?
