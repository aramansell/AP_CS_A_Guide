8.3a — 2D Array Algorithms: Row Sums, Transpose, Borders

Discovery Lab ~25 min

The AP exam loves 2D array algorithm questions. The patterns repeat: row operations, column operations, diagonals, borders, and flood fills. Master these five patterns and you can handle any 2D array FRQ.

Experiment: Row and Column Operations

Create GridAlgorithms.java. Build each as a separate method:

// Return sum of a specific row
public static int rowSum(int[][] grid, int row) { ... }

// Return sum of a specific column
public static int colSum(int[][] grid, int col) { ... }

// Return array of all row sums
public static int[] allRowSums(int[][] grid) { ... }

// Return index of row with largest sum
public static int maxRowIndex(int[][] grid) { ... }

Test with: {{1,2,3}, {4,5,6}, {7,8,9}}. Row sums: 6, 15, 24. Col sums: 12, 15, 18.

Experiment: Transpose (Flip Rows and Columns)

// Transpose: grid[r][c] becomes result[c][r]
public static int[][] transpose(int[][] grid) {
    int rows = grid.length;
    int cols = grid[0].length;
    int[][] result = new int[cols][rows];  // swapped!
    for (int r = 0; r < rows; r++)
        for (int c = 0; c < cols; c++)
            result[c][r] = grid[r][c];
    return result;
}

What happens if the grid isn't square? Try transposing a 2x5 grid.

Experiment: Border Sum

// Sum all elements on the outer border (first/last row, first/last col)
public static int borderSum(int[][] grid) { ... }

Test with a 4x4 grid. Be careful not to double-count corners.

Corner Double-Count: When summing borders, corners belong to BOTH the top/bottom row AND the left/right column. If you just sum row 0 + last row + col 0 + last col, you count corners twice. Read the Traversal Patterns docs for the clean border-sum pattern.

Socratic Checkpoint

  1. Transpose swaps dimensions. If you transpose twice, do you get the original back? Prove it.
  2. For a square grid, can you transpose in-place (without creating a new array)? What's the swap pattern?
  3. Read the docs section on the flood-fill algorithm. Why is it recursive?

Checkpoint — before moving on, you must be able to:

  • Compute row sums, column sums, and aggregate row statistics
  • Transpose a 2D array (both square and rectangular)
  • Sum border elements without double-counting corners
