2.5a — The Math Class: Static Methods

Basic Application ~25 min

+, -, *, /, % — you learned those in Unit 1. The Math class gives you more powerful operations. But Math works differently from the objects you've used so far.

Static vs Instance — What's Different?

// Instance method call (the pattern you know):
Scanner input = new Scanner(System.in);  // create an object FIRST
String s = input.nextLine();             // then call methods ON the object

// Static method call (new pattern):
double result = Math.sqrt(25);           // call directly on the CLASS

You never write new Math(). Math methods are static — they belong to the class itself, not to individual objects. Think of Math as a toolbox bolted to the wall — everyone reaches into the same toolbox.

Socratic Checkpoint

  1. Why do you need new Scanner(...) but NOT new Math()?
  2. If Math methods are static, what word is probably in their method signatures? (Hint: you saw static methods in Dice.java during Unit 1.)

The Essential Math Methods

Create MathLab.java. For each method, write your prediction, run it, then record what happened.

Math.abs(x) — distance, never negative

System.out.println(Math.abs(-7));     // predict: ____
System.out.println(Math.abs(7));      // predict: ____
System.out.println(Math.abs(-3.5));   // predict: ____

// RPG use: "How far apart are two coordinates?"
int diff = Math.abs(playerX - enemyX);

Math.pow(base, exp) — raise to a power

System.out.println(Math.pow(2, 3));    // predict: ____  (2³)
System.out.println(Math.pow(5, 2));    // predict: ____  (5²)
System.out.println(Math.pow(10, 0));   // predict: ____ (anything^0)

// Warning: Math.pow returns a double!
int squared = (int) Math.pow(7, 2);  // cast to int if you need an int

Math.sqrt(x) — square root

System.out.println(Math.sqrt(25));    // predict: ____
System.out.println(Math.sqrt(2));     // predict: ____ (not a clean number)
System.out.println(Math.sqrt(-1));    // predict: ____ (what happens?)

Math.min(a,b) and Math.max(a,b)

System.out.println(Math.min(3, 7));   // predict: ____
System.out.println(Math.max(3, 7));   // predict: ____

// Combine them to clamp a value:
int clamped = Math.max(0, Math.min(value, 100));
// If value = -5  → max(0, -5)  = 0
// If value = 50  → max(0, 50)  = 50
// If value = 150 → max(0, 100) = 100
