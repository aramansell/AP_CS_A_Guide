3.1a — Making Decisions: Relational Operators & if

Discovery Lab ~20 min

Every program you've written so far runs the same way every time. Declare, calculate, print. But real programs make decisions.

A relational operator asks a question about your data. It compares two values and answers true or false. The if statement uses that answer to decide what code to run.

The Six Operators

OperatorQuestion it asks
<Is the left less than the right?
>Is the left greater than the right?
<=Is the left less than or equal?
>=Is the left greater than or equal?
==Are they equal?
!=Are they NOT equal?

The if Statement

if (condition) {
    // runs ONLY if condition is true
}
// execution continues here regardless

else — The Other Path

if (condition) {
    // runs if true
} else {
    // runs if false
}
// Exactly ONE block runs. Never both. Always one.

Build: Temperature Checker

Create a file called TemperatureChecker.java. Write a program that:

  1. Declares an int temperature = 75 (or pick your own number)
  2. Uses if/else to print "It's hot!" if the temperature is above 80, or "It's not too hot." otherwise
  3. Change the number to 95 and run again. Then to 50. What prints each time?

Now add a second check: if temperature is below 32, also print "Watch out for ice!" Think carefully — does this go inside the else? Or is it a separate check?

Build: Coin Flip

Roll a random number from your Dice.java class. Use if/else to print "Heads!" or "Tails!" based on the result. How do you decide where the split point is?

Trap Ahead: There's a difference between = (one equals sign) and == (two equals signs). One assigns, one compares. If you mix them up, Java will yell at you — or worse, do something silently wrong. Try writing if (x = 5) and see what happens. Then try if (x == 5). What's the difference in what Java tells you?

Socratic Checkpoint

  1. If you run System.out.println(3 < 5), what prints? What about System.out.println(7 >= 7)? What about System.out.println(4 == 4.0) — predict before you test!
  2. What does if (false) { ... } do? Will the code inside ever run?
  3. Look at your Temperature Checker code. If you had three different temperature ranges (hot, cold, mild), would your current approach work? Why or why not?
  4. Open the Relational Operators docs. Find the Bug Hunt section. What trap does it warn you about with the single = sign? Why can't Java use it in an if condition?

Checkpoint — before moving on, you must be able to:

  • Write an if/else statement that makes a decision based on a comparison
  • Explain the difference between = and ==
  • Use all six relational operators in a program
  • Predict what a boolean expression evaluates to before running it
