7.1c — RPG Quest Log: Dynamic Quest Tracking

Expand Application ~30 min

Build: QuestLog Class

Quest logs grow as you accept quests and shrink as you complete them. You don't know how many quests there will be — ArrayList is perfect for this.

Create a class called QuestLog with an ArrayList<String> activeQuests and an ArrayList<String> completedQuests:

  1. void acceptQuest(String name): add to activeQuests. But first, check if it's already active or completed — if so, print a message and don't add.
  2. void completeQuest(String name): remove from activeQuests, add to completedQuests. If not found in active, print "No such active quest."
  3. void abandonQuest(String name): remove from activeQuests without adding to completed.
  4. void displayActive(): print all active quests with numbers (1-based).
  5. int totalCompleted(): return completedQuests.size().

Test Flow

Accept "Slay the Dragon", "Find the Herb", "Rescue the Smith". Complete "Find the Herb". Try to accept "Find the Herb" again — should refuse. Display active quests. What's left?
