1.2a — Printing & Concatenation

Discovery Lab ~20 min

To show things to the user, you print to the terminal. Java gives you System.out.println and System.out.print.

println vs. print

Concatenation (+) with Strings

Inside println, + glues text and values together. This is called concatenation. 

String name = "Aria";
int level = 5;
System.out.println("Character: " + name + ", Level: " + level);
Character: Aria, Level: 5

Tinker Challenge: The Math Illusion

Create a file called PrintTest.java. Run this code:

System.out.println("Sum: " + 3 + 5);

Wait, what?! Why does it print Sum: 35 instead of Sum: 8? What do you think Java is doing here?

Socratic Checkpoint

How can you use parentheses to force Java to do the math first, so that it prints Sum: 8? Test your theory.

What happened?

Java evaluates + left to right. When it sees a String and an int, it glues them together into a new String.

  1. "Sum: " + 3 → Java concatenates: "Sum: 3"
  2. "Sum: 3" + 5 → same thing: "Sum: 35"

Parentheses force the math to happen FIRST, before concatenation.

int a = 10;
int b = 3;
System.out.println(a + b);              // 13 (math, no string)
System.out.println("" + a + b);         // 103 (all concatenation)
System.out.println("" + (a + b));       // 13 (math first, then concatenate)
System.out.println(a + " + " + b + " = " + (a + b));  // 10 + 3 = 13

Formatting Tips

// Empty println gives a blank line
System.out.println();

// Separator lines
System.out.println("====================");

// Aligned columns (tabs)
System.out.println("Name\tLevel\tHP");
System.out.println("Aria\t5\t85");

\t is a tab character. It helps line things up.

Escape Sequences

CodeWhat it produces
\nNew line
\tTab

System.out.println("She said \"Roll for initiative!\"");
She said "Roll for initiative!"
