9.1a — Inheritance Basics: IS-A Relationships

Discovery Lab ~25 min

You've built Monster, Player, Item — each from scratch. But a Goblin and a Dragon share a lot: they both have HP, they both attack, they both take damage. Inheritance lets you write that shared code ONCE in a parent class and reuse it in child classes. The key question: IS-A. A Goblin IS-A Monster. A Dragon IS-A Monster. If the sentence makes sense, inheritance fits.

Experiment: Your First Parent-Child

Create Creature.java (parent) and Goblin.java (child):

// Creature.java — the parent
public class Creature {
    private String name;
    private int hp;
    private int attackPower;

    public Creature(String name, int hp, int attackPower) {
        this.name = name;
        this.hp = hp;
        this.attackPower = attackPower;
    }

    public int attack() {
        return attackPower;
    }

    public void takeDamage(int damage) {
        hp -= damage;
        if (hp < 0) hp = 0;
    }

    public boolean isAlive() { return hp > 0; }
    public String getName() { return name; }
    public int getHp() { return hp; }
}
// Goblin.java — the child
public class Goblin extends Creature {
    public Goblin() {
        super("Goblin", 20, 5);  // calls Creature constructor
    }

    // Goblin gets attack(), takeDamage(), isAlive() for FREE
    // You can add goblin-specific methods here
    public String screech() {
        return "EEEEK!";
    }
}

Create a main that makes a Goblin and calls attack(), takeDamage(8), isAlive(). These methods aren't in Goblin.java — they're inherited. Then call screech(), which IS in Goblin.

Private Fields Aren't Inherited Directly: A child class CANNOT access hp directly even though it inherits the field. You must use the parent's public methods (getHp()) or declare the field protected. Read the Inheritance Basics docs for the public/private/protected visibility rules.

Experiment: Multiple Children

Create two more child classes: Dragon.java and Skeleton.java. Each calls super() with different stats. Give Dragon a breatheFire() method. Give Skeleton a reassemble() method. Create an array of Creature references and store one of each:

Creature[] party = new Creature[3];
party[0] = new Goblin();
party[1] = new Dragon();
party[2] = new Skeleton();

for (Creature c : party) {
    System.out.println(c.getName() + " attacks for " + c.attack());
}

This works because a Goblin IS-A Creature, a Dragon IS-A Creature. But can you call c.breatheFire() inside the loop? Try it.

Socratic Checkpoint

  1. Could you have made Monster the parent of Goblin in Unit 5? What would you have needed to change?
  2. What does super() actually do? When MUST you call it?
  3. Look at the docs section on the Object superclass. Every class you've ever written extends Object. What methods do you get for free?

Checkpoint — before moving on, you must be able to:

  • Write a parent class and extend it with extends
  • Call parent constructor with super()
  • Explain what IS-A means and when inheritance is appropriate
  • Store child objects in a parent-typed array
