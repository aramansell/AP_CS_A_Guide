1.1b — Variables & Types

Type: Discovery Lab. Create a test file called VarTest.java and try everything below. Compile after each change.

Discovery Lab ~25 min

A variable is a named box that holds a value. You create one by saying what type of value it holds, what name you'll use, and what initial value to put in it. 

type name = value;

The three types you need right now:

TypeStoresExampleRPG uses
int Whole numbers 42, -5, 0 HP, score, level, gold
double Numbers with decimals 3.14, 0.75 Percentages, luck, damage multipliers
boolean True or false only true, false isAlive, hasKey, questComplete

Declaration vs. Initialization

int score;           // declaration — box created
score = 100;         // initialization — put a value in

int score = 100;     // both at once (most common)

Assignment (=)

In Java, = does not mean "equals." It means "put this value into this box." Read it right-to-left: take whatever is on the right and store it in the variable on the left. 

int hp = 100;        // put 100 into hp
hp = 75;             // replace with 75 — the old 100 is gone
hp = hp - 10;        // take current hp (75), subtract 10, store 65 back

Naming Rules

  • camelCase — start lowercase, capitalize each new word (playerHealth, goldAmount)
  • No spaces. No special characters (avoid _ and $).
  • Can't start with a number.
  • Can't use Java keywords (int, class, public, etc.).
  • Make names descriptive. hp is fine. x is not.

Bug Hunt: The Strict Compiler

Try these lines in VarTest.java. They won't compile. What rule is Java enforcing?

int x = 3.5;       // can you put a double into an int box?
int y = "hello";   // can you put a String into an int box?

Socratic Checkpoint

Try this line:

double z = 5;

Does this cause an error? Why does Java let you put an int into a double box, but not a double into an int box?

Tinker: Explore on Your Own

  1. Declare int x; and try to print it without assigning. Make a prediction first — then test. What happens?
  2. Create three variables: int hp = 100;, double luck = 0.75;, boolean alive = true;. Print all three on one line using concatenation (+).
  3. Write your own line that puts a double into an int. What error do you get? Can you read it?

Checkpoint — before moving on, you must be able to:

  • Declare and initialize int, double, and boolean variables
  • Explain what = really means in Java
  • Tell someone why int x = 3.5; won't compile
  • Explain the difference between declaration and initialization
