1.7c — Dice Roller: Testing & Integration

RPG — Part 3 ~15 min

Testing Protocol

For each method in Dice.java, prove it works:

  1. roll(sides): Call it 20 times for a d6. Record min and max. Did you ever get 0? Did you ever get 7?
  2. rollDice(count, sides): Call rollDice(3, 6) 20 times. Verify: min >= 3, max <= 18, never outside range.
  3. statRoll(): Generate 6 stats (STR, DEX, CON, INT, WIS, CHA). Print them labeled. Verify each is in the 3–18 range.

Write your test results:

roll(6) test — 20 rolls:
Min: 1  Max: 6  (range correct? ✓/✗)

rollDice(3, 6) test — 20 rolls:
Min: ___  Max: ___  (range 3-18? ✓/✗)

statRoll() — 6 stats:
STR: 12  DEX: 8  CON: 15  INT: 11  WIS: 14  CHA: 9
All in range 3-18? ✓/✗

Socratic Checkpoint

A classmate's roll(20) occasionally returns 0. Where is the bug? Walk through their formula step by step: 

return (int) Math.random() * sides;

What's happening? Why does it always return 0? Can you explain the fix to them in plain English?

Checkpoint — before moving on, you must be able to:

  • Systematically test every method you write — min, max, edge cases
  • Explain why the parentheses bug in the random formula causes 0
  • Generate a full set of RPG stats (3d6 for each of 6 attributes)
  • Use a Dice.java utility class from another file (DiceTester.java)
