6.1a — Array Basics: A Row of Lockers

Discovery Lab ~25 min

So far you've stored one value per variable: int hp = 20. But what if you need to store 100 monster HP values? Or all 52 cards in a deck? You're not writing monster1hp, monster2hp, monster3hp... for 100 monsters.

An array is a single variable that holds multiple values of the same type. Think of it as a row of numbered lockers — each locker holds one value, and you open it by its number (the index). Indexing starts at 0, not 1. This is the single most common source of off-by-one bugs in programming.

Experiment: Your First Array

Create ArrayIntro.java. Run each block separately:

// Declaration + initialization
int[] scores = {95, 87, 72, 100, 63};

// Access by index
System.out.println(scores[0]);   // first
System.out.println(scores[4]);   // last
System.out.println(scores[2]);   // middle

// How many elements?
System.out.println(scores.length);

Answer: what does scores[5] do? Try it. Read the error message carefully — this is ArrayIndexOutOfBoundsException. You'll see it a lot.

Experiment: Create + Fill

// Create empty array of size 5
int[] numbers = new int[5];

// Fill each slot
numbers[0] = 10;
numbers[1] = 20;
numbers[2] = 30;
numbers[3] = 40;
numbers[4] = 50;

// What's in numbers[0] after this?
numbers[0] = numbers[0] + 5;

// Print all values — we'll learn loops next
System.out.println(numbers[0]);
System.out.println(numbers[1]);
System.out.println(numbers[2]);
System.out.println(numbers[3]);
System.out.println(numbers[4]);

What are the default values in a new int[] before you fill it? Create one and print numbers[0] before setting it.

Index Off-By-One: An array of length 5 has indices 0, 1, 2, 3, 4. The last index is length - 1. The most common bug in this unit: writing i <= array.length instead of i < array.length. Read the Array Basics docs Bug Hunt section and find all the broken code samples.

Socratic Checkpoint

  1. Why do arrays start at 0 instead of 1? (Hint: the docs explain this.)
  2. What happens if you try to set numbers[-1] = 5? Make a prediction, then test it.
  3. What's the difference between int[] arr = {1,2,3} and int[] arr = new int[3]? Which one gives you more control?

Checkpoint — before moving on, you must be able to:

  • Declare and initialize an array three different ways
  • Access elements by index and explain why length - 1 is the last valid index
  • Read and understand ArrayIndexOutOfBoundsException
  • Use .length to get array size
