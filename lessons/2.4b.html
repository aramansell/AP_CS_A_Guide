2.4b — Character Creator: Build a Hero from Input

Basic Application ~20 min

Build: Interactive Character Sheet

Yesterday you discovered how Scanner reads user input. Today you'll build a character creator that asks the player questions and builds a hero from their answers.

Create a class called CharacterCreator:

  1. Ask for the character's name (String).
  2. Ask for their class: "Fighter", "Wizard", or "Rogue" (String).
  3. Ask for their level (int).
  4. Based on class, calculate starting stats (hardcode per class — you'll use if statements from the docs):
    • Fighter: HP = 30 + (level * 5), Attack = 10 + level
    • Wizard: HP = 20 + (level * 3), Attack = 8 + level
    • Rogue: HP = 25 + (level * 4), Attack = 9 + level
  5. Print a formatted character sheet with all the details.

Trap: Scanner Order

If you call nextInt() for level right after nextLine() for class, does it work? What if you call nextLine() for name, then nextInt() for level, then nextLine() for something else? Test and see where the leftover-newline bug bites you.
