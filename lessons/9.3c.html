9.3c — Polymorphism: Build a Turn-Based Battle System

Expand Application ~30 min

Now pull everything together: a complete turn-based battle system that operates entirely on Creature references. No Goblin, Dragon, or Skeleton types appear in the battle logic — yet each creature fights with its own unique behavior.

This is the pattern that makes RPG engines extensible. When you add a new creature type six months from now, the battle system works without changing a single line.

Build: BattleSystem.java

The system has three parts:

Part 1: Team Setup

  • Player's party: ArrayList<Creature> with 3 creatures (choose from Goblin, Dragon, Skeleton — give them names and unique stats)
  • Enemy party: ArrayList<Creature> with 3 creatures (different mix)
  • Store both teams in a Battle class that holds the two ArrayLists

Part 2: The Turn Loop

  • Each round: player's alive creatures each attack a random enemy
  • Then: each surviving enemy attacks a random player creature
  • After each attack: check if the target died (HP <= 0) — if so, remove from team
  • Continue until one team has no living creatures

Socratic Checkpoint

  1. Your selectRandomTarget(ArrayList<Creature> enemies) method works with Creature references. How do you ensure you only pick ALIVE targets?
  2. After a creature dies, should you remove it from the ArrayList immediately or mark it dead and skip it? What happens to the ArrayList indices if you remove during iteration?
  3. Each creature's attack() method returns different damage amounts (Goblin 3-8, Dragon 15-30, Skeleton 5-12). How does the battle loop handle this without knowing the type?

Tinker: Special Abilities

Add a specialAbility() method to Creature. Each subclass defines its own:

  • Goblin: Steal gold — reduces enemy attack by 2 next turn
  • Dragon: Fire breath — hits ALL enemies for 10-15 damage
  • Skeleton: Reassemble — if HP is 0, 20% chance to revive with 10 HP

Each creature has a 25% chance to use its special ability each turn instead of a normal attack. The battle loop checks Math.random() < 0.25 and calls specialAbility() — still polymorphically, still without knowing the creature type.

Bug Hunt: Concurrent Modification

If you remove dead creatures from the ArrayList inside a for-each loop, Java throws ConcurrentModificationException. Try it — then research two solutions:

  1. Use a standard for loop running backward (i--) so removal doesn't shift indices you haven't visited yet
  2. Use an Iterator with iterator.remove()

Which approach is cleaner? Which one avoids the bug entirely?

Checkpoint — before moving on, you must be able to:

  • Build a complete turn-based system using only parent-type references
  • Handle creature death and removal from battle safely
  • Add special abilities polymorphically without modifying battle logic
  • Debug ConcurrentModificationException and implement safe removal
