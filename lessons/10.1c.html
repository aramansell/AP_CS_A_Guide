10.1c — Recursive Dungeon Crawler: Explore All Rooms

Expand Application ~30 min

Build: Dungeon Explorer

Dungeons are a natural fit for recursion. A room has doors to other rooms, which have doors to more rooms... recursion explores them all without needing to know the layout in advance.

Create a class called DungeonExplorer with this dungeon represented as a 2D char array:

char[][] dungeon = {
    {'#', '#', '#', '#', '#'},
    {'#', '.', '.', '.', '#'},
    {'#', '.', '.', '.', '#'},
    {'#', '.', '.', '.', '#'},
    {'#', '#', '#', '#', '#'}
};
  1. void explore(int row, int col, boolean[][] visited): mark the current tile visited. Then recursively explore all 4 adjacent tiles that are floor ('.') and not yet visited.
  2. int countRooms(): start from any floor tile, call explore, and return the number of tiles visited.
  3. Add a treasure room: change one tile to 'T'. Your explore method should print a message when it finds treasure.

Base Case Check

What are ALL the base cases in your explore method? List them. Missing even one causes infinite recursion or array bounds errors.
