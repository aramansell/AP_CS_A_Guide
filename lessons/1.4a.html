1.4a — Slot Machine (Build Together)

Type: Basic Application. The teacher builds, you follow along. This is a "most basic application" — the simplest program that uses everything you've learned: variables, Math.random(), printing, and the three-step pattern.

Basic Application ~35 min

The Goal

Simulate a three-reel slot machine. Generate three random symbols, display them, and announce a winner if all three match.

=== SLOT MACHINE ===
Spinning...
[ Cherry | Lemon | Cherry ]
No match — try again!

Iteration 1: The Broken Reels

Here's the starting code. Copy this into SlotMachine.java and run it.

public class SlotMachine {
    public static void main(String[] args) {
        // Step 1: Declare
        int reel1;
        int reel2;
        int reel3;

        // Step 2: Generate
        reel1 = (int) Math.random() * 4;
        reel2 = (int) Math.random() * 4;
        reel3 = (int) Math.random() * 4;

        // Step 3: Display (numbers only for now)
        System.out.println("=== SLOT MACHINE ===");
        System.out.println("[ " + reel1 + " | " + reel2 + " | " + reel3 + " ]");
    }
}

Bug Hunt 1

Run the code 5 times. What is the output every single time? Why?

Hint: Look closely at the casting on the generation lines. This is the EXACT same trap from the Math.random() Tinker Challenge 3.

Fix the bug before moving on.

Iteration 2: Adding the Symbols

Once the reels generate random numbers from 0 to 3, we need to map them to Strings. Here's the logic for the first reel:

String symbol1;
if (reel1 == 0) symbol1 = "Cherry";
else if (reel1 == 1) symbol1 = "Lemon";
else if (reel1 == 2) symbol1 = "7";
else symbol1 = "Bell";

Tinker Challenge

Repeat this logic for reel2 → symbol2 and reel3 → symbol3. Update your print statement to use the Strings instead of the integers.

Iteration 3: The Win Condition

Here's the win-check code — with a logic error.

if (reel1 == reel2) {
    System.out.println("JACKPOT! All three match!");
} else {
    System.out.println("No match — try again!");
}

Bug Hunt 2

What's wrong with the win condition above? Think: reel1 is Cherry, reel2 is Cherry, and reel3 is Lemon. What does this code print?

Fix the if statement so ALL THREE must match.

Extend It (pick at least one)

  • Change the random generation to * 6 and add two more symbols ("Diamond" and "Star").
  • Start the player with 100 coins. If they win, add 50 coins. Print their final balance.
  • Give two-of-a-kind a "small payout" message.

Socratic Checkpoint

  2. Why do we declare variables at the top instead of creating them right before we use them?
  3. How is the random formula the same as what you discovered in 1.3? How is it different for a slot machine (which uses symbols instead of numbers)?
