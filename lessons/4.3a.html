4.3a — Loops + Strings: Character by Character

Discovery Lab ~25 min

A String is a sequence of characters. You know charAt() and length() from Unit 2. Now combine those with loops to visit every character — searching, counting, building new strings.

This is called string traversal: walking through a string one character at a time, doing something at each position.

The Standard Pattern

String word = "dragon";
for (int i = 0; i < word.length(); i++) {
    char c = word.charAt(i);
    System.out.println("Position " + i + ": " + c);
}

Experiment: charAt Boundaries

Create StringWalker.java. Run these experiments:

String s = "sword";
System.out.println(s.length());        // what prints?
System.out.println(s.charAt(0));       // first character?
System.out.println(s.charAt(4));       // last character?
// System.out.println(s.charAt(5));    // UNCOMMENT — what happens?

Open the Loops + Strings docs. Find the "Out of Bounds" Bug Hunt. Read the broken code. Why does weapon.charAt(weapon.length()) crash? What's the fix?

Build: Count and Search

Pick a spell name: "Fireball", "IceStorm", "Thunderbolt", or your own. Write a traversal loop that counts:

  1. How many vowels (a, e, i, o, u) are in the spell name?
  2. How many total characters?
  3. Does the spell name contain the letter 'z'?

Run on multiple spell names. Does your vowel counter handle uppercase? If the user types "FIREBALL", does it still count correctly?

Build: Reverse It

The docs show a reverse-string example. Read it. Then write your own version without looking — write it from memory.

Start with an empty String (not null — ""). Loop through the original string BACKWARDS. Use += to build the reversed version. Test: "sword" → "drows", "java" → "avaj".

Socratic Checkpoint

  1. Read "The Memory Cost" Socratic Checkpoint in the docs. If you use += inside a loop to build a 1000-character string, how many temporary String objects does Java create? Why is this wasteful?
  2. The docs show a "Combat Command Parser" that splits "attack goblin" into verb and target. Read it. Why does it search for a space? What happens if the player types a command with NO space?
  3. Look at the Character.isDigit() and isLetter() methods in the docs. Write a loop that prints "digit", "letter", or "other" for each character in "F1r3bAll!".

Checkpoint — before moving on, you must be able to:

  • Traverse a string with a for loop and charAt()
  • Count characters matching a condition (vowels, digits, etc.)
  • Build a new string by accumulating characters in a loop
  • Avoid off-by-one errors with length() and charAt()
