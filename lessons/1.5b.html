1.5b — Tip Calculator: Run, Test & Trace

Basic Application ~20 min

Once your Tip Calculator works, prove it's correct:

  1. Test with: bill $45.50, tip 20%. Expected tip: $9.10, total: $54.60. Does your output match?
  2. Edge case: bill $100.00, tip 0%. Does it work correctly? What should the tip and total be?
  3. Paper trace: Pick a bill amount and tip percentage. Walk through every line of code on paper. Write down the value of each variable after each step. This skill — tracing code — is the single best way to find bugs.

Bug Hunt: Remove 100.0

What happens if you remove the 100.0 and just write:

tip = billAmount * (tipPercent / 100);

Test it. Explain the result to a partner. Why does the .0 matter so much?

Socratic Checkpoint: Compare & Contrast

You've now built two programs: Slot Machine and Tip Calculator.

  1. List everything that's the SAME between them.
  2. List everything that's DIFFERENT.
  3. If I asked you to build a "Restaurant Bill Splitter" from scratch tomorrow — where would your three comment blocks go?

Checkpoint — before moving on, you must be able to:

  • Test a program with specific inputs and verify expected outputs
  • Trace code on paper — variable values after each line
  • Explain why tipPercent / 100 gives 0, but tipPercent / 100.0 works correctly
  • Write the three-step skeleton on a blank page without looking
