9.6c — Monster Spawner & Full Integration

RPG Build ~30 min

Part 1: MonsterSpawner.java

Your BattleEngine handles any Combatant. Now build a factory that creates them. A MonsterSpawner generates random enemies based on difficulty — this is the Factory pattern in action.

Create MonsterSpawner.java:

  • Constructor takes int difficulty (1-3)
  • spawn() returns a random Combatant:
    • Difficulty 1: Goblin or Skeleton (equal chance)
    • Difficulty 2: Goblin, Skeleton, or weakened Dragon (HP 40, ATK 12)
    • Difficulty 3: Goblin, Skeleton, or full Dragon
  • getBoss() returns a Dragon with +50% HP and ATK
  • spawnWave(int count) returns ArrayList of count random combatants

Socratic Checkpoint

  1. spawn() returns type Combatant, not Goblin or Dragon. Why? What happens if spawn() returned Dragon and a weaker monster needed to be returned?
  2. getBoss() creates a Dragon with modified stats. Is this a new class or the same Dragon with different constructor arguments? Which is simpler?

Part 2: Rewire DungeonCrawl

Replace your hard-coded combat with the spawner + BattleEngine:

// BEFORE
if (monsterName.equals("Goblin")) { ... }
else if (monsterName.equals("Dragon")) { ... }

// AFTER
MonsterSpawner spawner = new MonsterSpawner(difficulty);
Combatant enemy = spawner.spawn();
Combatant winner = BattleEngine.runBattle(player, enemy);

Add a dungeon run: 5 random monsters, then the boss. Player rests (heal 10) between fights. Track wins. After all 6 fights: victory or death screen.

Tinker: New Creatures

Add two new creature types WITHOUT changing BattleEngine:

  • Slime — 15 HP, 3 ATK. When it takes damage, it splits: if HP would drop below 0, set HP to 1 instead (survives with 1 HP once per fight).
  • Healer — 25 HP, 4 ATK. Every 3rd round, instead of attacking, it heals itself for 8 HP. You'll need a round counter.

Add them to MonsterSpawner. Does BattleEngine need any changes? (It shouldn't — that's the whole point.)

Socratic Checkpoint

  1. Look at the docs section on extensibility. What's the difference between adding a new creature (easy) and adding a new combat mechanic like "flee from battle" (hard)?
  2. Your DungeonCrawl now has Player, DynamicInventory, DungeonMap, MonsterSpawner, and BattleEngine. Draw the dependency diagram: which class depends on which?

Checkpoint — before moving on, you must be able to:

  • Build a factory class that returns interface types
  • Wire polymorphic combat into the game loop
  • Add new creatures without modifying the battle engine
  • Explain the Factory pattern and why it decouples code
