10.4a — Flood Fill: Reveal the Dungeon

RPG Build ~30 min

Your DungeonMap from Unit 8 is a grid, but the player can only see one tile at a time. What if they find a magic item that reveals all connected floor tiles? This is the classic flood fill algorithm — explore a tile, then recursively explore all its walkable neighbors. The same algorithm that powers the paint bucket tool in every drawing program.

Open the Recursive Dungeon (Student RPG Guide). Follow Part 1: Flood Fill.

Add reveal() to DungeonMap

// Starting from (row, col), mark all connected walkable tiles as visited
public int reveal(int row, int col, boolean[][] visited) {
    // Base case: out of bounds — stop
    if (row < 0 || row >= tiles.length || col < 0 || col >= tiles[0].length)
        return 0;
    // Base case: wall or already visited — stop
    if (tiles[row][col] == '#' || visited[row][col])
        return 0;

    visited[row][col] = true;  // mark current
    int count = 1;             // count this tile

    // Recurse in all 4 directions
    count += reveal(row - 1, col, visited);  // up
    count += reveal(row + 1, col, visited);  // down
    count += reveal(row, col - 1, visited);  // left
    count += reveal(row, col + 1, visited);  // right

    return count;  // total connected tiles found
}

Call it from the player's position. It returns how many tiles are reachable from where they stand — useful for checking if the exit is reachable.

Visualize the Reveal

Add a displayRevealed(boolean[][] visited) method:

  • Loop through the grid
  • If visited[row][col] is true, print the actual tile
  • If false, print '?' (fog of war)
  • Always print walls as '#' (they're visible but impassable)

Call displayRevealed() after reveal(). The player sees connected rooms but not disconnected areas. That's real fog of war.

Forgetting the Visited Check = Infinite Recursion: Without visited[row][col], reveal(0,0) calls reveal(1,0) which calls reveal(0,0) which calls reveal(1,0)... forever. The visited array breaks cycles. Also: check boundaries BEFORE indexing tiles[row][col]. Read the Recursive Dungeon docs for the complete algorithm.

Socratic Checkpoint

  1. Why does the visited array parameter start as new boolean[rows][cols]? What would happen if you created a new array inside reveal() on every recursive call?
  2. Flood fill visits tiles in depth-first order (explores a full path before trying the next direction). What would breadth-first look like? Which is better for revealing a dungeon?
  3. The return value counts connected tiles. How would you use this to verify that the exit tile 'E' is reachable from the player?

Acceptance — your flood fill must:

  • Stop at walls and out-of-bounds
  • Never visit the same tile twice
  • Return the correct count of connected tiles
  • Not overflow the stack on a 15×15 map
