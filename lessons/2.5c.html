2.5c — Math Class: Damage Formulas & Distance

Expand Application ~30 min

RPGs run on math formulas behind the scenes. Damage isn't just a flat number — it's modified by stats, randomness, distance, and defense. Today you implement two core RPG formulas and combine them.

Build 1: Damage Calculation Engine

Create DamageCalc.java. The formula for physical damage is:

rawDamage = weaponDamage + (strength * 0.5)
// Add randomness: rawDamage * random factor between 0.85 and 1.15
// Subtract defense: final = rawDamage - (defense * 0.3)
// Minimum: 1 damage always

The formulas are in the Math docs. You need to figure out which Math methods to use for each step — abs, min, max, pow, sqrt, or random.

Socratic Checkpoint

  1. The random factor needs to be between 0.85 and 1.15. Math.random() gives you 0.0 to 0.999.... How do you transform that range into 0.85 to 1.15? (Hint: multiply, then add.)
  2. After subtracting defense, damage could go negative. Which Math method prevents that?
  3. Why multiply defense by 0.3 instead of subtracting the full defense value? What would happen if defense fully negated damage every time?

Build 2: Distance Calculator

Create DistanceCalc.java. In grid-based RPGs, you often need to know how far apart two things are. The formula comes from geometry:

distance = sqrt((x1 - x2)² + (y1 - y2)²)

Implement this using Math methods. Test with these positions:

Player (0, 0), Enemy (3, 4) → distance: 5.0
Player (10, 5), Enemy (13, 9) → distance: 5.0
Player (0, 0), Enemy (1, 1) → distance: ~1.414

Tinker: Damage Falls Off With Distance

Combine both programs. Modify the damage formula so damage is reduced when the enemy is far away: 

distanceModifier = max(0.3, 1.0 - (distance / 20.0))
finalDamage = calculatedDamage * distanceModifier

Test at various distances: point-blank (distance=1), mid-range (distance=5), far (distance=15), extreme (distance=30). Does the modifier make sense? Is 0.3 a reasonable minimum, or should it be lower?

Bug Hunt: Negative Squares?

When calculating distance, you square (x1 - x2). What if x1 is smaller than x2 — does squaring a negative number work? Yes — but why? What would happen if you used Math.abs(x1 - x2) instead of squaring? Would the distance formula still be correct?

Checkpoint — before moving on, you must be able to:

  • Implement a damage formula with modifiers and randomness
  • Calculate distance between two points using sqrt and pow
  • Combine multiple Math methods to create game mechanics
  • Understand why squaring handles negative differences automatically
