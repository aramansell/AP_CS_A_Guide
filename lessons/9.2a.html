9.2a — Method Overriding: Same Name, Different Behavior

Discovery Lab ~25 min

Inheritance gives you the parent's methods for free. But sometimes you want a child to do the same thing DIFFERENTLY. A Dragon shouldn't just "attack" — it should breathe fire. Overriding lets you replace a parent method with a child-specific version. The magic: when you call attack() on a Creature reference that actually points to a Dragon, the DRAGON's version runs.

Experiment: Override Attack

Add to Creature, Goblin, and Dragon from yesterday:

// In Creature.java:
public int attack() {
    System.out.println(name + " attacks!");
    return attackPower;
}

// In Goblin.java — override:
@Override
public int attack() {
    System.out.println(getName() + " swings a rusty dagger!");
    return super.attack() + 2;  // call parent's version, then add bonus
}

// In Dragon.java — override:
@Override
public int attack() {
    System.out.println(getName() + " breathes fire!");
    return attackPower * 2;  // double damage
}

Now run this:

Creature[] party = {new Goblin(), new Dragon(), new Goblin()};
for (Creature c : party) {
    int dmg = c.attack();  // WHICH attack() runs?
    System.out.println("Dealt " + dmg + " damage\n");
}

The variable type is Creature, but the method that runs depends on the ACTUAL object type. This is polymorphism — the single most powerful concept in OOP.

@Override Is Not Optional (For You): Always use @Override. If you misspell the method name or get the parameters wrong, the compiler will catch it and say "method does not override anything." Without @Override, you just wrote a NEW method with a typo and your override silently fails. Read the Polymorphism docs for the full rules.

Build: toString() Override

Override toString() in Creature and each child:

// In Creature:
@Override
public String toString() {
    return name + " [HP:" + hp + " ATK:" + attackPower + "]";
}

// In Dragon:
@Override
public String toString() {
    return super.toString() + " ✦ DRAGON ✦";  // adds dragon flair
}

Create a mixed Creature array, loop, and System.out.println(c). Java automatically calls toString() — and the RIGHT version for each object.

Socratic Checkpoint

  1. In the loop for (Creature c : party), the variable is type Creature. Why does calling c.attack() run the Dragon's version when c points to a Dragon? How does Java know?
  2. What's the difference between super.attack() and this.attack()? When would calling super.attack() cause infinite recursion?
  3. Read the docs section on the override rules. Can you change the return type? Can you make it more restrictive (public to private)?

Checkpoint — before moving on, you must be able to:

  • Override a parent method with @Override
  • Call the parent version with super.methodName()
  • Explain why the ACTUAL object type determines which override runs
  • Override toString() to customize object printing
