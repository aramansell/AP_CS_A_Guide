5.3c — Damage with Validation: HP Can't Go Negative

Expand Application ~30 min

Build: Combatant Class

Every RPG character has HP that can go down... but never below zero. And healing shouldn't exceed max HP. Mutators enforce these rules so no other code can mess it up.

Create a class called Combatant:

  1. Private instance variables: String name, int hp, int maxHp, boolean isAlive
  2. Constructor takes name and maxHp. Sets hp = maxHp, isAlive = true.
  3. void takeDamage(int amount): subtract from hp. If hp <= 0, set hp = 0 and isAlive = false. Print a death message.
  4. void heal(int amount): add to hp. If hp > maxHp, set hp = maxHp. Print "Already at full health!" if hp was already max.
  5. Getter: getHp() — but no public setter for hp. Why? Because damage/healing should always go through validation.
  6. boolean isAlive() — getter for alive status.

Test Your Guards

Create a Combatant with 20 HP. Take 25 damage — should be at 0 HP and dead. Try to heal — does it work on a dead character? (You decide: should it?) Take 10 damage to a dead character — what happens?
