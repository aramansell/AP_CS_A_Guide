1.4b — Slot Machine: Run & Reflect

Basic Application ~15 min

Once your Slot Machine compiles and runs correctly:

  1. Probability test: Run it 10 times. How often do you get three matching reels?
  2. Do the math: With 4 symbols per reel, what percentage of spins should be winners (all three match)? Calculate the probability, then run it 50 times. Does reality match the math?
  3. Compare outputs: Look at someone else's output. Does theirs look exactly like yours? Why or why not?
  4. Add a fourth reel: Does the code change much? What does this tell you about the pattern? What if there were 10 reels — what would be tedious about our current approach?

Tinker: Winner Percentage

With 4 symbols and 3 reels, the probability of all three matching is: (1/4) × (1/4) = 1/16 ≈ 6.25%. Run your machine 50 times and count the wins. How close did you get? Why might your result differ from the exact probability?

Checkpoint — before moving on, you must be able to:

  • Trace every line of the Slot Machine and explain what it does
  • Identify the three-step pattern within the code
  • Explain how the casting-bug from Iteration 1 relates to the Math.random() parentheses trap
  • Calculate simple probability for random events
