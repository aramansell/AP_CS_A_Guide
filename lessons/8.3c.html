8.3c — Connect Four

Type: Expand Application. Tic-Tac-Toe was a warm-up. Now build a bigger board (6×7), gravity-based piece placement, and four-in-a-row win detection in every direction. This is a real game.

Expand Application ~35 min

The Goal

Build Connect Four. Pieces drop to the lowest empty row in the chosen column. Win detection checks horizontal, vertical, and both diagonal directions for four in a row.

Build It: Gravity First

Create ConnectFour.java. Use a char[][] board of 6 rows × 7 columns. The key difference from Tic-Tac-Toe: the player only picks a COLUMN. The piece falls to the lowest empty row in that column.

// Drop a piece in the given column for the given player
// Returns the row it landed in, or -1 if column is full
public static int dropPiece(char[][] board, int col, char player) {
    // Start from the BOTTOM row and go UP
    for (int row = board.length - 1; row >= 0; row--) {
        if (board[row][col] == ' ') {
            board[row][col] = player;
            return row;
        }
    }
    return -1;  // column full
}

Socratic Checkpoint

  1. Why does the loop start at board.length - 1 and go down to 0? What would happen if it went the other way?
  2. How does return row inside the loop end the method immediately? Why is that useful here?

Bug Hunt

The obvious approach is 4 separate loops, one per direction. But there's a smarter way: use direction arrays. Two arrays define 4 directions as (rowDelta, colDelta) pairs: right (0,1), down (1,0), down-right (1,1), down-left (1,-1). One loop nests inside another. Can you figure out the pattern?
