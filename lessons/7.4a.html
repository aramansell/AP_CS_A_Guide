7.4a — Dynamic Inventory: ArrayList-Powered RPG Backpack

RPG Build ~60 min

Your FixedInventory from Unit 6 works, but it has a hard limit. If the player finds a Legendary Sword in room 12 but their 20-slot backpack is full, they can't pick it up. That's a terrible RPG experience. It's time to replace the fixed array with an ArrayList that grows forever.

Open the Dynamic Inventory (Student RPG Guide). The specification is nearly identical to FixedInventory — same method names, same Item objects — but the implementation is dramatically simpler.

Build: DynamicInventory.java

Create DynamicInventory.java using ArrayList<Item>:

  • void add(Item item) — one line: items.add(item). No capacity check.
  • boolean remove(String name) — find by name, call items.remove(index). Auto-shift.
  • boolean contains(String name) — same logic as before, but cleaner
  • int getTotalValue() — sum traversal
  • void display() — numbered list with rarity and value
  • void sortByValue() — selection sort (from 7.3a)

Count the lines of code. Compare to FixedInventory. Where did all the array management code go?

Integrate Into Your RPG

Swap FixedInventory for DynamicInventory in your DungeonCrawl game loop. The method signatures are the same — just change the constructor call:

// Before:
FixedInventory inventory = new FixedInventory(20);

// After:
DynamicInventory inventory = new DynamicInventory();

Add a sort option: after combat, let the player type "sort" to see their inventory sorted by value. Add "filter rare" to show only rare+ items.

Socratic Checkpoint

  1. What happens when you call items.remove("Potion") and the player has three Potions? Which one gets removed?
  2. ArrayList is built on top of a plain array. Where does the capacity logic go? How does it know when to grow?
  3. If you add 1,000,000 items to your DynamicInventory, does it slow down? What operation gets slower — add, remove, or contains?

Checkpoint — before moving on, you must be able to:

  • Replace array-based collection with ArrayList equivalent
  • Integrate dynamic inventory into RPG game loop
  • Add sort and filter operations powered by ArrayList methods
