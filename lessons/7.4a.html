7.4a — DynamicInventory: ArrayList-Powered Backpack

RPG Build ~30 min

Your FixedInventory from Unit 6 works, but it has a hard limit. If the player finds a Legendary Sword in room 12 but their 20-slot backpack is full, they can't pick it up. That's a terrible RPG experience. It's time to replace the fixed array with an ArrayList that grows forever.

Open the Dynamic Inventory (Student RPG Guide). The specification is nearly identical to FixedInventory — same method names, same Item objects — but the implementation is dramatically simpler.

Build: DynamicInventory.java

Create DynamicInventory.java using ArrayList<Item>:

  • void add(Item item) — one line: items.add(item). No capacity limit.
  • boolean remove(String name) — find by name, call items.remove(index). ArrayList handles the shift.
  • boolean contains(String name) — same logic as FixedInventory
  • int getTotalValue() — sum traversal with enhanced for-loop
  • void display() — numbered list with rarity and value

Count the lines of code. Compare to FixedInventory. Where did all the array management code go?

Socratic Checkpoint

  1. ArrayList is built on top of a plain array. Where does the capacity logic go? How does it know when to grow?
  2. Your remove() in FixedInventory manually shifted elements. ArrayList's remove() does it for you. But someone still wrote that shift code — who, and where is it?
  3. What happens when you call items.remove("Potion") and the player has three Potions? Which one gets removed?

Checkpoint — your DynamicInventory must:

  • Accept unlimited items (no capacity parameter in constructor)
  • Match FixedInventory method signatures exactly
  • Pass all acceptance tests in the Student RPG Guide
