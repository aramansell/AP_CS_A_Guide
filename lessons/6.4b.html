6.4b — High Score Table (Build Together)

Type: Basic Application. The teacher builds, you follow along. Your first parallel array pattern — two arrays, one index, synchronized data. Plus: finding the max, printing a sorted view.

Basic Application ~30 min

The Goal

Store player names and their scores in parallel arrays. Find the highest score, print the leaderboard, and compute the average.

=== HIGH SCORES ===
1. Alice      - 950
2. Bob        - 720
3. Charlie    - 610
4. Diana      - 880
5. Eve        - 540
---
HIGH SCORE: Alice with 950
AVERAGE: 740.0

Reference: High Score Table Walkthrough — full reference.

Iteration 1: Two Arrays, One Index

Create HighScoreTable.java:

public class HighScoreTable {
    public static void main(String[] args) {
        String[] names = {"Alice", "Bob", "Charlie", "Diana", "Eve"};
        int[] scores = {950, 720, 610, 880, 540};

        // Print them as a table
        for (int i = 0; i < names.length; i++) {
            System.out.println((i+1) + ". " + names[i] + " - " + scores[i]);
        }
    }
}

Socratic Checkpoint

  1. Why are these called "parallel arrays"? What makes them parallel rather than independent?
  2. What happens if names has 5 elements but scores only has 3? What error would you get and on which line?

Iteration 2: Find the Max (Broken!)

Add this code to find the highest score. It's broken:

int maxScore = 0;
String topPlayer = "";

for (int i = 0; i < scores.length; i++) {
    if (scores[i] > maxScore) {
        maxScore = scores[i];
        topPlayer = names[i];
    }
}
System.out.println("HIGH SCORE: " + topPlayer + " with " + maxScore);

Bug Hunt

Run it. It works. But what if ALL scores are negative? (Imagine a game where scores can go below zero.) The max would be reported as 0 with an empty name. Why? What should maxScore be initialized to instead?

Iteration 3: Compute the Average

Add a loop that sums all scores, then divides by the length.

Tinker Challenge — Sort by Score

Print the table sorted by score, highest first. The algorithm: find the highest remaining score, print it, mark it as "used," repeat. You'll need a boolean[] used parallel array.

Don't look up the full solution. Start with pseudocode on paper. What does "find the highest remaining" look like as a loop?

Socratic Checkpoint

  1. How is the parallel array pattern similar to what your RPG inventory system will need? (Think: item names and item counts.)
  2. Why do sorting algorithms need to "mark as used" rather than just moving elements?
