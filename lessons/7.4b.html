7.4b — Sort, Filter & Wire It In

RPG Build ~30 min

Part 1: Add Sort and Filter

Your DynamicInventory stores items — now add the operations that make an RPG inventory useful. Sorting and filtering are traversal problems.

Add to DynamicInventory.java:

  • void sortByValue() — selection sort by goldValue (ascending). Use the same algorithm from 7.3a. ArrayList lets you use .get(index) and .set(index, item) instead of [index].
  • ArrayList<Item> filterByRarity(String rarity) — returns a NEW ArrayList containing only items matching the given rarity. Use an enhanced for-loop.
  • ArrayList<Item> filterByMinValue(int minValue) — returns items worth at least minValue.

Socratic Checkpoint

  1. Your filter methods return a NEW ArrayList. Why not modify the original? What would break if filter removed items instead of returning a new list?
  2. Selection sort on ArrayList uses .set(index, item) to swap. How is this different from array[] = syntax?

Part 2: Swap and Extend

Open DungeonCrawl. Replace FixedInventory with DynamicInventory:

// Before:
FixedInventory inventory = new FixedInventory(20);

// After:
DynamicInventory inventory = new DynamicInventory();

The method signatures match. Nothing else in your game loop should change.

Add commands to your game loop:

  • "sort" — calls inventory.sortByValue(), then displays
  • "filter epic" — calls inventory.filterByRarity("Epic"), displays only epic items

Socratic Checkpoint

  1. If you add 1,000 items to your DynamicInventory, which operations get slower — add(), remove(), or contains()? Why?
  2. You swapped FixedInventory for DynamicInventory and nothing broke. That's because both have the same public methods. What is this design principle called? (The docs have the answer.)

Checkpoint — before moving on, you must be able to:

  • Implement selection sort on ArrayList objects
  • Write filter methods that return new collections
  • Swap one inventory implementation for another without breaking game logic
