2.6c — Wrapper Classes: Build a Stat Serializer

Expand Application ~30 min

RPGs need to save and load character data. You can't write int directly to a file — you convert numbers to Strings first. Wrapper classes handle both directions: int → String for saving and String → int for loading.

Serialization is the process of converting data into a format that can be stored or transmitted. Deserialization is converting it back. Wrapper classes are Java's simplest serialization tools.

Build 1: StatSerializer — Save and Load

Create StatSerializer.java with two main sections:

  • Save: Convert character stats to a formatted String
  • Load: Parse a formatted String back into stats
// SAVE
int hp = 85;  double luck = 0.625;  boolean alive = true;
String saved = hp + "|" + luck + "|" + alive;
System.out.println("Saved: " + saved);
// Output: Saved: 85|0.625|true

// LOAD
String loaded = "85|0.625|true";
// Split on "|", parse each piece back:
// Piece 0 → Integer.parseInt → int
// Piece 1 → Double.parseDouble → double
// Piece 2 → Boolean.parseBoolean → boolean
System.out.println("hp=" + hp + " luck=" + luck + " alive=" + alive);

Socratic Checkpoint

  1. You used "|" as a separator. What would happen if a stat value contained a pipe character? (It can't — but what if you stored a String field instead?)
  2. Why use Double.parseDouble() instead of Integer.parseInt() for luck? What would Integer.parseInt("0.625") do?
  3. You used String + to concatenate. What's another way to build this string? Check the String docs for alternatives.

Build 2: Multi-Stat Serializer

Now handle more complex data: multiple stats, min/max validation on load.

// Save format: name,str,dex,int,gold,alive
"Thorn|14|18|10|25.5|true"

// Load: verify each stat is in valid range
// str 1-20, dex 1-20, int 1-20, gold >= 0
// If any stat is out of range, clamp it and print a warning

Tinker: What If It Goes Wrong?

Build a safeParseInt(String s, int defaultValue) method:

  • Try to parse the string with Integer.parseInt()
  • If it throws NumberFormatException, return the defaultValue instead
  • But wait — you haven't learned try-catch yet. Is there another way?

Without try-catch, you'd need to check every character manually. With it, you just catch the exception. Look at the Wrapper docs — exceptions section for the syntax. This is your first encounter with exception handling — notice how much simpler it makes the code.

Bug Hunt: The Reversed Delimiter

A teammate changed the save format from "|" to ":". Your load code still splits on "|". What happens when you try to parse "85:0.625" using split("|")? How many pieces do you get? Why?

Checkpoint — before moving on, you must be able to:

  • Serialize multiple stats into a delimited String
  • Parse a delimited String back into typed values with wrapper classes
  • Validate parsed values against acceptable ranges
  • Recognize when to use try-catch vs manual character checking
