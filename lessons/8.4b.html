8.4b — Rooms, Corridors, and Entity Placement

RPG Build ~25 min

Your grid exists. Now carve rooms, connect them with corridors, and place game entities. This is where 2D array traversal patterns start solving real problems — rectangular fill, straight-line path drawing, and counting tiles.

Open the Dungeon Map Builder (Student RPG Guide). Follow Parts 2 and 3.

Part 1: Room Placement

Add to DungeonMap:

  • addRoom(int startRow, int startCol, int height, int width) — carves a rectangular room: walls '#' border, floor '.' inside. Must stay within map bounds (don't overwrite the outer border).
  • addDoor(int row, int col) — sets tile to '+' (door). Must be on an existing wall tile.
  • addCorridor(int row1, int col1, int row2, int col2) — draws a straight-line '.' path between two points. Keep it simple: horizontal then vertical.
Watch the outer border: Your addRoom() should never change row 0, row rows-1, col 0, or col cols-1. Those are the dungeon perimeter walls. If a room tries to extend outside bounds, reject it.

Part 2: Entity Placement

Add entity markers:

  • placePlayer(int row, int col) — sets tile to 'P'. Only one allowed.
  • placeMonster(int row, int col) — sets tile to 'M'
  • placeTreasure(int row, int col) — sets tile to 'T'
  • count(char symbol) — returns how many tiles match the given symbol. Traverse the entire grid.

Socratic Checkpoint

  1. addRoom() fills a rectangular area with nested for loops. Which loop controls rows? Which controls columns? What's the right order?
  2. Your addCorridor draws horizontal-then-vertical. What would happen if you drew vertical-then-horizontal instead? Does it matter?
  3. placePlayer() only allows one 'P'. How do you enforce that? What if the previous 'P' position matters?

Acceptance — your DungeonMap must:

  • Place rooms without destroying outer walls
  • Draw corridors as connected floor paths
  • Place and count entities correctly
