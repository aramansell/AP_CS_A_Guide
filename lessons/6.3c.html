6.3c — Combat Initiative Tracker: Sort and Shift

Expand Application ~30 min

Build: Turn Order System

In tactical RPGs, combatants act in turn order based on their speed stat. When someone's turn ends, they move to the back of the queue. You'll build this using array shift operations.

Create a class called TurnOrder:

  1. Declare arrays: String[] combatants and int[] speeds — parallel arrays of size 6.
  2. Initialize with 4 combatants: "Hero"(12), "Goblin"(8), "Mage"(15), "Orc"(6).
  3. Method void sortBySpeed(): sort combatants and speeds together by speed descending (highest first).
  4. Method String takeTurn(): return the name of the first combatant, then rotate the array — move first element to the end, shift everything else left.

Test Multiple Rounds

Call takeTurn() 8 times in a loop and print who acts. Does the order cycle correctly? After a full cycle, is the order the same as the start?
