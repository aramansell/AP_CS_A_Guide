1.1a — Hello World & Compile

Discovery Lab ~15 min

Java is a compiled language. You write human-readable .java files, then a program called javac translates them into .class files that the computer can execute.

Golden Rule: Every time you change your .java file, you MUST recompile before running.
If you edit code but skip javac, you're running old code. This is the #1 mistake new Java programmers make.

Step 1: Create Hello.java

public class Hello {
    public static void main(String[] args) {
        System.out.println("Hello, adventurer!");
    }
}

Every Java program needs this skeleton. The filename must match the class name. Hello.java → class Hello.

Step 2: Compile and run

$ javac Hello.java      (no output = success)
$ java Hello             Hello, adventurer!

Note: javac uses the full filename with .java. java uses just the class name — no extension.

Bug Hunt: The Compiler's Complaints

Try intentionally making these mistakes one at a time. After each one, fix it, then make the next. Read every error message — can you find the line number it points to?

  1. Rename the class to hello (lowercase). Compile. What error?
  2. Fix it. Compile successfully. Now run java Hello.java instead of java Hello. What happens?
  3. Delete the semicolon at the end of the println line. Compile. What error?

Terminal commands you need:

CommandWhat it does
pwdPrint Working Directory — shows where you are
lsList files in the current directory
cd folderChange Directory — move into a folder
cd ..Go up one directory

Checkpoint — before moving on, you must be able to:

  • Write the full boilerplate from memory (class + main method)
  • Compile with javac and run with java
  • Read a compiler error and find the line number it references
  • Navigate directories with pwd, ls, and cd
