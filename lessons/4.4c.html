4.4c — ASCII Dungeon Map: Nested Loops Meet RPG

Expand Application ~30 min

Build: Dungeon Map Renderer

Roguelike games render dungeons using ASCII characters: # for walls, . for floors, @ for the player. You'll use nested loops to render a dungeon map from a 2D array of characters.

Create a class called DungeonRenderer:

  1. Declare a 2D char array (hardcoded map): 
    char[][] map = {
    {'#', '#', '#', '#', '#'},
    {'#', '.', '.', '.', '#'},
    {'#', '.', '@', '.', '#'},
    {'#', '.', '.', '.', '#'},
    {'#', '#', '#', '#', '#'}
};
  2. Use nested for loops to print the map row by row.
  3. Add a border around the map using * characters.

Extend: Fog of War

The player shouldn't see the whole map. Add a int visionRange = 1. In your nested loop, check the distance from the player (@ position) to each tile. If it's within vision range, print the tile; otherwise print ?. You'll need to know where @ is — use variables playerRow and playerCol.
