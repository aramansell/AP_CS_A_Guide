4.5a — Tracing Loops: How Many Times?

Discovery Lab ~25 min

You can run code and see what happens. But on the AP exam, you can't run anything — you have to trace it on paper. This is the skill that separates guessing from knowing.

A trace table is a grid where each row is one loop iteration and each column is a variable. You write down what every variable equals at every step. No computer. No compiler. Just you and the logic.

Trace Table Format

int sum = 0;
for (int i = 1; i <= 4; i++) {
    sum += i;
}

Practice: Trace on Paper

Open the Tracing Loops docs. There's a while-loop trace table (HP, damage, hits). Study it. Now, WITHOUT looking at the docs, trace this loop on paper: 

int hp = 15;
int hits = 0;
while (hp > 0) {
    hp -= 4;
    hits++;
}

What's the final value of hits? What's the final value of hp? How many iterations ran? Write your trace table, then run the code to verify.

Practice: How Many Times?

The docs have a "How Many Times?" table with six common loop patterns. Study it. Then, WITHOUT looking, answer THESE:

  1. for (int i = 0; i < 10; i++) — how many iterations? What are the values of i?
  2. for (int i = 1; i <= 10; i++) — how many iterations? Values of i?
  3. for (int i = 5; i > 0; i--) — how many iterations? Values of i?
  4. for (int i = 0; i <= 10; i += 2) — how many iterations? Values of i?

Write your answers, then check against the doc.

Off-By-One Trap: Read the off-by-one section in the docs. The loop for (int i = 0; i <= 5; i++) runs 6 times, not 5. The loop for (int i = 5; i < 5; i++) runs 0 times. The difference between < and <= is exactly one iteration. When does each matter?

Socratic Checkpoint — AP Exam Prep

  1. Read the "Exam Prep" Socratic Checkpoint in the docs. Which of the four loops produces the same output as for (int i = 0; i < 5; i++)? Answer WITHOUT running. Explain WHY the others are wrong.
  2. Trace this nested loop: how many times does the print statement execute? 
    for (int i = 0; i < 3; i++) {
    for (int j = 0; j < 4; j++) {
        System.out.print("*");
    }
    System.out.println();
}
  3. What's the difference between break and continuing through the normal loop exit? When would you use break inside a combat loop?

Checkpoint — before moving on, you must be able to:

  • Create a trace table for any loop and fill it correctly
  • Predict the number of iterations from a loop header
  • Identify and fix off-by-one errors
  • Determine if two loops are equivalent without running them
