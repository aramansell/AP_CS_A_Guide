4.2b — for Loops: Item Lister & Stat Printer

Basic Application ~25 min

for loops excel at two things: repeating an action a known number of times, and walking through a range of numbers. Today you build RPG tools that do both.

Build 1: Inventory Printer

Create ItemLister.java. Given an array of item names, print a numbered inventory:

String[] items = {"Sword", "Potion", "Shield", "Key", "Torch"};

// Output:
═══ INVENTORY ═══
1. Sword
2. Potion
3. Shield
4. Key
5. Torch
6 slots used, 10 total

The array has 5 items, but the inventory has 10 slots. Print all 10 rows — items 6-10 should say "(empty)". You need a for loop that goes from 0 to 9, and decides what to print based on whether the index has a corresponding item.

Bug Hunt: The ArrayIndexOutOfBounds

If you write for (int i = 0; i <= 10; i++), what happens when i = 5 and you access items[5]? Why does the loop keep going to 10 if the array only has 5 elements?

The fix: check if (i < items.length) before accessing the array. This is called bounds checking — and it's how every professional programmer avoids this crash.

Build 2: Multi-Table Roller

Create LootRoller.java. A loot table has multiple items with different drop rates. Roll for each one:

String[] table = {"Gold (50%)", "Potion (30%)", "Sword (10%)", "Armor (5%)", "Dragon Egg (1%)"};
double[] rates = {0.50, 0.30, 0.10, 0.05, 0.01};

System.out.println("Rolling loot...");
for (int i = 0; i < table.length; i++) {
    double roll = Math.random();
    if (roll < rates[i]) {
        System.out.println("★★★ " + table[i] + " — DROPPED!");
    }
}

Socratic Checkpoint

  1. Why does Math.random() < 0.50 give a 50% chance? If you wanted 75%, what number would you compare to?
  2. If you run LootRoller 10 times, will you always get the Dragon Egg at least once? Is there a guarantee? (Think: 1% chance, 10 trials...)
  3. What's the relationship between the for loop counter i and the tables array? Why must they be the same length?

Checkpoint — before moving on, you must be able to:

  • Use for loops to iterate over arrays with bounds checking
  • Combine Math.random() with a loop to roll multiple probabilities
  • Print formatted numbered lists with conditional logic inside the loop
  • Explain why the loop counter and array index are the same variable
