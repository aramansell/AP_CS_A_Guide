9.6a — Combat System: Polymorphic RPG Battles

RPG Build ~60 min

Your DungeonCrawl has one monster type at a time. But a real RPG has goblins, dragons, skeletons, slimes — each with different stats and behaviors. Inheritance makes this clean. One Combatant interface. One battle loop. Infinite monster variety.

Open the Combat System (Student RPG Guide). Follow the specification.

Part 1: Refactor to Inheritance

Convert your Monster class into a hierarchy:

  • Combatant interface — attack(), takeDamage(), isAlive(), getName(), getAttackPower()
  • Creature abstract class — HP, name, attackPower fields, implements Combatant
  • Goblin extends Creature — 20 HP, 5 ATK, bonus: attacks deal +2 damage
  • Dragon extends Creature — 80 HP, 20 ATK, attack() returns random 15-25 damage
  • Skeleton extends Creature — 30 HP, 8 ATK, 30% chance to dodge (take 0 damage)
  • Player implements Combatant — keep your existing Player, add the interface

Part 2: Polymorphic Battle Engine

Create BattleEngine.java with ONE method:

public static Combatant runBattle(Combatant a, Combatant b) {
    while (a.isAlive() && b.isAlive()) {
        System.out.println(a.getName() + " [" + a.getHp() + "] vs " + b.getName() + " [" + b.getHp() + "]");
        b.takeDamage(a.attack());
        if (b.isAlive()) a.takeDamage(b.attack());
    }
    return a.isAlive() ? a : b;
}

This method knows NOTHING about Goblin, Dragon, or Skeleton. It works with ANY Combatant — now and forever.

Part 3: Monster Spawning

Create MonsterSpawner.java:

  • Takes a difficulty parameter (1-3) in constructor
  • spawn() returns a random Combatant: difficulty 1 = Goblin or Skeleton, 2 = Goblin/Skeleton/Dragon(weakened), 3 = any
  • getBoss() returns a Dragon with +50% HP and attack

Replace DungeonCrawl's hard-coded monster with spawner.spawn(). Player fights through 5 random monsters, then the boss.

Socratic Checkpoint

  1. Your battle engine calls a.attack(). If a is a Skeleton, it has a 30% dodge chance — but the engine doesn't know that. How does the dodge happen?
  2. Why is Creature abstract instead of concrete? What happens if you try new Creature("???", 10, 5)?
  3. Look at the docs section on extensibility. How would you add a Healer that restores HP? What interface would you use?

Checkpoint — before moving on, you must be able to:

  • Build a class hierarchy with interface, abstract class, concrete classes
  • Write polymorphic battle logic independent of specific creature types
  • Create a factory/spawner pattern for random enemy generation
