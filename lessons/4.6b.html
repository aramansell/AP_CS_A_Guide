4.6b — Dungeon Loop: Room Exploration

RPG Build ~25 min

Part 2: runDungeon() — The For Loop

Your combat while loop works. Now build the dungeon that uses a for loop to generate a fixed number of rooms. Each room has a random encounter: combat, treasure, or trap. This is where you see the while-vs-for distinction in action — combat uses while (unknown rounds), rooms use for (known count).

Open the Dungeon Loop System (Student RPG Guide). Follow Part 2: runDungeon().

Add this method to DungeonLoop.java:

public static void runDungeon(int difficulty)

Requirements:

  • Print header: "=== DUNGEON CRAWL (Difficulty: [N]) ==="
  • Track: playerHP (start 20), gold (start 0), roomsSurvived (start 0)
  • For loop: room 1 to 5. At top of each iteration: if playerHP ≤ 0, break
  • Each room: random type 0/1/2:
    • Combat (0): Scale a monster. HP = 5 + room × difficulty, AC = 10 + room. Call runCombat(). Win → add 10 × room gold. Lose → playerHP = 0.
    • Treasure (1): Add 5 + Dice.roll(10) × difficulty gold. Print what was found.
    • Trap (2): Subtract Dice.roll(6) from playerHP. Print damage and remaining HP.
  • Increment roomsSurvived after each room
  • After loop: print summary (rooms explored, gold, HP remaining)

Socratic Checkpoint

  1. Why is the dungeon loop a for loop but the combat loop is a while loop? Could you swap them? What would break?
  2. You use break when the player dies. What happens to the remaining rooms? What if you used return instead?
  3. HP carries over across rooms. Gold accumulates. Where are these variables declared — inside the for loop or outside it? Why?

Tinker: Escalating Difficulty

Make room difficulty scale: monster HP and AC increase by 1 per room. By room 5, monsters are significantly tougher. Run the dungeon at difficulty 1, then difficulty 3. How often does the player survive? What difficulty feels "fair"?

Acceptance Test:

DungeonLoop.runDungeon(1);
// Verify: exactly 5 rooms (or fewer if player dies)
// Room types vary (re-run to see all three)
// Combat, treasure, and trap rooms work as described
// HP and gold carry across rooms
// Player death skips remaining rooms
RPG Build ~10 min

Part 3: main() — Wire It Together

Add a main method that tests both pieces:

  1. Call runCombat() once with hardcoded stats to verify the while loop in isolation
  2. Call runDungeon(1) to run the full game

Run DungeonLoop.main() several times. Results should vary — dice rolls make every run different.

Checkpoint — before moving on, you must be able to:

  • Explain when to use while vs for based on the problem
  • Write a while loop for indeterminate combat rounds
  • Write a for loop for a fixed series of dungeon rooms
  • Use break to exit a loop early
  • Call a method from inside a loop and use its return value
