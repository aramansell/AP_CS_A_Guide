3.5c — Choose Your Adventure

Type: Expand Application. You've built the number guessing game with one branch — now chain multiple branches into a story. Same if/else pattern, stretched into a narrative.

Expand Application ~30 min

The Goal

Build a text adventure where the player makes choices at each turn and the story branches. At least 3 decision points with at least 2 choices each.

You wake up in a dark forest. Moonlight filters through the trees.
To the NORTH, a stone tower looms against the sky.
To the EAST, you hear a stream bubbling.

Which way? (north/east)
> north
You approach the tower...

Reference: Choose Your Adventure Walkthrough — the full reference with example branches.

Build It: Your Adventure

Create Adventure.java. Start with this skeleton:

import java.util.Scanner;

public class Adventure {
    public static void main(String[] args) {
        Scanner input = new Scanner(System.in);

        System.out.println("You wake up in a dark forest.");
        System.out.println("NORTH: a stone tower. EAST: a stream.");
        System.out.print("Which way? (north/east): ");
        String choice = input.nextLine();

        if (choice.equals("north")) {
            System.out.println("You approach the tower...");
            // YOUR SECOND BRANCH GOES HERE
        } else if (choice.equals("east")) {
            System.out.println("You follow the sound of water...");
            // YOUR SECOND BRANCH GOES HERE
        } else {
            System.out.println("You stand still, unsure...");
        }
    }
}

Bug Hunt

What happens if the player types "North" with a capital N? Try it. Then fix it. (Hint: the Scanner docs show how).

Tinker Challenge — Deeper Branches

Add at least a SECOND decision point inside each branch (you'll write another if/else inside the first one). That means at minimum:

  • North → enter tower or knock? → each leads to a different third scene.
  • East → drink from stream or cross it? → each leads to a different third scene.

That gives you at least 4 distinct endings.

Socratic Checkpoint

  1. Why does comparing Strings need .equals() instead of ==? What does == actually check for objects?
  2. How many if/else blocks do you have in total? Count them.
  3. What's the deepest nesting level in your code? How many braces deep does the innermost branch go?
  4. How is this structurally similar to the combat engine your RPG needs? Where would an attack branch lead vs. a defend branch?

Extend It (pick one)

  • Add an inventory: the player picks up an item in one branch, and it affects a later branch.
  • Add a health system: bad choices reduce HP.
  • Add a "go back" option that loops to the previous decision (you'll need a while loop).
