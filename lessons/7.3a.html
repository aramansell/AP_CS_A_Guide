7.3a — Searching & Sorting: Finding Needles, Organizing Haystacks

Discovery Lab ~25 min

Your inventory can hold items. But finding a specific item by scanning the whole list every time is slow. And displaying items in a useful order (by value, by rarity) requires sorting. These are the two most common operations on collections and the AP exam tests them relentlessly.

Experiment: Linear vs Binary Search

Create SearchComparison.java:

ArrayList<Integer> nums = new ArrayList<Integer>();
for (int i = 0; i < 20; i++) nums.add(i * 5);
// nums = [0, 5, 10, 15, ... 95]  — already sorted!

// Linear search: check every element in order
int target = 65;
int steps = 0;
for (int i = 0; i < nums.size(); i++) {
    steps++;
    if (nums.get(i) == target) {
        System.out.println("Linear found at " + i + " in " + steps + " steps");
        break;
    }
}

Now implement binary search. Read the searching & sorting docs for the algorithm. Compare step counts for target values at the beginning, middle, and end of the list. Which search wins and when?

Binary Search Requires Sorted Data: If the list isn't sorted, binary search can return wrong answers — not just be slow. The AP exam often asks "what does binary search return on this unsorted array?" The answer is usually wrong. Read the docs Bug Hunt for the most common binary search mistakes.

Build: Inventory Sorter

Using your Item class and ArrayList:

  • Write sortByValue(ArrayList<Item> items) — selection sort by item value (lowest first). Swap entire Item objects.
  • Write sortByRarity(ArrayList<Item> items) — selection sort by rarity (Common < Uncommon < Rare < Epic). Map rarities to numbers first.
  • Write binarySearch(ArrayList<Item> sorted, String name) — assumes sorted by name. Returns index or -1.
  • Test: create 8 items, sort by value, display. Sort by rarity, display. Search for an existing and non-existing item name.

Socratic Checkpoint

  1. Selection sort finds the smallest remaining element and swaps it to the front. On a list of N elements, how many comparisons does it make? How many swaps?
  2. Binary search splits the search space in half each step. If you have 1,000,000 items, how many steps max? (Hint: log₂)
  3. Your insert method from Unit 6 took O(N) time. ArrayList.add(index, elem) also takes O(N) — why? What's happening under the hood?

Checkpoint — before moving on, you must be able to:

  • Implement linear search and binary search
  • Implement selection sort on an ArrayList of objects
  • Explain when binary search is safe to use
  • Compare O(N) vs O(log N) at an intuitive level
