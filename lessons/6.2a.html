6.2a — Array Traversal: Walking the Row

Discovery Lab ~25 min

Yesterday you accessed array elements one at a time. But printing 100 elements with 100 println statements is insanity. You need to walk the array — visit every element in order using a loop. This is called traversal.

Experiment: For-Loop Traversal

Create ArrayWalk.java. Try each pattern:

int[] scores = {95, 87, 72, 100, 63};

// Pattern 1: standard for loop
for (int i = 0; i < scores.length; i++) {
    System.out.println("scores[" + i + "] = " + scores[i]);
}

// Pattern 2: enhanced for loop (for-each)
for (int score : scores) {
    System.out.println(score);
}

Run both. Answer: what's the difference? When would you use each?

Experiment: Modify During Traversal

// Double every score
for (int i = 0; i < scores.length; i++) {
    scores[i] = scores[i] * 2;
}

// Try the same with for-each:
for (int score : scores) {
    score = score * 2;  // Does this change the array?
}
// Print the array after each attempt. What happened?
For-Each Modification Trap: Enhanced for loops give you a COPY of each element. Modifying score doesn't change the array. If you need to modify elements, use the standard for loop with an index. Read the Array Traversal docs for the full explanation.

Build: Statistics Calculator

Create ArrayStats.java:

  • Hard-code an array of 8 test scores: {88, 92, 75, 64, 100, 83, 77, 91}
  • Traverse to compute: sum, average (as double!), minimum, maximum
  • Count how many scores are above average
  • Print all results with labels

Socratic Checkpoint

  1. You computed average as a double. Why? What happens if you use int for the average — what data do you lose?
  2. Look at the traversal docs section on reverse traversal. Why would you ever want to go backwards through an array?
  3. If your array has 100,000 elements, would your for-loop traversal still work? What about the for-each? Is one faster?

Checkpoint — before moving on, you must be able to:

  • Traverse an array with both standard for and enhanced for
  • Compute sum, average, min, max during a single pass
  • Explain when for-each CANNOT replace standard for
  • Count elements matching a condition during traversal
