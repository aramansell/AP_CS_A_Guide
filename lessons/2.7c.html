2.7c — Character Creator: Build a Party

RPG — Part 3 ~20 min

Part 4: Make It Real — Generate a Full Party

One character is fine. A party of four is a game. Create a CharacterTester.java with a main method that generates and displays a full adventuring party.

Requirements:

  • Create 4 Character objects
  • Name them (Grog the Warrior, Elara the Mage, etc.)
  • Call generateStats() on each
  • Call printCharacterSheet() on each

Tinker: The Weakest Link

After generating all four characters, find and print the name of the character with the LOWEST total HP and the character with the HIGHEST strength. You'll need to compare values across objects.

Final Socratic Checkpoint — Unit 2 Kickoff

  1. Trace through the entire Character Creator. Where is the three-step pattern? Which section of the pattern is in Character.java vs CharacterTester.java?
  2. You use Dice.statRoll() in your Character class. How is this different from copying the Math.random() formula directly into Character.java?
  3. If you wanted to add an Elf race that gets +2 DEX and -1 CON, where would you add that logic?
  4. You've now started Unit 2. What patterns, habits, and mental models do you want to carry through the rest of Unit 2?

Checkpoint — before moving on, you must be able to:

  • Create and configure multiple instances of a class
  • Call methods on objects from a separate tester class
  • Compare values across objects (highest/lowest)
  • Explain the full architecture: Dice.java → Character.java → CharacterTester.java
