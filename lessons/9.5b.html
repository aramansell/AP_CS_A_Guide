9.5b — Shape Hierarchy (Build Together)

Type: Basic Application. The teacher builds, you follow along. Your first inheritance hierarchy — a parent class with abstract methods, concrete children, and polymorphic arrays. The full OOP toolkit in one build.

Basic Application ~35 min

The Goal

Build a Shape parent class with getArea() and getPerimeter(). Create Circle, Rectangle, and Triangle as children. Store them all in a Shape[] array and print each one's area — without knowing which specific shape it is.

Circle (radius 5.0): area = 78.54, perimeter = 31.42
Rectangle (4.0 x 6.0): area = 24.0, perimeter = 20.0
Triangle (base 3.0, height 4.0): area = 6.0, perimeter = 12.0

Iteration 1: The Parent

Create Shape.java. This defines what EVERY shape can do:

public abstract class Shape {
    protected String name;

    public Shape(String name) {
        this.name = name;
    }

    public abstract double getArea();
    public abstract double getPerimeter();

    public String toString() {
        return name + ": area = " + getArea()
             + ", perimeter = " + getPerimeter();
    }
}

Socratic Checkpoint

  1. Why is Shape abstract? What would happen if you tried new Shape("blob")?
  2. Why can toString() call getArea() even though getArea() has no body? When does that call get resolved?
  3. What does protected mean? How is it different from private and public?

Iteration 2: Circle

Create Circle.java. Circle HAS-A radius:

public class Circle extends Shape {
    private double radius;

    public Circle(double radius) {
        super("Circle (radius " + radius + ")");
        this.radius = radius;
    }

    public double getArea() {
        return Math.PI * radius * radius;
    }

    public double getPerimeter() {
        return 2 * Math.PI * radius;
    }
}

Bug Hunt

What happens if you forget the super(...) call in Circle's constructor? Try it. The compiler error tells you exactly why: "no default constructor available in Shape." Why doesn't Shape have a default constructor? (You wrote one with a parameter — same issue as Card in Unit 5!)

Iteration 3: Rectangle and Triangle

Create Rectangle.java and Triangle.java following the same pattern as Circle. Each needs:

  • Private fields for dimensions
  • A constructor that calls super(...)
  • getArea() and getPerimeter()

For Triangle, use the formula 0.5 * base * height for area. For perimeter, you'll need all 3 sides. Start with a right triangle (two legs + hypotenuse from Pythagorean theorem).

Iteration 4: Polymorphic Array

Create ShapeTester.java. The magic: store different types in one array:

public class ShapeTester {
    public static void main(String[] args) {
        Shape[] shapes = new Shape[3];
        shapes[0] = new Circle(5.0);
        shapes[1] = new Rectangle(4.0, 6.0);
        shapes[2] = new Triangle(3.0, 4.0, 5.0);

        for (Shape s : shapes) {
            System.out.println(s);
            // Which getArea() gets called?
        }
    }
}

Socratic Checkpoint

  1. When the loop calls s.getArea(), how does Java know whether to use Circle's formula or Rectangle's formula? (This is dynamic dispatch.)
  2. Why can you store a Circle in a Shape[] array but not in an int[] array?
  3. How is this pattern used in your RPG? Think about your Enemy class and its subclasses.
