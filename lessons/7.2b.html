7.2b — Traversal with Removal: Don't Skip Elements

Basic Application ~20 min

Build: Cleanup Tool

Removing elements while traversing an ArrayList is tricky — indexes shift! You'll learn the right pattern for filtering out unwanted items.

Create a class called InventoryCleanup with an ArrayList<String> items:

  1. Fill with items: "Rusty Sword", "Health Potion", "Broken Shield", "Mana Potion", "Torn Cloak", "Elixir".
  2. Remove all items containing "Broken" or "Torn" or "Rusty".
  3. First try: use a standard for loop with i++. What goes wrong?
  4. Fix it: traverse backwards (from size()-1 down to 0). Why does backward traversal work?
  5. Alternative: use a while loop with an Iterator (or just check the index carefully).

The Trap

This is one of the most common ArrayList bugs. If you remove index 2, what was at index 3 is now at index 2. If your loop increments i, you skip it. Draw the indexes on paper to see why.
