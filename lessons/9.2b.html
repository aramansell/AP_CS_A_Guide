9.2b — Override vs Overload: Know the Difference

Basic Application ~20 min

Build: Weapon System

Override changes behavior in a subclass (same signature). Overload provides multiple versions of the same method name (different parameters). These are easily confused — you'll build both.

Create Weapon (parent):

  1. Field: String name, int baseDamage.
  2. Method: int getDamage() — returns baseDamage.

Create EnchantedWeapon extends Weapon:

  1. Additional field: int bonusDamage.
  2. Override getDamage() — returns baseDamage + bonusDamage. Mark it with @Override.
  3. Overload getDamage(String element) — if element matches the weapon's element, add 5 more. Use @Override? Why or why not?

Test both. Call getDamage() and getDamage("fire") on an EnchantedWeapon reference. Now store it in a Weapon variable — which methods are accessible?
