1.1c — Arithmetic & Operators

Type: Discovery Lab. Create a file called MathTest.java and try every expression below. Run it, break it, figure it out.

Discovery Lab ~15 min

Java can do math. The operators look like what you'd expect — but there's one critical trap that surprises nearly every new programmer. Let's find it.

The Five Operators

Tinker Challenge: The Division Trap

Run this code. Make a prediction before you run it — then check.

System.out.println(10 / 3);
System.out.println(7 / 2);
System.out.println(1 / 2);

Wait — why does 1 / 2 print 0 instead of 0.5? What pattern do you see across all three lines?

Socratic Checkpoint

How can you force Java to give you the exact decimal answer for 10 ÷ 3? Change the numbers in your code and run it again. What happens if you use a number with a decimal point? What happens if you write double anywhere in the expression? Experiment until you find at least two different ways to get 3.333... from 10 ÷ 3.

The Rule: When both operands are int, Java does integer division. It chops off everything after the decimal point — no rounding, just gone. If any operand is a double, the whole expression uses floating-point math.

Bug Hunt: The Broken Average

A student wrote this code to calculate their test average. It compiles and runs, but the answer is wrong.

int test1 = 92;
int test2 = 87;
int test3 = 95;
double average = (test1 + test2 + test3) / 3;
System.out.println(average);   // prints 91.0 — but it should be ~91.33

The math is right — (92+87+95) ÷ 3 = 91.33. So why does Java give 91.0? Hint: look at the types on the division line.

Fix it. Make the code give the correct decimal answer.

Modulus (%) — the remainder operator

Modulus gives you the remainder after division. It's more useful than it looks.

Tinker Challenge: Modulus Patterns

Run this and look at the pattern:

System.out.println(10 % 3);
System.out.println(11 % 3);
System.out.println(12 % 3);
System.out.println(13 % 3);

What is the maximum value you could possibly get when using % 3? What about % 10? If you had a random integer from 1 to 100, how could you use % to check if it's even or odd?

Compound Assignment Operators

Shorthand for "do math on a variable and store it back."

ShorthandEquivalent
x += 5;x = x + 5;
x -= 3;x = x - 3;
x *= 2;x = x * 2;
x /= 2;x = x / 2;
x %= 3;x = x % 3;
int hp = 100;
hp -= 15;    // took 15 damage — now 85
hp += 25;    // drank a potion — now 110

Operator Precedence: * / % before + -. Use parentheses when in doubt.

System.out.println(5 + 3 * 2);       // 11 (3*2=6, then 5+6)
System.out.println((5 + 3) * 2);     // 16 (5+3=8, then 8*2)

Your Turn

  1. Write your own RPG example using += and -= for HP changes. Test: start at 100, take 30 damage, drink a potion for 15, take 5 more damage. What's the final HP?
  2. Write an expression that calculates the area of a rectangle (width × height). Print it with a label.
  3. What does (double) 7 / 2 print? What about 7 / (double) 2? Are they different? Why?

Checkpoint — before moving on, you must be able to:

  • Explain the integer division trap and how to avoid it
  • Use modulus (%) to check if a number is even or odd
  • Use compound assignment operators (+=, -=, etc.)
  • Write and fix arithmetic expressions that mix int and double
