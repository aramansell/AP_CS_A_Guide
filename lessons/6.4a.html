6.4a — Arrays of Objects: Not Just Numbers

Discovery Lab ~25 min

Arrays aren't limited to int and double. You can store objects — Monsters, Items, Players, Strings. But arrays of objects have a critical difference from arrays of primitives: each slot starts as null, not zero. Forget this and you'll meet your new enemy: NullPointerException.

Experiment: Null Is Not Zero

Create ObjectArray.java:

// Create array — all slots are null
Weapon[] armory = new Weapon[3];

// This CRASHES: armory[0].getName();  // NullPointerException!

// You must create objects first
armory[0] = new Weapon("Sword", 8);
armory[1] = new Weapon("Axe", 10);
armory[2] = new Weapon("Dagger", 4);

// Now this works
for (Weapon w : armory) {
    System.out.println(w.getName() + " deals " + w.getDamage());
}

Now create an array of size 5 but only fill slots 0-2. Loop through all 5 with for-each. What happens at slot 3? Fix it with a null check.

NullPointerException: The most common runtime error in Java. It means "you tried to call a method on something that doesn't exist yet." Always check if (obj != null) before calling methods on array elements — especially when the array isn't full. Read the Arrays of Objects docs for the complete null-safety patterns.

Build: Loot Table

Create LootTable.java with your Weapon class from Unit 5:

  • Create an array of 6 Weapons with varied names and damage values
  • Write getRandomLoot() — returns a random Weapon from the array
  • Write getMostPowerful() — returns the Weapon with highest damage
  • Write countByDamage(int minDamage) — counts weapons above a threshold
  • Test: call getRandomLoot() 10 times and print what you got

Socratic Checkpoint

  1. Why does new Weapon[3] create null slots instead of default Weapon objects? What would Java need to know to create default objects?
  2. Your getRandomLoot() returns a Weapon reference. If the caller modifies the returned Weapon (changes its damage), does it affect the original in the array? Why or why not?
  3. Read the docs section on sorting objects. Why can't you just use < to compare two Weapons?

Checkpoint — before moving on, you must be able to:

  • Create an array of objects and initialize each slot
  • Null-check before calling methods on array elements
  • Traverse an object array to find max, count by condition, random pick
  • Explain why new ClassName[N] gives null slots
