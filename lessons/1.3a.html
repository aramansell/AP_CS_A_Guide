1.3a — Math.random() Deep Dive

Math.random() is the engine behind every dice roll, loot drop, and random event in your RPG. Today you figure out how it works — not by being told, but by running experiments.

Discovery Lab ~45 min

Official Documentation

public static double random()

Returns a double value with a positive sign, greater than or equal to 0.0 and less than 1.0. Returned values are chosen pseudorandomly with (approximately) uniform distribution from that range.

When this method is first called, it creates a single new pseudorandom-number generator, exactly as if by the expression new java.util.Random(). This new pseudorandom-number generator is used thereafter for all calls to this method and is used nowhere else.

Returns:
a pseudorandom double greater than or equal to 0.0 and less than 1.0.

Socratic Checkpoint: Reading the Docs

Answer these before you write any code:

  1. What data type does this method return?
  2. Is it possible for this method to return exactly 0.0?
  3. Is it possible for this method to return exactly 1.0?
  4. If you wanted a random integer between 0 and 9 (inclusive), what mathematical operations could you apply to the result?

Tinker Challenge 1: The Raw Call

Create a file named RandomTest.java with a main method. Run this 5 times:

System.out.println(Math.random());

Observation: Do your results match what the documentation promised? What is the highest value you saw?

Tinker Challenge 2: Scaling Up

Games usually use whole numbers (like rolling a 6, not 0.832). Change your code:

System.out.println(Math.random() * 6);

Observation: Run it 5 times. What is the new possible range of values? Can it ever output 6.0? Why or why not, based on the documentation?

Tinker Challenge 3: Chopping the Decimal

We only want whole numbers. In Java, we "cast" a double to an int. Add the cast:

System.out.println((int) Math.random() * 6);

Observation: Run it 5 times. Wait — why does it only print 0?

Hint: Think about order of operations. Does the computer cast Math.random() to an int first, or does it multiply first?

Now fix the code so the multiplication happens before the cast. Run it 5 times. What is the range of integers you get now?

Tinker Challenge 4: Shifting the Range

You should be getting numbers from 0 to 5. But a standard die rolls from 1 to 6. Modify your print statement to shift the entire range up by 1.

Synthesis: The General Formula

Based on your experiments above, how would you write a single line of code to get a random integer where you define:

  • The number of possible values (the "range")
  • The starting value (the "min")

Write your formula down before checking! Test it: can you generate a random number from 10 to 99? From 1 to 20? From 3 to 18?

Checkpoint — before moving on, you must be able to:

  • Explain the range of Math.random() (0.0 ≤ x < 1.0)
  • Write the general formula for generating a random integer in any range — test it with at least three different ranges to be sure
  • Explain WHY parentheses are necessary around the multiplication
  • Generate: a d6 (1–6), a d20 (1–20), 3d6 (3–18), a number from 10–99
  • Explain the formula to a partner — in your own words, not memorized
