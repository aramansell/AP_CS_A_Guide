10.2c — Recursive Loot Finder: Best Path Through Treasure

Expand Application ~30 min

Build: Treasure Path Optimizer

You're in a dungeon with rooms containing gold. You can only move right or down. What's the maximum gold you can collect from start to exit? This is a classic recursive optimization problem.

Create a class called TreasurePath with a 2D gold grid:

int[][] gold = {
    {5, 3, 2, 1},
    {2, 8, 4, 3},
    {1, 2, 6, 5},
    {3, 1, 2, 9}
};
  1. int maxGold(int row, int col): returns the maximum gold from this tile to the bottom-right, moving only right or down.
    • Base case 1: out of bounds → return 0
    • Base case 2: at bottom-right → return gold[row][col]
    • Recursive: gold[row][col] + max of going right vs going down
  2. Call maxGold(0,0) to find the best path.
  3. BONUS: also print the actual path (the row,col coordinates) of the optimal route.

Performance Question

This recursive solution explores ALL paths. How many total paths exist in a 4×4 grid? Is there a more efficient way?
