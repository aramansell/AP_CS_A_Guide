2.5b — Math Class: Stat Calculations & Bounds

Basic Application ~25 min

RPG stats are never raw numbers — they're always bounded, scaled, or modified. Today you build a stat calculator that uses Math methods to enforce the rules of the game.

Review: Math methods are static. You call them on the class directly — Math.min(a, b), never new Math(). If you're not sure why, check the Math docs — static section.

Build: StatCalculator.java

Create a program that calculates RPG stats from raw inputs, enforcing these rules:

  • HP: base 50 + (strength * 2). Minimum 10, maximum 200.
  • Damage: weapon damage * (1 + strength / 20.0). Minimum 1.
  • Critical chance: dexterity * 0.5%. Maximum 25%.
  • Defense: sqrt(armor * 2). Round to nearest int.
// Test with these values:
Strength: 15, Weapon Damage: 8, Dexterity: 12, Armor: 50
→ HP: 80, Damage: 14.0, Crit: 6.0%, Defense: 10

Socratic Checkpoint

  1. You need to clamp HP to 10-200. Which Math methods do you use? Why do you need TWO of them?
  2. Damage must be minimum 1. Which single Math method guarantees this?
  3. sqrt(armor * 2) returns a double. How do you round it to the nearest int? Check the Math docs — is there a method specifically for rounding?

Tinker: Extreme Values

Test your calculator with these edge cases. Before running, predict what each output will be:

// Edge case: negative strength?
Strength: -5  →  HP should be at least 10

// Edge case: zero damage
Weapon: 0, Strength: 0  →  Damage should be at least 1

// Edge case: absurd dexterity
Dexterity: 200  →  Critical chance should cap at 25%

Did your predictions match? If any output surprised you, trace through the formula by hand.

Bug Hunt: The Clamp Order Trap

A student wrote Math.min(10, Math.max(hp, 200)) to clamp HP between 10 and 200. Run it with hp = 50. What prints? Why is it wrong?

Clamp order matters. Math.min(a, Math.max(x, b)) is different from Math.max(a, Math.min(x, b)). Figure out which one correctly enforces "between a and b" — and explain why the other one doesn't.

Checkpoint — before moving on, you must be able to:

  • Use Math.min and Math.max to clamp values between bounds
  • Use Math.sqrt and Math.pow for stat formulas
  • Round doubles to ints correctly (not just cast)
  • Explain why clamp requires TWO Math calls and in what order
