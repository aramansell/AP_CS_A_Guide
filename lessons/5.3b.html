5.3b — Accessors & Mutators: Getters with Guardrails

Basic Application ~20 min

Build: Backpack with Limits

Yesterday you learned why accessor (getter) and mutator (setter) methods protect your data. Today you'll build an inventory system that uses mutators to enforce rules — no negative gold, no items over the weight limit.

Create a class called Backpack:

  1. Private instance variables: int gold, int currentWeight, int maxWeight
  2. Constructor takes maxWeight. Gold starts at 0, currentWeight at 0.
  3. Getter getGold(), getter getCurrentWeight().
  4. Mutator boolean addItem(int itemWeight): returns false if it would exceed maxWeight.
  5. Mutator void addGold(int amount): refuses to add negative amounts. Print a warning if someone tries.
  6. Mutator boolean spendGold(int amount): returns false if not enough gold.

Test Edge Cases

Create a Backpack(maxWeight=20). Try addItem(25) — should fail. Try addGold(-50) — should refuse. Try spendGold(100) when gold is 0 — should fail. Do all your guards work?
