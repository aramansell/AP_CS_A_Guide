10.1a — Recursion Basics: A Method That Calls Itself

Discovery Lab ~25 min

Every loop you've written so far has been iterative — a block that repeats. Recursion is different: a method calls itself to solve a smaller version of the same problem, until the problem is so simple it can be answered directly. This sounds impossible — how can a method call itself without running forever? The answer is the base case: a condition that stops the recursion.

Experiment: Countdown, Recursively

Create RecursionIntro.java. Write a recursive countdown:

public static void countdown(int n) {
    if (n <= 0) {                    // base case: stop
        System.out.println("Blastoff!");
        return;
    }
    System.out.println(n);           // do work
    countdown(n - 1);                // smaller problem
}

Call countdown(5). Trace it on paper — each call creates a new "frame" with its own n value. Draw the call stack.

Missing Base Case = StackOverflowError: Remove the if(n <= 0) check. Run countdown(5). Java crashes because the calls never stop — the call stack fills up all available memory. This is infinite recursion. Read the Recursion Basics docs for the three rules every recursive method must follow.

Experiment: Factorial

The classic recursive example:

// factorial(5) = 5 * 4 * 3 * 2 * 1 = 120
public static int factorial(int n) {
    if (n <= 1) return 1;            // base case
    return n * factorial(n - 1);     // recursive case
}

Call factorial(5) and trace the call stack: factorial(5) → 5 * factorial(4) → 5 * 4 * factorial(3) ... down to factorial(1) = 1. The multiplication happens on the way BACK up.

Build: Iterative vs Recursive

Write BOTH versions of these methods and compare:

  • sumToN(int n) — sum of 1 + 2 + ... + n
  • power(int base, int exp) — base^exp, multiplication only, no Math.pow()

For each, trace factorial(10000) both ways. Which one crashes? Why?

Socratic Checkpoint

  1. Why does factorial(10000) crash with recursion but not with iteration? How many stack frames is that?
  2. What happens if your base case is if (n == 1) and you call factorial(0)?
  3. Read the docs section on call stack depth. What's the typical stack limit in Java (~10,000 frames) and why does it exist?

Checkpoint — before moving on, you must be able to:

  • Identify base case and recursive case in a recursive method
  • Trace recursive calls on paper showing the call stack
  • Convert simple loops to recursion and vice versa
  • Explain why infinite recursion causes StackOverflowError
