7.2a — ArrayList Traversal: Walking a Growing List

Discovery Lab ~25 min

You know how to walk an array with for and for-each. ArrayList works the same way — but with one critical difference: if you remove elements while traversing, the indices shift under your feet. This is the most common ArrayList bug on the AP exam.

Experiment: The Shifting Index Trap

Create RemoveExperiment.java:

ArrayList<Integer> nums = new ArrayList<Integer>();
for (int i = 1; i <= 5; i++) nums.add(i * 10);
// nums = [10, 20, 30, 40, 50]

// ATTEMPT 1: Remove all numbers < 30
for (int i = 0; i < nums.size(); i++) {
    if (nums.get(i) < 30) {
        nums.remove(i);
    }
}
System.out.println(nums);  // What actually got removed?

Trace it on paper. When you remove index 0, what was at index 1 slides to index 0 — but the loop already checked index 0, so it moves to index 1 and SKIPS the element that just slid in. Fix it: decrement i after removal, or traverse backwards.

ConcurrentModificationException: If you use for-each and try to remove, you get this exception. For-each uses an internal iterator that detects mid-traversal changes and throws. Only standard for loops (with index tracking) can safely remove during traversal. Read the ArrayList Traversal docs for the complete rules.

Build: Inventory Filter

Using your Item class from Unit 6, create InventoryFilter.java:

  • ArrayList<Item> with 8-10 items of various rarities and values
  • removeJunk() — removes all "Common" items. Handle the shifting index properly.
  • removeExpensive(int threshold) — removes items above threshold value
  • displayByRarity(String rarity) — prints all items of given rarity (no removal)
  • Test: display all, filter, display again. Verify correct items removed.

Socratic Checkpoint

  1. Why does traversing backwards (i = size()-1; i >= 0; i--) fix the removal problem? Draw a list and remove an element from the middle while going backwards.
  2. Look at the docs section on filtering with a new list. What's the trade-off between removing in-place vs building a new filtered list?
  3. If you remove ALL elements from an ArrayList during traversal, what's the time complexity? How many shifts happen?

Checkpoint — before moving on, you must be able to:

  • Traverse ArrayList with standard for and for-each
  • Safely remove elements during traversal (reverse or decrement)
  • Explain why for-each throws ConcurrentModificationException on remove
  • Filter a list by building a new ArrayList vs modifying in-place
