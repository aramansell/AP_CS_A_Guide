1.6a — Split Between Friends

Type: Expand Application. You've seen the basic pattern. Now stretch it — add features, solve new problems. Same skeleton handles every new requirement.

Expand Application ~20 min

The basic tip calculator handles one person. Extend it to split the bill.

Add these variables to your existing TipCalculator:

  • int numberOfPeople — how many are splitting the bill
  • double perPerson — what each person owes

Modify your program:

  1. After calculating the total, divide by numberOfPeople to get perPerson
  2. Print: "Each of N people pays: $X.XX"
  3. Print: "Tip per person: $X.XX" (tip divided by number of people)

Test: $87.50 bill, 18% tip, 3 people. Each should pay $34.42 (total is $103.25, divided by 3).

Socratic Checkpoint

Which section of the three-step pattern changed when you added this feature? Did you need to change Step 1? Step 2? Step 3? Or all three?

Tinker: Add Sales Tax

Add a tax rate variable (like 8.25%). Apply tax to the bill before calculating the tip. Where does this new calculation go in the three-step pattern — before the tip calculation or after?
