3.4b — String Comparison: Password & Command Checker

Basic Application ~20 min

Build: Password Validator

Yesterday you learned that == doesn't work for comparing String contents — you need .equals(). Today you'll build a password checker that also handles case-insensitive matching.

Create a class called PasswordChecker. Hardcode a String userInput and a String correctPassword (set to "DragonFire42").

  1. Use .equals() to check for an exact match. Print "Access granted" or "Access denied".
  2. Add a check using .equalsIgnoreCase() — if the user typed the right letters but wrong case, print "Check your caps lock!"
  3. Add a length check: if the input is shorter than 8 characters, print "Password too short."

Trap to Discover

Try comparing with == instead of .equals(). Make userInput = "DragonFire42" (same text). What happens? Now make userInput = new String("DragonFire42"). What happens now? Why are they different?
