6.5b — FixedInventory.java: Array-Based Backpack

RPG Build ~25 min

You have Item objects. Now you need somewhere to put them. You'll build FixedInventory.java — a container that holds items in a plain array, tracks how many slots are used, and handles add/remove/search operations. This is the array-based collection pattern you'll see everywhere in CS.

Open the Inventory Builder (Student RPG Guide) File 2: FixedInventory.java.

Build: FixedInventory.java

Create FixedInventory.java:

  • Private fields: Item[] items and int count
  • Constructor takes capacity, creates the array
  • add(Item item) — adds at position count, then count++. Return false if inventory is full.
  • remove(String name) — find first match by name, shift all later items left to close the gap. Set last used slot to null.
  • contains(String name) — returns true if any item matches
  • getTotalValue() — sum of all item goldValues
  • display() — numbered list: "1. Sword [Epic] — Wt:12 Val:250g"

Socratic Checkpoint

  1. Why do you need both items.length (capacity) and count (how many are stored)? Why not use items.length for both?
  2. Your remove() shifts elements left. If you remove the first item from a full 20-slot inventory, how many shifts happen? What if you remove the last item?
  3. What happens if you add 21 items to a 20-slot inventory? Is the 21st item silently lost, or does your code tell you something?

Acceptance — your FixedInventory must:

  • Reject add() when full (return false)
  • Handle remove() on empty inventory gracefully
  • Shift elements correctly — no gaps after remove
  • Pass all acceptance tests in the Student RPG Guide
