4.1a — while Loops: Repeating While True

Discovery Lab ~25 min

Every program you've written runs its code once and stops. One attack roll. One tip calculation. But games don't work that way — combat rounds repeat. You explore rooms until the dungeon runs out. Your player keeps drinking potions until their inventory is empty.

A while loop is an if statement that runs again. It checks a condition, and if it's true, runs the body. Then checks again. And again. Until the condition becomes false.

The Core Idea

while (condition) {
    // runs REPEATEDLY while condition is true
}
// execution continues here when condition becomes false

Experiment: Your First Loop

Create FirstLoop.java. Run this code:

int count = 0;
while (count < 5) {
    System.out.println("Count is: " + count);
    count++;
}
System.out.println("Done!");

Write down: how many times does the println inside the loop run? What's the first value of count printed? The last? Now change < 5 to <= 5 and run again. What changed?

Infinite Loop Trap: Open the while Loops docs. Find the Bug Hunt section "The Frozen Game." Read the broken code. Before looking at the answer: why does it freeze? What is the loop MISSING? Write down your answer, then run the broken code yourself. (Save your work first — you'll need Ctrl+C to stop it.)

Build: Combat Round Loop

You already have Combat.java from Unit 3. Now wrap a combat round in a while loop. Create CombatLoop.java:

  • Initialize int playerHP = 20 and int enemyHP = 12
  • Use a while loop: keep fighting while BOTH are alive
  • Inside: the player attacks, then — only if the enemy is still alive — the enemy counterattacks
  • Print HP after each round
  • When someone dies: print victory or defeat

This is the same combat from 3.6c, but now you understand WHY the while loop goes there instead of blindly copying it.

Socratic Checkpoint

  1. In your CombatLoop, what would happen if you forgot to subtract damage from enemyHP? What about playerHP? Try it — but save first.
  2. Read the docs section on sentinel loops. What's the difference between a counted loop and a sentinel loop? Which one is combat?
  3. The docs say every while loop needs three parts: initialize, test, update. Identify all three in your combat loop.
  4. Look at the sentinel loop example in the docs (the one with Scanner and "quit"). What's the sentinel value? Why does the loop use !command.equals("quit") instead of command.equals("quit")?

Checkpoint — before moving on, you must be able to:

  • Write a while loop with all three parts (initialize, test, update)
  • Identify what causes an infinite loop and fix it
  • Use a while loop for indeterminate repetition (combat, sentinel)
  • Explain the difference between while and if
