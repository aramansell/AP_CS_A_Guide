2.7a — Character Creator: StatBlock

Type: RPG Integration. You've built a Dice utility class. Now build a Character class. This is where the RPG project comes alive. This is the largest build so far — a full 3-day assignment.

RPG — Part 1 ~25 min

The Goal

Create a Character.java class that represents an RPG character with stats, name, HP, and a formatted display. The dice rolling is powered by the Dice.java utility you finished in 1.7.

Part 1: The Fields (What a Character IS)

Every character has these properties. But what data types do you use? What should HP be capped by? What stats does an RPG character have? Check the Character Creator guide for the full spec — it lists every field and the recommended types.

Socratic Checkpoint

  1. Why are these instance variables (not static)? What happens if you make them static?
  2. Why does HP need two variables? What's the difference between current HP and maximum HP?
  3. What data type did you choose for the stats? Why not double?

Part 2: generateStats() — Summon the Dice God

Add a method that fills in all six stats automatically. The formula for each stat is in your Dice.java from Unit 1.7. The HP formula is in the Character Creator guide.

Requirements:

  • Method should be public void generateStats()
  • Use Dice.statRoll() for each stat
  • HP scales with constitution — check the docs for the exact formula
  • Character starts at full health

Tinker Challenge

Instead of 3d6 for every stat, give warriors a boost: roll 4d6 for strength and constitution, but only 2d6 for intelligence. Use Dice's methods to build this out.

Acceptance Test:

Character c = new Character();
c.name = "Grog";
c.generateStats();
System.out.println(c.strength); // should be 3-18
System.out.println(c.maxHp);    // should be constitution * 2
