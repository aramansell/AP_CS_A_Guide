4.6a — Dungeon Loop: Combat While Loop

Type: RPG Integration. You've studied while loops, for loops, nested loops, and loop analysis. Now you put all four concepts into one system: a dungeon crawler where combat uses while loops and room exploration uses for loops. This is a 3-day assignment.

RPG Build ~25 min

The Goal

Create a DungeonLoop.java class. Today you build the combat engine — a while-loop that keeps two combatants fighting until one falls. You already wrote combat logic in Unit 3's Combat.java. Now you wrap it in a loop that runs an unknown number of rounds.

Open the Dungeon Loop System (Student RPG Guide). Follow the specification for Part 1: runCombat().

Part 1: runCombat() — The While Loop

Create DungeonLoop.java. Add this method:

public static boolean runCombat(String playerName, int playerHP, int playerAC,
                                int playerAttackBonus, String monsterName,
                                int monsterHP, int monsterAC,
                                int monsterAttackBonus, int monsterDamageDie)

Requirements:

  • Print a combat header: "=== COMBAT: [player] vs [monster] ==="
  • Use a while loop. Condition: playerHP > 0 && monsterHP > 0
  • Each iteration: print current HP, player attacks, monster counterattacks (only if still alive)
  • Attack pattern: roll d20, add bonus. Natural 20 = critical (d8 × 2). Total ≥ AC = hit (d8). Otherwise miss.
  • After loop: return true if player won, false otherwise

Socratic Checkpoint

  1. You don't know how many rounds this fight will last. Why is this a while loop and not a for loop? What's the rule for choosing?
  2. Why does the monster only counterattack when monsterHP > 0? What would happen if you removed that check?
  3. Trace through one round: player attacks, damage is subtracted, then monster attacks. Does the order matter? What if the monster went first?

Tinker: Variable Damage

Right now both sides roll d8 for damage. Make the player's damage die a parameter too. Then make the monster's critical hit do double damage (dmgDie × 2 on a natural 20). Does the fight feel more dangerous?

Acceptance Test:

boolean result = DungeonLoop.runCombat("Hero", 20, 14, 3, "Goblin", 10, 12, 2, 6);
System.out.println("Result: " + (result ? "Won" : "Lost"));
// Run 5 times. Verify combat ends when one side hits 0 HP.
// Verify the boolean matches the winner.
