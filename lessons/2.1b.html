2.1b — Object Inspection: Call Methods, Read Fields

Basic Application ~20 min

Build: Object Inspector

Yesterday you discovered that objects have state (fields) and behavior (methods). Today you'll create multiple objects and inspect them — calling their methods, reading their fields, and seeing how each object is independent.

Create a class called ObjectInspector with a main method:

  1. Create two String objects with different values: "Hello" and "Goodbye".
  2. Call .length(), .toUpperCase(), and .charAt(0) on each. Print all results.
  3. Create a Rectangle object (from the engine): new Rectangle(0, 0, 50, 30). Call getWidth() and getHeight().
  4. Create a second Rectangle with different values. Call the same methods. Are the results the same?

Observe

Two objects of the same class, created with different constructor arguments, return different values from the same method call. What does this tell you about where the data lives?
