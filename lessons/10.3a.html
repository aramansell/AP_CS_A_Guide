10.3a — Merge Sort: Divide and Conquer

Discovery Lab ~25 min

Selection sort from Unit 7 takes O(N²) time — 10,000 items means ~100 million comparisons. Merge sort is a recursive sorting algorithm that's much faster: O(N log N). The idea is simple: split the array in half, sort each half recursively, then merge the two sorted halves together.

Experiment: Merge Two Sorted Arrays

Create MergeSort.java. First, just merge:

// Merge two sorted arrays into one sorted array
public static int[] merge(int[] left, int[] right) {
    int[] result = new int[left.length + right.length];
    int i = 0, j = 0, k = 0;

    while (i < left.length && j < right.length) {
        if (left[i] <= right[j]) {
            result[k] = left[i];
            i++;
        } else {
            result[k] = right[j];
            j++;
        }
        k++;
    }

    // Copy remaining elements
    while (i < left.length) { result[k] = left[i]; i++; k++; }
    while (j < right.length) { result[k] = right[j]; j++; k++; }

    return result;
}

Test: merge({1,4,7}, {2,3,6,8}) should produce {1,2,3,4,6,7,8}. Trace the while loops on paper.

Experiment: The Full Merge Sort

public static int[] mergeSort(int[] arr) {
    // Base case: array of size 0 or 1 is already sorted
    if (arr.length <= 1) return arr;

    // Split into two halves
    int mid = arr.length / 2;
    int[] left = Arrays.copyOfRange(arr, 0, mid);
    int[] right = Arrays.copyOfRange(arr, mid, arr.length);

    // Recursively sort both halves
    left = mergeSort(left);
    right = mergeSort(right);

    // Merge the sorted halves
    return merge(left, right);
}

Test with {38, 27, 43, 3, 9, 82, 10}. Draw the recursion tree — each split creates two children until you hit single elements, then merge on the way back up.

Copy Arrays, Don't Modify In-Place: Merge sort creates new arrays at every level. For N=8, you create ~15 arrays total. This is why merge sort uses O(N) extra memory. Selection sort uses O(1) extra memory — but it's much slower. Read the Merge Sort docs for the time/space trade-off.

Socratic Checkpoint

  1. Merge sort splits the array log₂(N) times. For N=1,000,000, that's about 20 levels. How many total comparisons?
  2. Why does merge sort need extra arrays? Could you implement it in-place?
  3. Read the docs section on stable sorting. Merge sort is stable — what does that mean?
