9.5c — ShapeCollection: Polymorphic Operations

Type: Expand Application. The array worked for 3 shapes. Now build a collection that can hold any number of shapes, compute totals, and filter by type — all using polymorphism.

Expand Application ~25 min

The Goal

Build a ShapeCollection that uses an ArrayList to store any mix of shapes. Add methods to compute total area, find the shape with the largest perimeter, and count how many of each type exist.

=== SHAPE COLLECTION ===
Total shapes: 6
Total area: 283.64

Circles: 2, Rectangles: 3, Triangles: 1

Build It: ShapeCollection

Create ShapeCollection.java:

import java.util.ArrayList;

public class ShapeCollection {
    private ArrayList<Shape> shapes;

    public ShapeCollection() {
        shapes = new ArrayList<>();
    }

    public void add(Shape s) {
        shapes.add(s);
    }

    public double getTotalArea() {
        double total = 0;
        for (Shape s : shapes) {
            total += s.getArea();  // WHICH getArea()? Depends on the object
        }
        return total;
    }

Socratic Checkpoint

  1. The getTotalArea() loop calls s.getArea() on every shape. How does Java know which formula to use without any if-statements?
  2. What does the instanceof keyword do? Why might you need it for countByType()?
  3. Is instanceof a sign of good design or a code smell? When is it acceptable vs. when does it mean your parent class is missing methods?

Tinker Challenge

Add a printReport() method that prints the full collection sorted by area (largest first). You'll need to adapt the selection sort from 7.3c to work with Shapes instead of Players. What changes?

Bug Hunt

If you add a new shape type (like Square that extends Rectangle), does countByType() still work? A Square IS-A Rectangle, so instanceof Rectangle returns true for Squares too. Is that what you want? How would you distinguish between a Rectangle and a Square?
