2.6b — Wrapper Classes: Parsing & Validation

Basic Application ~25 min

When users type input, it always arrives as a String. But game logic needs numbers. Wrapper classes bridge that gap — they convert Strings to numbers and numbers back to Strings. Today you build programs that validate and convert player input.

Reminder: Integer.parseInt("hello") crashes with a NumberFormatException. You must either validate input before parsing or handle the error. Today we validate first.

Build 1: Stat Input Validator

Create StatInput.java. The program reads three lines of input and converts each to a number:

  • Read a strength value as String → parse to int
  • Read a luck value as String → parse to double
  • Read an alive status as String → parse to boolean
// Example session:
Enter strength: 15
Enter luck: 0.75
Enter alive (true/false): true

Parsed: strength=15, luck=0.75, alive=true

Socratic Checkpoint

  1. You used Integer.parseInt() for strength. What wrapper class parses doubles? What parses booleans?
  2. Boolean.parseBoolean() is unusual. Try it with "true", "false", "yes", and "goblin". What happens with non-boolean inputs? Does it crash or return false?
  3. Which parsing method is the most dangerous (most likely to crash)?

Build 2: Safe Parser

Create SafeParser.java. This time, don't let the program crash. For each input:

  • Try to parse the String to an int
  • If the String contains anything other than digits (and an optional minus sign), print an error and ask again
  • Keep asking until the user enters a valid number

The Wrapper docs don't show how to detect if a String is parseable — but think: what if you check every character? Open the String docs and look for methods that let you inspect individual characters.

Bug Hunt: Negative Numbers and Empty Strings

Test your SafeParser with these tricky inputs:

"-5"      // negative — is that valid?
"3.14"    // has a decimal point — should this parse as int?
""        // empty string
"   42"   // has leading spaces
"42 "     // trailing space

Which ones does Integer.parseInt() accept? Which ones crash? Does your validation handle them the same way?

Build 3: Stat Range Checker

Add range validation using wrapper constants:

  • Strength must be between 1 and 20
  • If the player enters 25, print "Strength cannot exceed 20" and ask again
  • Use Integer.MIN_VALUE and Integer.MAX_VALUE in your error messages to show the valid range

Checkpoint — before moving on, you must be able to:

  • Use Integer.parseInt(), Double.parseDouble(), and Boolean.parseBoolean()
  • Validate input before parsing to avoid NumberFormatException
  • Use wrapper constants (MAX_VALUE, MIN_VALUE) in range checks
  • Explain why Boolean.parseBoolean() behaves differently from the other parsers
