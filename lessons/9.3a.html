9.3a — Polymorphism: One Interface, Many Forms

Discovery Lab ~25 min

Polymorphism (Greek: "many forms") is what makes inheritance useful. A single variable of type Creature can hold a Goblin, a Dragon, or a Skeleton — and calling attack() runs the right version automatically. This means you can write code that works with the PARENT type and it handles all children correctly, even children you haven't written yet.

Experiment: Polymorphic Battle

Create BattleArena.java. No Goblin or Dragon types anywhere — only Creature:

public class BattleArena {
    // This method works with ANY Creature, present or future
    public static void battle(Creature a, Creature b) {
        System.out.println(a.getName() + " vs " + b.getName() + "!");
        while (a.isAlive() && b.isAlive()) {
            b.takeDamage(a.attack());
            if (b.isAlive()) a.takeDamage(b.attack());
        }
        Creature winner = a.isAlive() ? a : b;
        System.out.println(winner.getName() + " wins!");
    }

    public static void main(String[] args) {
        battle(new Goblin(), new Dragon());
        battle(new Skeleton(), new Goblin());
        // Add a new creature subclass in 3 weeks — battle() still works
    }
}

Read that code carefully. There is zero mention of Goblin, Dragon, or Skeleton in the battle() method. It works with ANY Creature. That's polymorphism.

Parent Reference = Limited Method Access: A Creature variable can only call methods declared in Creature. If Dragon has breatheFire() but Creature doesn't, you CANNOT call c.breatheFire() on a Creature reference, even if the object is a Dragon. You'd need to cast: ((Dragon)c).breatheFire(). Read the Downcasting docs — and the Bug Hunt for when casting goes wrong.

Build: Creature Army

Create CreatureArmy.java:

  • Build an ArrayList<Creature> with 10 creatures of mixed types
  • getTotalAttackPower() — sum of all creature attack values
  • getLivingCount() — count of creatures with hp > 0
  • applyDamageToAll(int damage) — every creature takes damage
  • getStrongest() — returns the Creature with highest attackPower
  • Test: apply 15 damage to all, then print who's still alive and the strongest survivor

Socratic Checkpoint

  1. Why does polymorphism let you add new Creature subclasses later without changing battle()?
  2. If you override attack() in a child but accidentally write public int attak() (typo), what happens? Does your override work? Does @Override save you?
  3. Read the docs section on static vs dynamic binding. Which method calls are resolved at compile time and which at runtime?

Checkpoint — before moving on, you must be able to:

  • Write polymorphic code that works with parent types
  • Store mixed child objects in parent-typed collections
  • Explain the difference between compile-time type and runtime type
  • Identify when a method call on a parent reference will fail (method not in parent)
