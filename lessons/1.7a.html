1.7a — Dice Roller: roll() & rollDice()

Type: RPG Integration. This is where everything comes together. You've discovered the concepts, built basic applications, and now you apply it all to your RPG project. Instead of giving you code, this guide provides Acceptance Criteria — it's up to you to write code that satisfies the spec.

RPG — Part 1 ~20 min

The Setup

Create two files in your project:

  1. Dice.java — contains all your dice logic
  2. DiceTester.java — contains a main method to run tests

In Dice.java, start with a basic class outline:

public class Dice {
    // Your methods will go here
}

Part 1: roll() — Roll a Single Die

Requirements:

  • Method signature: public static int roll(int sides)
  • Generates a random integer between 1 and sides (inclusive)
  • This is the formula you discovered in 1.3 — parameterize it

Socratic Checkpoint

Look back at your Math.random() lab. What is the "range" based on the sides parameter? What is the "min"?

Acceptance Test:

System.out.println("Rolling d20: " + Dice.roll(20));
System.out.println("Rolling d6: " + Dice.roll(6));

Run it many times. Do you ever see 0? Do you ever see a number higher than the sides? If so, your formula is wrong.

Part 2: rollDice() — Roll Multiple Dice and Sum

Requirements:

  • Method signature: public static int rollDice(int count, int sides)
  • Rolls count dice, each with sides, returns the sum
  • Since we haven't learned loops yet, hardcode it to call roll(sides) three times and add the results

Tinker Challenge

How do you store a "running total" across multiple lines? You need a variable that starts at 0 and accumulates each roll.

Acceptance Test:

System.out.println("Rolling 3d6: " + Dice.rollDice(3, 6));

Run it. Results should be between 3 and 18.

Checkpoint — before moving on, you must be able to:

  • Write a method that parameterizes the random formula
  • Call your own method from another method (roll() from rollDice())
  • Use a running total to accumulate multiple dice rolls
