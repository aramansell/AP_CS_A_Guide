4.3c — String Traversal: Build a Scroll Decoder

Expand Application ~30 min

Ancient scrolls in the dungeon are encoded with a Vigenère cipher — a Caesar cipher where the shift changes for each character based on a keyword. Today you decrypt one.

Vigenère cipher: Like Caesar, but the shift varies. If the keyword is "KEY", the first letter shifts by K (10), second by E (4), third by Y (24), fourth by K again, and so on.

Build: ScrollDecoder.java

The encoded message is: "UYMKPV XQJ WQJ"

The keyword is: "CIPHER"

Decode it:

  1. For each character in the encoded message:
  2. Get the corresponding keyword character (keyword[i % keyword.length()])
  3. Calculate the shift: keyword char - 'A' (e.g., 'C' - 'A' = 2)
  4. Shift the encoded char BACKWARD by that amount (Caesar decrypt)
  5. Spaces pass through unchanged

Socratic Checkpoint

  1. How do you access keyword.charAt(i % keyword.length())? Why modulo the index by the keyword length?
  2. When you shift backward past 'A', where should you wrap to? What's the formula?
  3. If the keyword contained lowercase letters, how would you handle them? Could you convert everything to uppercase first?

Build 2: Message Analyzer

Create MessageAnalyzer.java that answers questions about a decrypted message:

  • Count each vowel (a, e, i, o, u) separately — not just a total
  • Find the most frequent character (ignoring spaces)
  • Print a frequency table: "a: 3, b: 1, c: 0, ..."

For the frequency table, you'll need an array of 26 counters. Loop through the message and increment the right counter based on each character. Think: counts[ch - 'a']++.

Tinker: Auto-Decrypt (Frequency Analysis)

If you don't know the keyword, you can still break the cipher. In English, 'e' is the most common letter. Count the most frequent letter in the encoded message — if it's 'M', and 'e' should be most common, the shift is probably 8 (M - E = 8). Try it: can you auto-detect the shift for a multi-paragraph encrypted text?

Bug Hunt: The Wrapping Formula

A student wrote decoded = (char)((encoded - 'A' - shift) % 26 + 'A'). Test it with encoded='A', shift=1. The result should be 'Z'. What does Java's modulo do with negative numbers in this context? Does it give you what you want?

Checkpoint — before moving on, you must be able to:

  • Implement a Vigenère cipher using indexed keyword access with modulo
  • Build frequency tables using character-to-array-index mapping
  • Handle negative modulo wrap-around for decryption
  • Use Character methods (isLetter, toUpperCase) from the Wrapper docs
