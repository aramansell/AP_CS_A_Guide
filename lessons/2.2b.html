2.2b — String Methods: Apply and Build

Basic Application ~25 min

Yesterday you explored String methods by experimenting. Today you apply them to solve problems. You'll write methods that take Strings in and return something useful out — exactly what RPG systems do behind the scenes.

Recap: The Methods You'll Need

Open the String docs and have them ready. You'll use:

  • length() — how many characters
  • substring(begin, end) — extract a piece
  • indexOf(String) — find where something starts
  • equals(Object) — compare contents
  • toUpperCase() / toLowerCase() — case conversion
  • trim() — strip leading/trailing whitespace

Trap Review: == vs equals()

Before you start, make sure you can answer this: why does "sword" == "sword" sometimes work but a.equals(b) always works? If you can't explain it, re-read the Objects & Classes docs — look for the section on reference comparison.

Build 1: Weapon Name Formatter

Create WeaponName.java. Write a method called formatWeapon that takes a raw weapon name and returns it properly formatted:

  • Trim whitespace from both ends
  • Convert to lowercase
  • Capitalize the first letter
// input → output examples (test these):
"sWoRD  "   → "Sword"
"   aXE"   → "Axe"
"BOW"       → "Bow"

You'll need a way to get the first letter, convert it to uppercase, and combine it with the rest of the word. The String docs show you how. This is a two-step process — don't try to do it in one line.

Socratic Checkpoint

  1. What happens if your method receives an empty string ""? Does it crash? Test it. If it does, what check could you add?
  2. What happens if it receives null? Try it. This is a different kind of error. What's it called?

Build 2: Email Extractor

Create EmailExtractor.java. In RPGs, items often have metadata embedded in strings. Given a string like "player@dungeon", extract:

  • The username — everything before the '@'
  • The domain — everything after the '@'
// input → output:
"warrior@dungeon"  → username: "warrior", domain: "dungeon"
"mage@castle.org"  → username: "mage",    domain: "castle.org"

You'll need indexOf("@") to find the split point, then substring to extract both halves. The String docs have examples of exactly this pattern.

Bug Hunt: What If There's No '@'?

Run your EmailExtractor with "justaname" — a string with no '@'. What happens? Where does indexOf return -1, and why does that break substring?

Fix it: if the '@' isn't found, return the original string as the username and "" as the domain. You'll need an if statement. If you're not sure how, open the Boolean Expressions docs for a refresher.

Checkpoint — before moving on, you must be able to:

  • Combine substring, indexOf, and trim to parse structured strings
  • Handle edge cases: empty strings, missing delimiters
  • Use equals() (not ==) to compare String contents
  • Explain what happens when you call a method on a null reference
