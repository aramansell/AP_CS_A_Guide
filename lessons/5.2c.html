5.2c — Enemy Factory: Scaling Monsters by Level

Expand Application ~30 min

Build: Enemy Class with Scaling

In RPGs, enemies scale with player level. A level-1 goblin has 10 HP; a level-10 goblin has 100. You'll build a constructor system that handles this automatically.

Create a class called Enemy:

  1. Instance variables: String type, int level, int hp, int maxHp, int attack, int defense.
  2. Constructor that takes type and level.
  3. Calculate stats based on type AND level:
    • Goblin: base HP=10, ATK=3, DEF=2 — scale by 1.5× per level
    • Skeleton: base HP=15, ATK=5, DEF=3 — scale by 1.3× per level
    • Dragon: base HP=50, ATK=12, DEF=8 — scale by 2.0× per level
  4. Overloaded constructor: takes type only — defaults to level 1.

Verify Scaling

Create a level-1 goblin and a level-5 goblin. Print their stats side by side. Does the scaling make sense? Is a level-5 goblin roughly 5× as strong?
