3.4c — Command Interpreter: Text-Driven Combat

Expand Application ~30 min

Build: Combat Command Parser

Text-based games let players type commands like "attack", "defend", or "flee". Your job is to build the parser that interprets those commands — using everything you know about String comparison and if/else-if.

Create a class called CommandInterpreter. Hardcode a String command (test with "attack", "ATTACK", "defend", "flee", "dance", and a misspelling).

Rules

  1. Use .equalsIgnoreCase() so "attack", "Attack", and "ATTACK" all work.
  2. "attack" → print damage dealt (hardcode a number).
  3. "defend" → print "You brace for impact. Damage reduced by 50%."
  4. "flee" → print "You attempt to escape!" then check a boolean canFlee variable — if true, escape; if false, "The enemy blocks your path!"
  5. "status" → print current HP and MP (hardcoded values).
  6. Anything else → print "Unknown command. Try: attack, defend, flee, status."

Extend: Command Prefixes

What if a player types "attack goblin"? Use .startsWith("attack") to detect attack commands that include a target. Print "You attack the [target]!" by extracting the target from the command string.
