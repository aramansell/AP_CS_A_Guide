5.4a — Methods & Static: Behavior That Belongs to the Class

Discovery Lab ~25 min

Every method you've written so far has been tied to an object — sword.getDamage() reads from one specific sword. But some behavior doesn't belong to any one object. It belongs to the class itself. The static keyword makes this possible.

Experiment: Static vs Instance

Add a static kill counter to your Monster class:

private static int defeatedCount = 0;

public void defeat() {
    hp = 0;
    defeatedCount++;
}

public static int getDefeatedCount() {
    return defeatedCount;
}

Test it: create three monsters, call defeat() on two of them, then print Monster.getDefeatedCount().

Notice: you call it on the CLASS, not on an instance. Now add a method that DOESN'T work:

public static String getFormattedCount() {
    return name + " kills: " + defeatedCount;  // COMPILE ERROR
}

Read the error. Why can't a static method use name?

Static Context Trap: Static methods can only access static variables and other static methods. They have no this — there's no object to refer to. This is a common FRQ mistake. If you see "non-static variable cannot be referenced from a static context," you're trying to use an instance variable inside a static method.

Build: Combat Formulas

Create CombatFormulas.java — a utility class with all static methods:

  • static int rollD20() — returns 1-20
  • static boolean isHit(int roll, int bonus, int ac) — true if roll + bonus >= ac
  • static int rollDamage(int dieSize) — returns 1-dieSize
  • static String combatResult(int roll, int bonus, int ac) — returns "HIT", "MISS", or "CRITICAL HIT!"

Test by simulating 20 attacks and counting hits vs misses.

Socratic Checkpoint

  1. Your CombatFormulas class has only static methods. Could you write new CombatFormulas()? Should you? What's the purpose of a class that you never instantiate?
  2. Math.random() and Math.sqrt(25) are static. Look up the static methods docs. Why did the Java designers make these static instead of instance methods?
  3. If defeatedCount were NOT static, what would happen to the count? Would each monster track its own defeats, or would they share one count?

Checkpoint — before moving on, you must be able to:

  • Distinguish instance methods from static methods
  • Explain why static methods can't access instance variables
  • Create utility classes with all static methods
  • Use static variables for shared counts/totals
