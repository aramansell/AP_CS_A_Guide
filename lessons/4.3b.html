4.3b — String Traversal: Build a Cipher

Basic Application ~25 min

RPGs often hide secrets in scrambled text — encrypted messages, coded scrolls, password-protected chests. String traversal is the foundation of all text manipulation. Today you build a cipher system.

Build 1: The Caesar Cipher

Create CaesarCipher.java. The Caesar cipher shifts every letter by the same amount:

// Shift of 1: A→B, B→C, ..., Z→A
"ATTACK" + shift 1 → "BUUBDL"
"BUUBDL" + shift -1 → "ATTACK"

// Shift of 3 (the classic):
"DRAGON" + shift 3 → "GUDJRQ"

You'll need:

  • A for loop that visits every character
  • charAt(i) to get each character
  • Cast the char to an int to get its ASCII/Unicode value, add the shift, cast back
  • Wrap around: 'Z' + 1 should become 'A', not '['
ASCII Trap: Casting char to int gives you the Unicode code point. 'A' is 65, 'Z' is 90, 'a' is 97, 'z' is 122. If you shift 'Z' by 1, you get 91 (which is '['). You need to detect when a character passes 'Z' or 'z' and wrap it back to 'A' or 'a'. The % operator can help — but there's a trick to getting the math right. Open the String Loops docs for the pattern.

Socratic Checkpoint

  1. You only want to shift letters. What about spaces, numbers, and punctuation? Should they pass through unchanged?
  2. If the shift is 30 (larger than 26), what should happen? Hint: shift of 26 brings every letter back to itself. How do you normalize a large shift?
  3. What's the difference between uppercase and lowercase ASCII ranges? Does your cipher handle both?

Build 2: Reverse Cipher

Create ReverseCipher.java. Simpler than Caesar: just reverse the string.

"dragon" → "nogard"
"secret password" → "drowssap terces"

Two approaches (try both):

  • Build the reversed string by walking forward and prepending each character
  • Walk backward through the original string

Which is more efficient? Which reads more naturally?

Bug Hunt: The Index Off-by-One

The last valid index is length() - 1. A student wrote for (int i = s.length(); i >= 0; i--) to reverse the string. What happens on the first iteration when i = s.length()? Where should the loop actually start?

Checkpoint — before moving on, you must be able to:

  • Shift ASCII characters with wrap-around using modulo arithmetic
  • Skip non-letter characters during traversal (Character.isLetter())
  • Build a new string character-by-character inside a loop
  • Traverse a string in reverse without index errors
