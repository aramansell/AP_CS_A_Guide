7.1a — ArrayList Basics: An Array That Grows

Discovery Lab ~25 min

Your FixedInventory from Unit 6 has one glaring problem: it can hold exactly 20 items and no more. You coded capacity checks, "inventory full" messages, and manual shift loops. ArrayList handles ALL of that for you — it grows automatically, shifts on removal, and never runs out of space. Under the hood it's still an array, but you don't have to manage it.

Experiment: First ArrayList

Create ListIntro.java:

import java.util.ArrayList;

ArrayList<String> names = new ArrayList<String>();

// add — appends to the end, auto-grows
names.add("Gandalf");
names.add("Aragorn");
names.add("Frodo");

// size() — how many elements (NOT length!)
System.out.println(names.size());   // 3

// get(index) — access by position
System.out.println(names.get(0));   // Gandalf
System.out.println(names.get(2));   // Frodo

// set(index, value) — replace at position
names.set(1, "Boromir");
System.out.println(names.get(1));   // Boromir

// remove(index) — removes AND shifts, returns removed element
String removed = names.remove(0);
System.out.println(removed);        // Gandalf
System.out.println(names.size());   // 2

Compare every line to what you wrote in FixedInventory. What code disappeared?

Primitive Type Trap: ArrayList only stores objects, not primitives. ArrayList<int> is a compile error. Use the wrapper class: ArrayList<Integer>. Java auto-boxes for you — list.add(5) automatically wraps 5 into an Integer. Read the ArrayList Basics docs for the complete wrapper class story.

Build: Score Tracker

Create ScoreTracker.java:

  • ArrayList<Integer> to store game scores
  • addScore(int score) — appends to list
  • getAverage() — returns mean as double
  • getHighScore() — returns max value
  • getLowScore() — returns min value
  • removeBelow(int threshold) — removes all scores below threshold
  • Test with 8-10 scores. Print before and after removeBelow(50).

Socratic Checkpoint

  1. Your FixedInventory used items.length for capacity and count for actual items. ArrayList uses only size(). Where did the capacity tracking go?
  2. What happens if you call names.get(-1)? Or names.get(100)? Make a prediction, test it, read the error.
  3. Look at the docs section comparing arrays and ArrayLists. When would you choose a plain array over an ArrayList?

Checkpoint — before moving on, you must be able to:

  • Create, add to, get from, set in, and remove from ArrayList
  • Use size() correctly — never confuse with .length
  • Use wrapper classes (Integer, Double, Boolean) for primitives
  • Explain the pros/cons of ArrayList vs array
