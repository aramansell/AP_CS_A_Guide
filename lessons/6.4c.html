6.4c — Card Deck: Array of Objects

Type: Expand Application. You've built arrays of primitives — now fill an array with your own Card objects. Shuffle, deal, and print. Every card game starts here.

Expand Application ~30 min

The Goal

Build a Deck class that creates 52 Card objects, stores them in an array, shuffles them, and deals 5 cards.

=== DECK (first 5 after shuffle) ===
7 of Hearts
King of Diamonds
3 of Spades
Ace of Clubs
10 of Hearts

Reference: Card Deck Walkthrough — full reference.

Build It: Deck

Create Deck.java. You'll need your Card.java from 5.4b. Start with this skeleton:

public class Deck {
    private Card[] cards;
    private int nextCard;  // tracks which card to deal next

    public Deck() {
        cards = new Card[52];
        nextCard = 0;

        String[] suits = {"Spades", "Hearts", "Diamonds", "Clubs"};
        String[] ranks = {"Ace","2","3","4","5","6","7","8","9","10",
                          "Jack","Queen","King"};

        // YOUR CODE: nested loops to fill cards[]
        // Outer loop: suits. Inner loop: ranks.
        // Each iteration: cards[something] = new Card(suit, rank);
    }
}

Bug Hunt

You have two arrays (suits has 4, ranks has 13) and need to fill 52 slots. How do you compute the array index? If you use cards[i * 13 + j], trace it: suit index 0 rank index 0 goes to slot 0. Suit index 1 rank index 0 goes to slot 13. Check your math before coding.

Shuffle Method

Here's the shuffle algorithm. Type it yourself — don't copy-paste:

public void shuffle() {
    for (int i = 0; i < cards.length; i++) {
        int swapIndex = (int) (Math.random() * cards.length);
        Card temp = cards[i];
        cards[i] = cards[swapIndex];
        cards[swapIndex] = temp;
    }
    nextCard = 0;  // reset the deal position
}

Socratic Checkpoint

  1. Why is the swap temporary variable Card temp and not int temp? What type must it be?
  2. What does setting nextCard = 0 after shuffle do? What would happen if you left it unchanged?
  3. If you shuffled 52 cards, how many total swaps does this loop do? Is every swap productive?

Deal Method

public Card deal() {
    if (nextCard < cards.length) {
        Card c = cards[nextCard];
        nextCard++;
        return c;
    }
    return null;  // deck is empty
}
