10.2a — Recursive Traversal: Arrays, Strings, and Trees

Discovery Lab ~25 min

Recursion isn't just for math problems. It's a natural fit for anything with a self-similar structure: arrays (process first element, recurse on the rest), strings (process first character, recurse on the substring), and trees (process root, recurse on children). The pattern is always the same: handle a small piece, then recurse on what's left.

Experiment: Recursive Array Sum

Create RecursiveArray.java:

// Sum array from index to end — recursive
public static int sum(int[] arr, int index) {
    if (index >= arr.length) return 0;           // base case: nothing left
    return arr[index] + sum(arr, index + 1);     // this element + sum of rest
}

// Call: sum(myArray, 0)

// Find max — recursive
public static int max(int[] arr, int index) {
    if (index >= arr.length - 1) return arr[index];  // last element
    int restMax = max(arr, index + 1);
    return Math.max(arr[index], restMax);
}

// Count occurrences — recursive
public static int count(int[] arr, int index, int target) {
    if (index >= arr.length) return 0;
    int match = (arr[index] == target) ? 1 : 0;
    return match + count(arr, index + 1, target);
}

Convert each to an iterative version. Compare line counts and readability.

Index Must Progress: Every recursive call MUST move closer to the base case. If you pass the same index, you get infinite recursion. The most common bug: sum(arr, index) instead of sum(arr, index + 1). Read the Recursive Traversal docs for the complete traversal patterns.

Experiment: Recursive String Processing

// Reverse a string recursively
public static String reverse(String s) {
    if (s.length() <= 1) return s;                          // base case
    return reverse(s.substring(1)) + s.charAt(0);            // recurse on tail + first char
}

// Is palindrome?
public static boolean isPalindrome(String s) {
    if (s.length() <= 1) return true;                       // base case
    if (s.charAt(0) != s.charAt(s.length() - 1)) return false;
    return isPalindrome(s.substring(1, s.length() - 1));     // strip both ends
}

Trace isPalindrome("racecar") on paper. How many calls? What substring does each call see?

Socratic Checkpoint

  1. Why does s.substring(1) guarantee progress toward the base case? When would it fail to make progress?
  2. isPalindrome() creates a new String on every call. How many String objects does isPalindrome("racecar") create?
  3. Read the docs section on tail recursion. What is it and why do some languages optimize it?

Checkpoint — before moving on, you must be able to:

  • Write recursive traversal methods for arrays and strings
  • Ensure index/string parameters progress toward base case
  • Accumulate results across recursive calls (sum, count, filter)
  • Compare recursive vs iterative traversal trade-offs
