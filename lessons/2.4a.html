2.4a — Scanner + Strings Practice

Expand Application ~30 min

Today you combine everything: Scanner to read input, String methods to process it. Pick one project below and build it. Read the docs. No copy-paste.

Rule of the day: If you get stuck, your FIRST move is to open the docs — String page or Scanner page. Try something. Only ask for help after you've made a real attempt.

Project A: Name Analyzer

Read a full name from the user and analyze it:

  • Print the full name as entered
  • Print just the first name
  • Print just the last name
  • Print the initials (first letter of first + first letter of last)
  • Print the total character count (don't count spaces)

You'll need methods from the String docs. Which ones let you find a space? Which ones let you extract part of a String? Figure out the combination.

Enter your full name: Aragorn son of Arathorn
Full name: Aragorn son of Arathorn
First name: Aragorn
Last name: Arathorn
Initials: AA
Character count (no spaces): 19

Tinker Challenge: Middle Name

What if the user has a middle name? "Aragorn Elessar Arathorn" — can you extract all three parts? Where does indexOf find the first space vs the last space?

Project B: Password Validator

Read a password from the user and check ALL of these rules:

  • At least 8 characters long
  • Contains at least one digit (0-9)
  • Contains at least one uppercase letter
  • Does NOT contain the word "password" (case-insensitive)

Print "VALID" or list every rule it failed.

Enter password: hello
FAILED:
- Must be at least 8 characters
- Must contain at least one digit
- Must contain at least one uppercase letter

Socratic Checkpoint

  1. How do you check for uppercase letters? There's no method called isUpperCase() on a String. What methods DO exist that could help? Open the String docs and find something that transforms case.
  2. How do you check if a password contains the word "password" regardless of case? You'll need to combine two String methods. The docs have all of them — which pair solves this?

Project C: Mad Libs

Build a fill-in-the-blank story generator:

  1. Read 6 words from the user: a noun, a verb, an adjective, another noun, a number, and an exclamation
  2. Insert them into a pre-written story template using concatenation
  3. Print the completed story
Noun: dragon
Verb: danced
Adjective: sparkly
Noun: potato
Number: 42
Exclamation: By the gods!
---
The sparkly dragon danced through the dungeon carrying a potato.
"By the gods!" shouted the knight. "There are 42 of them!"
Buffer Trap Alert: If you mix nextInt() (for the number) with nextLine() (for the other words), remember to eat the leftover Enter key. Where in your code does it go?

Checkpoint — before moving on, you must be able to:

  • Combine Scanner input with String processing in one program
  • Diagnose and fix buffer traps without being told where they are
  • Use substring + indexOf together to extract parts of a String
  • Find the docs page you need without asking the instructor
