2.4a — Scanner: Let the User Talk Back

Discovery Lab ~20 min

Explore: Hello Scanner

So far your programs talk AT the user. Today you'll let the user talk BACK. Java's Scanner class reads what the user types.

Create a class called ScannerExplore. Type this code exactly:

import java.util.Scanner;

public class ScannerExplore {
    public static void main(String[] args) {
        Scanner input = new Scanner(System.in);

        System.out.print("What is your name? ");
        String name = input.nextLine();

        System.out.print("How old are you? ");
        int age = input.nextInt();

        System.out.println("Hello, " + name + "! You are " + age + " years old.");
        input.close();
    }
}

Experiment

  1. Run it. Type your name and age. Does it work?
  2. Now type your full name (like "Ada Lovelace"). What happens to age? Why?
  3. After the nextInt() call, add a second input.nextLine() — what does that line do? Why is it needed?
  4. Try nextDouble() instead of nextInt(). What happens if you type "twelve" instead of 12?

Document

Answer in your lab notes:

  1. What does new Scanner(System.in) do?
  2. What's the difference between nextLine() and nextInt()?
  3. What's the "leftover newline" problem and how do you fix it?
