6.5a — Item.java: Immutable RPG Loot

RPG Build ~20 min

Your Player has stats. Your Monster has attacks. But RPG characters carry things — weapons, potions, armor, keys. An inventory starts with a single item. Before you can build a backpack, you need to model what goes inside it.

Open the Inventory Builder (Student RPG Guide). This is a step-by-step guided build. Follow File 1: Item.java. Your completed Item class must pass the acceptance tests.

Build: Item.java

Create Item.java — immutable, like Card from Unit 5:

  • Private final fields: name (String), weight (int), goldValue (int), rarity (String)
  • Constructor takes all four. Validate rarity: must be "Common", "Uncommon", "Rare", or "Epic". Reject with a clear message if invalid.
  • Validate weight: 1–50. Validate goldValue: ≥ 0.
  • Getters for all fields
  • toString(): "Sword [Epic] — Wt:12 Val:250g"

Test: create items of each rarity. Try creating one with invalid rarity — does it reject it? What about weight 0? weight 51?

Socratic Checkpoint

  1. Item is immutable — all fields are final. Once created, you can't change a Sword into a Potion. Is that a problem? When WOULD you want an item to change?
  2. Why validate rarity in the constructor? What happens if you don't, and someone creates new Item("Sword", 10, 100, "Legendary") when "Legendary" isn't in your enum?

Acceptance — your Item.java must:

  • Reject invalid rarity with a clear message
  • Reject weight outside 1–50
  • Reject negative goldValue
  • Pass all acceptance tests in the Student RPG Guide
