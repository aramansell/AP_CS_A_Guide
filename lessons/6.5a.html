6.5a — Inventory Builder: Item + FixedInventory

RPG Build ~60 min

Your Player has stats. Your Monster has attacks. But RPG characters carry things — weapons, potions, armor, keys. It's time to build an inventory system. You'll create an Item class and a FixedInventory that uses a plain array under the hood with a tracking variable for how many items are actually stored.

Open the Inventory Builder (Student RPG Guide). This is a step-by-step guided build with acceptance tests. Follow the specification exactly.

Part 1: Item.java

Create Item.java:

  • Private fields: name (String), weight (int), value (int), rarity (String)
  • Constructor takes all four. Validate rarity: must be "Common", "Uncommon", "Rare", or "Epic"
  • Getters for all fields
  • toString(): "Sword [Rare] — Wt:10 Val:100g"

Test: create items of each rarity. Try creating one with invalid rarity — does it reject it?

Part 2: FixedInventory.java

Create FixedInventory.java:

  • Private fields: Item[] items and int count
  • Constructor takes capacity, creates the array
  • add(Item item) — adds at position count, then count++. Check capacity first.
  • remove(String name) — find first match by name, shift left to close the gap
  • contains(String name) — returns true if any item matches
  • getTotalValue() — sum of all item values
  • display() — prints each item in a numbered list

Run the acceptance tests from the Student RPG Guide.

Integrate Into Your RPG

Add a FixedInventory field to your Player class (or keep it separate in DungeonCrawl). After combat victories, loot the monster:

inventory.add(new Item("Goblin Sword", 8, 25, "Common"));

Add a "show inventory" option to your game loop. Print total value.

Socratic Checkpoint

  1. Why do you need both items.length (capacity) and count (how many are actually stored)? Why not just use items.length for both?
  2. Your remove() shifts elements left. If you remove the first item from a full inventory, how many shifts happen? What if you remove the last item?

Checkpoint — before moving on, you must be able to:

  • Build an array-based collection with a count tracking variable
  • Add, remove, and search items with proper shift and null handling
  • Integrate inventory into the RPG game loop
